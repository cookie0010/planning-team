## 개념 모델 (prd-v9 §2 = 권위)
- 계층 = **프로젝트 > 그룹(선택) > 캔버스 > 페이지 > 객체**. 프로젝트=최상위 통(PoC=사실상 1개, 다중 프로젝트 화면 비대상). 그룹 = '프로젝트 내 캔버스 묶음'(폴더, 그룹명+소속 캔버스 썸네일). 복제 단위 = **캔버스**(반별 재사용, 복제본 독립·QR 각자).
- 캔버스: **권한별 QR/URL**·학생 진입 단위·**진입자 20명(편집자·참여자·뷰어 모두 포함)**·최대 30페이지·상태(진행중/잠금/보관).
- 페이지: 편집자가 템플릿 붙여 구성, 좌우 추가·이동, 최대 30(고정).
- 역할 **4단계**(prd-v7에서 뷰어 추가): **소유자 / 편집자 / 참여자 / 뷰어**. 편집자 임명 단위=캔버스, 로그인 필수. 참여자=객체 작업+공감투표, 페이지 추가·템플릿 수정 불가. 뷰어=보기 전용(객체·페이지 변경 불가, 공감투표 기본 불가=OQ-21).
- 공유=공유 패널에서 **초대 권한(편집자/참여자/뷰어) 먼저 선택 → 권한별 별개 QR/URL 발급**(prd-v8 OQ-22 해소). 비로그인 참여자·뷰어=**닉네임+비밀번호 4글자(고정·필수)**. 로그인 참여자·뷰어=재진입 혜택(뷰어는 보기 전용 재진입, prd-v9).
- 진입: 퀴즈앤에서 **링크 이동**(데이터·도메인 별개, v2 내부 진입점 설계 범위 밖).

## ⭐ ia-v7 → v8 핵심 결정 (교사 area 전체 HTML — A 빈상태·B·C에서 역반영)
- **① T → 공용 모달.** 'T 템플릿 선택'을 독립 화면에서 빼고 **새 캔버스 모달**(공용 `wf-screen.js`)로 정정. A·A-빈상태·A-그룹 내부 + **B [+페이지 추가]**에서 같은 코드로 뜸. **페이지 추가 = 그 페이지에 깔 템플릿 고르기(F3)**. F1이 이미 자동/수동 동일 흐름·'템플릿 선택 화면/모달'을 규정하므로 화면 결정이지 새 스펙 아님.
- **② A 카드·폴더 인터랙션 단순화.** 카드 콘텐츠 클릭=캔버스 열기(→B), 폴더 클릭=그룹 내부. **'열기'·'크게 보기'·'그룹 열기' 버튼 전부 제거.** 카드 메타=**'참여 N/20'만**(객체 수·페이지 수 표기 삭제).
- **③ A-빈 상태 + C 캔버스 설정 화면 추가**(v7 '다음 패스'였던 것). A-빈상태=중앙 단일 CTA→모달. C(F6)=편집자 목록(임명/임명해제·QR초대 안내) + 익명 토글 + 닉네임+비번4(끌 수 없는 고정) + 캔버스 삭제. **B [설정]에서 진입, 브레드크럼('← 1모둠 편집/설정')으로 B 복귀.**
- **④ B 포스트잇 = 작성자명(좌하단 상시) + 감정표현 버튼 ☺(우하단).** 감정표현=F10 공감/반응 계열, E-참여자 공감투표와 같은 계열 — **통일 여부는 미정**(§4, OQ-16).
- **⚠️⚠️ 상위(PRD)로 돌려보낼 갭 1건 — F7 드릴다운 진입점.** ②로 카드 클릭을 'B 열기'로 단순화하며 **F7 드릴다운(자기 화면 확대 열람) 전용 진입점이 사라짐**. F7은 '크게 열어 들여다보기(원본 열람·진입)+타이머·잠금'을 P0로 요구. 화면으로 못 메움 — **제품 질문: 드릴다운(읽기 중심 확대 열람)과 편집(B 진입)이 같은 동작인가 다른가?** 같으면 카드 클릭=B로 OK(F7 서술 정리), 다르면 드릴다운 진입점 재도입. OQ-20과 엮임 → **@prd-writer 라우팅 권장**. 그 전까지 IA는 카드 클릭=B를 잠정값, ⑥·§4·§5에 갭 표시. ia-v8 §5에 상세.

## ⭐ ia-v6 → v7 핵심 결정 (B 편집 + A 카드, HTML 와이어프레임에서 역반영)
- **B(편집자 뷰) 4영역 구체화** — ① 상단바: 캔버스명+상태 / **타이머 제어**(일시정지 ⏸ + ▾드롭다운: 잔여시간·정지·리셋·+5/+10/+30/+60분) / **잠금** 버튼(타이머 옆, 수동 잠금) / presence / [설정]→C / [공유]. ② **페이지 레일**: 페이지 칩 + [+페이지 추가] + **N/30 표시**. ③ 캔버스: 객체에 **작성자명 상시 표시**(편집자 뷰는 항상), 템플릿=페이지별. ④ 하단 **플로팅 도구 팔레트**(선택·이동 | 텍스트·포스트잇·펜·도형 | +더보기 | 취소·다시) — **도구 목록 미확정(예시)**.
- **공유 패널** = [공유]→오버레이, 권한 탭(참여자 기본/편집자/뷰어) → 권한별 QR·URL·[링크 복사] (F6). 학생 입장(D) 분기 결정.
- **⚠️ v6 정정 1 — 카드 ⋮ 메뉴 = 복사하기·보관·삭제(모든 카드 상시).** v6은 "보관·삭제, 잠금 카드만 복제"였으나 와이어프레임은 진행중·잠금 가리지 않고 **복사하기(=복제 F2) 상시**. 화면 결정이지 새 스펙 아님(F2는 어느 캔버스든 복제).
- **⚠️ v6 정정 2 — A 상태 칩 = 전체/진행 중/잠금(잠금은 배지+필터 둘 다).** v6은 "잠금=배지일 뿐 필터 아님, 칩=전체·진행중·보관"이었으나 `t-a-list-v2.html`은 잠금을 필터 칩에 둠. **'보관'이 칩 줄에 끼는지 별도 보기인지는 미정**(§4). 상태 enum 3종(진행중/잠금/보관)은 유지.
- **이번 범위 밖(다음 패스)**: A **빈 상태** + **C 캔버스 설정**은 작업 중 — v7 미반영, C는 v5/v6 정의 유지.

## ⭐ ia-v5 → v6 핵심 결정 (A 화면, HTML 와이어프레임에서 역으로 반영)
- **그룹 위상 승격**: v5='A 위 정리 동작(새 화면 아님)' → v6=**A의 드릴인 하위 뷰(A-그룹 내부)**. 그룹은 Figma 프로젝트 폴더처럼 미니썸네일 2×2+그룹명+캔버스개수 폴더 카드로 A 그리드에 캔버스 카드와 섞여 표시, **클릭하면 A-그룹 내부(그 그룹 캔버스 목록)로 진입**, 브레드크럼으로 A 복귀. A와 A-그룹 내부는 같은 종류 화면(캔버스 카드 그리드), 담는 내용만 다름.
- **캔버스가 주연, 그룹은 부수.** 캔버스=라이브 썸네일 카드 직접 노출.
- **생성 동선**: A 헤더 [+캔버스 생성](→T)·[그룹 만들기], 그리드 맨 앞 '새 그룹' 타일, 그룹 내부 [+이 그룹에 캔버스].
- **상태 ≠ 필터**: 상태 enum 3종(진행중/잠금/보관) 유지하되 **잠금은 카드 배지일 뿐 필터 칩 아님**(필터 칩=전체/진행중/보관 2보기).
- **'지휘석'은 A의 개념적 역할로만, UI 라벨 노출 안 함**(드릴다운=카드 '크게 보기').
- 학생 area(B·C·D·E·F·G)는 v6에서 안 건드림 — v5 그대로(뷰어 정원 포함·G 뷰어 재진입 등).
