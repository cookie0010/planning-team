Softn의 멀티-서비스/플랫폼 아이디어를 다룰 때, "개별 신규 제품이 충분히 매력적인가"를 단일 최대 가정으로 잡지 말 것. 대신 "기존 사용자 기반을 신규 서비스로 흘려보내는 distribution 레버리지"를 핵심 프레임으로 삼아라.

**Why:** Softn ID 세션에서 "첫 PoC 매력도가 단일 최대 가정"이라고 압박하자 사용자가 재프레이밍함 — 진짜 목적은 퀴즈앤 사용자 기반을 신규 서비스의 배포 채널로 전환하는 것. 이 관점이 더 정확했다.

**How to apply:** distribution 가설을 핵심 검증 대상으로 두되, "접근성은 trial의 필요조건이지 충분조건은 아니다"(형편없는 PoC까지 쓰게 만들진 못함)는 균형만 짧게 짚을 것. 또한 발견(discovery)과 시도-마찰-제거(trial)를 분리해서 볼 것 — 통합 계정은 후자만 푼다.
