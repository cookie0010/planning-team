Softn ID = sourced_id 기반 통합 계정(카카오·구글 소셜로그인) + 통합 홈페이지(홈페이지/Labs/Blog).

핵심 정의: 통합 계정은 **"시도 마찰 제거(frictionless trial) 레이어"**다 — 발견(discovery)을 푸는 도구가 아니라, 알게 된 서비스를 재가입 없이 원클릭으로 써보게 하는 레이어. (사용자와 합의됨)

**Why:** 12개월 내 5개+ 서비스 출시 확정. 앵커=퀴즈앤(교육 도메인)이 기존 사용자 기반 보유 → 신규 PoC의 배포 레버리지. distribution-first 목적.

**🔔 최신 결정(2026-06-08): ① 통합 홈페이지(포털) 보류 해제 → 진행(이번 PRD 본체). ② softn-id는 결정 사항 → PRD 직행(1-pager 건너뜀), 지금은 PRD 전 내용 구체화 단계. ③ 무게중심 = 지표 목표치/마이그레이션 → "동작하는 UI 프로토타입 우선"(화면으로 설득). ④ UI 레퍼런스 = 토스 'Apps in Toss'(toss.im/apps-in-toss, /blog) — 플랫폼 안 미니앱 진열 구조.**

**의도적 스코프 아웃:** 지표 목표치(프로토로 먼저 검증), SSO 신원 기반, 퀴즈앤 내 진입점 설계. 첫 화면 주인공=교사.

**확정된 화면 설계:**
- IA: Softn 홈 3탭 [소개/Blog/Labs] + 별도 [로그인].
- [1] 랜딩: 토스식 빅 히어로. 헤드라인 "교실의 모든 순간을, 더 가볍게", CTA "Labs 둘러보기".
- [A] 로그인(초안): "하나의 계정으로, 소프트앤의 모든 도구를" + 카카오·구글 + "이미 퀴즈앤 회원?" 분기. 톤 추후 재정렬.
- [3] Labs 탭: 토스 블로그 리스트 스타일, 필터=도메인(교육·평가·수업운영). 항목→Lab 상세→[써보기]→로그인[A]→try.
- [3-상세] Lab 상세: 설계 중. 스크롤 스토리텔링(제목+서브+써보기 → 데모블록 → "이런 적 있으셨죠" 공감 → 작동 3스텝 → 반복CTA → "함께 보면 좋은 Labs" → 푸터).
- 둘러보기 전략 4종 화면 매핑: ①콘텐츠훅=Labs 리스트 제목/훅 ②체험데모=상세 데모블록 ③내러티브=상세 공감섹션 ④관련추천=상세 하단. 한계: 무관 도메인 간 둘러보기 약함 → 교육 도메인 안으로 한정.

**열린 질문(PRD 전 닫을 것):** ①프로토타입 학습 질문(지표 목표치 대신 성공/실패 판단기준) — 가장 큰 구멍. ②[3-상세] 써보기 클릭 시 로그인/비로그인 분기 UX. ③데모블록 충실도(정적/GIF/임베드, 파트너=GIF추천). ④account linking 자발적 연동 정체 리스크(계속 미검증).

**상태/다음 단계:** [3-상세] 확정 → [A] 로그인 톤 재정렬 → [2] Blog(경량) → 전체 흐름 확정 → idea-deep-dive/one-pager 갱신 → @prd-writer 핸드오프. 정리본=docs/softn-id/working-notes.md. (참고: 학습 질문 1번을 안 박고 화면부터 확정 시 "무엇을 증명하는 프로토인지 모르는" 리스크 — 화면과 병렬로 박아둘 것 권고함.)
