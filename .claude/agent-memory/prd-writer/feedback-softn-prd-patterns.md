## 반복 갭 패턴 (1-pager에서 자주 빠지는 것들)

1. **퍼널 진입 동선 위치**: 신규 Labs 기능의 경우 기존 제품(퀴즈앤) 어디에서 진입하는지가 distribution 가설의 핵심인데 1-pager에서 미결정인 경우가 있음. PRD에서 blocking OQ로 눈에 띄게 처리.
2. **프로토타입 검증 규모·기간**: PoC 성격 프로젝트에서 "얼마나, 얼마 동안"이 미확정인 채로 내려오는 경우 있음. NFR 규모·비용 산정에 영향이므로 OQ로 명시.
3. **성공 지표 수치 근거**: 사용자가 "수치 감 모르겠다"고 솔직히 밝히는 경우, [잠정] 태그 + 해당 OQ 연결로 표기.

**Why:** Softn 팀은 검증 전 과확신을 경계하며, PRD에서 불확실성을 숨기지 않는 정직성을 선호.
**How to apply:** 위 3가지 갭은 PRD 작성 시 사전에 체크하고, 미확정이면 OQ 또는 TBD로 처리. 발명하지 말 것.

## 워딩·문맥 원칙 (사용자 직접 피드백 — 2026-06-10~11)

1. **요약·문두에는 핵심이 먼저.** "이 제품이 무엇인가"(예: 한국 교실용 실시간 협업 canvas 보드)를 문두에, 수단·메커니즘(Labs 30초 진입·무계정 QR)은 뒤에. 진입 방식으로 요약을 시작했다가 직접 지적받음: "요약에는 핵심이 우선적으로 들어가야 합니다."
2. **미입증 caveat는 한 번이면 충분.** prd-v3에서 사용자가 '빌드의 성격'에 반복된 desk research 미입증 문장을 직접 삭제함. 정직성은 유지하되 같은 경고를 문서 곳곳에 중복하지 말 것.
3. **§ 기호 금지** — "3절", "6절"처럼 풀어쓰기. "인수 기준"이라는 워딩도 어색하다고 교정받음 → **"완료 조건"** 사용(현 템플릿 반영됨).
4. **군더더기·현학 라벨 금지, 일상어** — 메인 세션의 doc-writing-style 피드백과 동일 결. 사용자가 산출물 워딩을 직접 미세 교정하는 일이 잦음(예: "30초 안에 켜고" → "30초 안에 수업 환경에 접속") — 자연스러운 한국어 제품 문장으로 쓸 것.

## 개념 모델 교훈 (prd-v1 사고)

prd-v1에서 F4는 "모둠당 캔버스 1개", F6은 "모둠별 썸네일들" — 같은 문서가 서로 다른 객체 모델을 전제하는 사고 발생. 이것이 현 템플릿 2절(핵심 개념 모델 및 용어) 신설의 직접 동기.
**How to apply:** 객체가 3개 이상이거나 계층·복제·상태가 있으면 2절을 반드시 먼저 확정하고, 모든 기능 정의가 같은 모델을 전제하는지 자가검증. 모호하면 발명하지 말고 사용자에게 모델을 직접 물을 것 — quizn-board의 "세션=모둠 동일 라인" 정정처럼 사용자가 머릿속에 명확한 모델을 갖고 있는 경우가 많음.

## 템플릿 운용 (2026-06-11 전면 개편)

- docs/template/prd-template.md가 0~9절+부록 구조로 교체됨(요약/사용자·권한/개념모델/기능/스토리·완료조건/가정/지표·기각/비목표/NFR/OQ/변경이력). Handoff 섹션 없음 — 핸드오프 정보는 채팅 메시지로.
- 핵심 규칙: 스냅샷 원칙(본문 이력·취소선 금지, 부록 한 줄), [잠정] 태그+OQ 연결, 측정 가능 완료 조건("매끄러운/직관적인/빠른" 금지), 비목표 = 이유+재검토 시점, OQ는 blocking/설계 2단(담당 열 없음), 모호 단위 본문 정의(예: "3액션" = 클릭 1회).
- 버전 관례: 내용 개정 = vN 새 파일, 워딩·가독성만 = vN.1 (one-pager·PRD 공통, 덮어쓰기 절대 금지).

## 정직성 원칙 (research 판정 반영)

- deep-research 판정이 미입증이면 해당 가정을 "입증된 전제"가 아니라 "PoC가 측정할 검증 대상"으로 명시 — 핵심 가정 표(위험도 포함)와 성공 지표가 그 표현 위치.
- 발산 기능 과대설계 금지: 입증되지 않은 수요를 전제로 기능을 과설계하지 않음.
- 미검증 아이디어를 '해결책'으로 단정하지 않음 — '제안/가설'로.
