## ★ 개념 모델 v3 (사용자가 직접 정의 — 절대 v1/v2 모델로 회귀 금지)

```
프로젝트 (소유자 생성 — 틀의 원본이자 권한·관리 단위)
 └── 캔버스 1~N개 (= 인스턴스/방. 같은 틀에서 수업별·모둠별 복수 찍어냄, QR/URL 진입 단위)
      └── 페이지 1~N장 (기본 1장, 좌우 추가·이동, 유한 — 무한 줌 아님)
           └── 객체 (텍스트·그리기·포스트잇·도형)
```

- v1·v2의 "세션→모둠→캔버스" 계층은 **폐기**. 사용자 정정: "세션과 모둠은 동일한 라인" — 둘 다 같은 틀에서 찍어낸 인스턴스(수업용이든 모둠용이든).
- 페이지 해석: "캔버스는 한 페이지로 구성" vs "페이지 좌우 추가" 사이 모호성을 메인 세션이 '캔버스 안에 페이지 여러 장(미리캔버스·알로식)'으로 해석, 사용자 이의 없음.
- **캔버스 정의 대전환**: 구조화 틀 전용(v2까지) → **일반 화이트보드 기본기 제공**(텍스트·그리기·포스트잇·도형). 템플릿 = 그 도구들을 조합한 틀일 뿐. 우선 3종 = **빈 캔버스·브레인스토밍·마인드맵** (만다라트·클러스터링 탈락).
- **권한 3단계**: 소유자/관리자/참여자. 소유자·관리자 = 로그인 필수, 임명은 프로젝트 단위(관리자→관리자 임명 가능, 임명 시 그 프로젝트 전 캔버스 접근). 비로그인 참여자 = 동일 닉네임/비밀번호 입력 시만 동일인(옵션은 소유자·관리자가 on/off). 로그인 참여자 = 참여했던 캔버스 언제든 재진입.
- 학생 대상 = **초·중·고**(초등 편입 → 만 14세 미만 개인정보 이슈 확정적).
- 용어: 문서에서 "보드" 단독 사용 금지(사내에서 wall 제품 지칭) — 프로젝트/캔버스/페이지로 통일.
