# Skill Usage Log — ux-designer

각 실행에서 어떤 UX 스킬을 무엇에·왜·결과가 뭐였는지 누적 기록. 날짜 역순.

## 2026-06-15 · canvas 박스 와이어프레임 v1

**ux-designer (foundation)** — 전 화면 토대.
- 무엇/왜: 학생 태블릿·폰 화면 설계의 모바일 규칙(thumb zone·44px·하단 도구바)과 협업 presence 패턴 근거로 `08-mobile-ux.md`, `12a-presence-awareness.md` 정독.
- 결과: 도구바를 하단 중앙(FigJam형)으로 확정 — 벤치마크와 thumb zone(Hoober) 둘 다가 같은 결론. presence는 아바타 스택(3~5 후 +N) + 원격 커서("▷ 이름")로 표준화. 잘 통함: 모바일 레퍼런스가 "왜 하단 도구바인가"를 한 줄로 정당화해 줘서 비용 낮음.

**design:ux-copy** — 안내(F)·권한·공감 용어 카피 점검.
- 무엇/왜: 초·중·고 학생 대상이라 문구가 쉬워야 함. 정원초과/잠금/보관 3종 안내, 입장 헬퍼, 권한 배지 문구를 점검.
- 결과: 대부분 이미 적절. 실제 바꾼 것 1건 — F-정원초과 CTA "다시 시도"(로봇틱) → "다시 들어와 보기"(본문 "다시 들어와 주세요"와 일치, 더 사람스러움). 교훈: 한국어 아동향 카피에선 명령형 동사보다 청유형("~해 보기")이 자연스럽다.

**design:accessibility-review** — 박스 레벨 가능 항목만(색 대비는 하이파이 이연).
- 무엇/왜: 학생 터치 화면의 타깃 크기·라벨·시간 제한을 WCAG 2.1 AA 관점으로 점검.
- 결과: 타깃 ≥44px·입력 라벨·헬퍼 모두 충족(의도적 사이징). 시간 제한(타이머→잠금)은 타이머 상시 노출+데이터 보존 안내+연장 가능으로 완화. 플래그 1건(노트로만): 비번 4글자 placeholder "○ ○ ○ ○"가 스크린리더에 pre-filled로 읽힐 수 있음 → 하이파이에서 빈 입력+힌트로 분리. 교훈: 박스 단계에선 a11y가 "타깃·라벨·시간제한" 세 가지에 집중되고, 그게 redraw 없이 노트 플래그로 충분.

**design:design-critique** — 화면별 위계·사용성·일관성.
- 무엇/왜: 14개 프레임 그린 뒤 구조 점검.
- 결과: 블로킹 없음(박스 v1 적절). 강점 = 포스트잇/타이머/presence/페이지내비를 교사 B와 학생 E에서 동일 패턴 재사용해 일관성 높음, 뷰어 변형이 보기 전용임이 명확(도구바 제거+안내 pill). 사용성 결정 1건 명시화 = 공유 패널 '참여자' 탭 기본 선택(2액션 절약) → 노트 6절에서 사용자 확인 항목으로 올림. 교훈: critique가 "고칠 것"보다 "결정으로 굳힌 것을 사용자에게 확인받게 표면화"하는 데 더 유용했다.

**figma-use / figma-create-new-file** — 드로잉.
- 교훈(재사용): ① auto-layout 자식 프레임을 고정 크기로 두려면 `primaryAxisSizingMode/counterAxisSizingMode = "FIXED"`를 명시해야 함(기본 HUG가 박스를 0폭으로 collapse — QR 박스·템플릿 미리보기에서 두 번 발생). ② 평범한 frame(layoutMode=NONE) 자식에는 `layoutPositioning="ABSOLUTE"` 못 씀 → 그냥 x/y로 배치. ③ `node.query('[name=한글]')`는 한글 속성값에서 selector 에러 → findAll로 우회. ④ 회색 박스는 0.85~0.86 명도가 화면에서 또렷, 0.91은 거의 안 보임.
