# prd-to-pdf

PRD 마크다운을 표지 포함 배포용 PDF로 변환한다. 변환 로직은 전부 번들 스크립트에 있으므로 **새 코드를 작성하지 말고 스크립트를 실행**한다.

## 사용법

```bash
python3 <skill-dir>/scripts/prd_to_pdf.py <입력.md>
```

- 출력 기본값: 입력과 같은 폴더에 같은 이름의 `.pdf`. 성공 시 stdout에 출력 경로가 찍힌다.
- 끝나면 사용자에게 출력 경로를 알려주고, 가능하면 PDF를 열어 1쪽(표지)과 본문 표 렌더링을 확인한다.

| 옵션 | 용도 |
|---|---|
| `-o 경로.pdf` | 출력 위치 지정 |
| `--no-cover` | 표지 없이 본문만 |
| `--title "제목"` | 표지 제목 덮어쓰기 (기본: 문서 첫 H1) |
| `--cover-template 경로` | 다른 표지 템플릿 사용 |
| `--style 경로` | 다른 CSS 사용 |

## 동작 방식 (스크립트가 자동 처리 — 직접 하지 말 것)

- **표지 메타 자동 추출**: 문서 첫 H1과 `> Project:` / `> Date: ... / Version: ...` 머리 블록에서 제목·프로젝트·버전·날짜를 읽어 표지에 채운다. 머리 블록이 없는 문서도 변환된다(제목=H1 또는 파일명, 날짜=오늘).
- **내부 주석 제거**: HTML 주석(`<!-- -->`, 템플릿 작성 지침 등)은 배포본에서 자동 삭제된다.
- **중복 제거**: 표지를 쓸 때 본문 첫 H1·머리 블록은 본문에서 빠진다(표지에 이미 있음).
- **의존성 부트스트랩**: `markdown`/`weasyprint`가 없으면 `~/.cache/prd-to-pdf/venv`를 만들어 자동 재실행한다. 첫 실행만 수십 초 걸릴 수 있다.

## 커스터마이징

사용자가 표지 문구·로고·색·폰트·여백 변경을 요청하면 코드가 아니라 에셋을 수정한다:

- `assets/cover.html` — 표지 레이아웃·문구. 플레이스홀더 `{{TITLE}}` `{{PROJECT}}` `{{DATE}}` `{{VERSION}}`
- `assets/style.css` — 색(`:root` 변수)·폰트·표·쪽번호·여백. 표지 스타일은 `.cover` 블록

일회성 변형이면 에셋을 복사해 수정한 뒤 `--cover-template`/`--style`로 지정하고, 영구 변경이면 에셋을 직접 수정한다.

## 트러블슈팅

- **weasyprint import 실패가 venv에서도 계속되면**: 시스템 라이브러리 문제 — `brew install weasyprint` 후 재시도.
- **한글이 깨지면**: macOS 기본 Apple SD Gothic Neo를 쓰므로 보통 문제없다. 리눅스라면 `fonts-noto-cjk` 설치 후 `style.css`의 font-family 확인.
- **표가 페이지를 넘칠 만큼 넓으면**: `style.css`의 `.content table { font-size }`를 줄이거나 해당 표의 긴 셀을 문서에서 다듬는다.
