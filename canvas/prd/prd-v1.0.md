## 0. 요약

**한국 교실용 실시간 협업 canvas 보드** — 학생들이 같은 캔버스 위에서 동시에 텍스트·그리기·포스트잇·도형으로 함께 만든다. 편집자(전형적으로 교사)는 하나의 프로젝트 안에 여러 캔버스를 만들고, 각 캔버스는 여러 페이지로 구성한다. 잘 만든 캔버스는 통째로 복제해 반별로 재사용하고, 캔버스끼리는 그룹으로 묶어 정리한다. 자기 화면에서 특정 캔버스를 크게 드릴다운해 보는 것이 곧 지휘다. 진입은 퀴즈앤 채널을 탄다 — 교사는 퀴즈앤에서 링크로 넘어와 30초 안에 수업 환경에 접속하고, 학생은 무계정 QR/URL로 들어온다.

> Why(배경·근거·차별화)는 one-pager 참조: `one-pager-v5.1.md`
> 캔버스 UI 레퍼런스(구속력 없음): `reference/` — FigJam·Miro·알로·미리캔버스 스크린샷

**포지셔닝 한 문장:** 퀴즈앤 기존 '보드'(wall — 각자 제출해 모아 붙이기)가 못 하는 것, 즉 **같은 캔버스 위에서 여러 명이 동시에 함께 조작하는 공동 구성**(canvas)을 제공한다.

**이 빌드의 성격:** [x] PoC/실험 — 목적은 기능 완성이 아니라 **"교사가 실시간 공동참여 canvas를 수업에 반복해서 쓰는가"(수요)의 검증**이며, 성패는 6절 지표로 판정한다.

> **제품 명칭:** "board-v2"는 가명(코드네임)이다. 정식 명칭은 미정 — 로고 영역에는 우선 'board-v2' 텍스트로 표기한다.

## 1. 대상 사용자 및 권한

교실 맥락의 기본 그림: 교사 = 소유자, 학생 = 참여자. 권한 구조 자체는 역할 일반형(소유자/편집자/참여자/뷰어)으로 설계한다.


| 역할  | 정의                                   | 계정 요건                                                                   | 핵심 권한                                                                                            |
| --- | ------------------------------------ | ----------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| 소유자 | 프로젝트 생성자 (전형적으로 교사)                  | **로그인 필수** (퀴즈앤 계정)                                                     | 프로젝트 전체 제어, 캔버스 생성·복제, 편집자 임명·해제, 닉네임/비밀번호·익명 옵션 설정, 권한별 QR/URL 발급, **모든 객체 컨트롤·잠금**, **편집자까지 내보내기(강제 퇴장)**, **캔버스 잠금/해제(잠금 중에는 본인도 캔버스 작업 불가, 잠금 해제는 가능)**             |
| 편집자 | 소유자 또는 다른 편집자가 임명, 또는 편집자 QR/URL로 초대 | **로그인 필수** — QR/URL로 초대되어도 입장하려면 반드시 로그인                                | 임명된 **캔버스**의 페이지 추가·템플릿 설정·캔버스 편집, 편집자 임명·해제, 닉네임/비밀번호·익명 옵션 설정, 권한별 QR/URL 발급, **모든 객체 컨트롤·잠금**, **참여자·뷰어 내보내기(강제 퇴장)**, **캔버스 잠금/해제(잠금 중에는 본인도 캔버스 작업 불가, 잠금 해제는 가능)** |
| 참여자 | 캔버스에 입장한 사람 (전형적으로 학생)               | 불필요 — 무계정 입장 가능(로그인해도 됨). **비로그인 입장 시 닉네임(최대 15자) + 비밀번호 4글자 필수**, 로그인 시 재진입 혜택 | 편집자가 설정해 둔 페이지에서 캔버스 작업(객체 생성·이동·편집), 포스트잇 감정표현(1인 1감정/포스트잇). **객체 컨트롤(수정·이동·삭제)은 본인이 작성한 객체만, 객체 잠금은 불가**. **캔버스 잠금 시 작업 동결(커서 숨김·객체 변경 멈춤), 보기·페이지 넘김은 가능**  |
| 뷰어  | 캔버스에 입장해 **보기만** 하는 사람               | 불필요 — 무계정 입장 가능(로그인해도 됨). **비로그인 입장 시 닉네임(최대 15자) + 비밀번호 4글자 필수**, 로그인 시 재진입 혜택 | 캔버스를 **보기 전용**. 객체 생성·이동·편집·컨트롤·잠금 불가, **페이지 추가·구성 불가(단 좌우 페이지 넘김은 자유 — 자유 열람 가능)**, **감정표현 불가**                    |


**참여자·뷰어 식별 규칙:**

- 로그인 사용자(참여자·뷰어 모두): 본인이 참여했던 캔버스에 언제든 재진입 가능. 뷰어로 참여했던 캔버스에는 보기 전용으로 재진입한다.
- 비로그인 사용자: **닉네임(최대 15자) + 비밀번호 4글자(고정)**를 입력해 입장한다. 동일한 닉네임/비밀번호를 다시 입력했을 때만 동일 사용자로 취급. 그렇지 않으면 진입 당시 브라우저 세션별로 분리되어 다른 사용자로 인식.
- 비밀번호는 **4글자 고정**, 닉네임은 **최대 15자**. 닉네임+비밀번호4는 비로그인 입장의 **필수 식별 정보**다(옵션 토글이 아님).
- **중복 닉네임:** 닉네임은 **같은 캔버스 안에서 유일**하다. 이미 같은 닉네임을 쓰는 사람이 있는 캔버스에 같은 닉네임으로 입장하려 하면 **입장(D) 화면에서 "이미 같은 닉네임을 쓰는 사람이 있어요" 식 안내를 띄워 차단**하고 다른 닉네임을 입력하게 한다(닉네임이 같으면서 비밀번호가 다른 동시 사용자를 막는다). **닉네임 중복은 D에서 미리 막으므로 입장 거절(F)의 사유가 되지 않는다.**
- **비밀번호 분실:** 비회원(참여자·뷰어)의 **비밀번호 찾기·재설정 장치는 두지 않는다**. 비로그인 사용자가 닉네임·비밀번호를 잊으면 그 식별로의 복구 수단은 없다(같은 닉네임/비밀번호4를 다시 입력해야만 동일 사용자로 복원되므로, 잊으면 새 식별로 입장하게 된다).
- **신원 연속성(미해결 — OQ-3):** 비로그인으로 닉네임 입장한 사람이 **이후 로그인을 해도 그 닉네임 세션과 로그인 계정이 동일인으로 이어지지 않는다 — 시스템은 '다른 사용자'로 인식**한다. 닉네임 세션과 로그인 계정을 잇는 장치는 아직 없다. 이 미해결은 **OQ-2(로그인 계정 체계)와 직결**이며, 풀리기 전까지 비로그인→로그인 전환 시 활동(작성 객체·G 내 활동 재진입)이 이어지지 않는다(9절 OQ-3).

**역할별 권한 경계:**

- 편집자: 페이지 추가 생성, 각 페이지에 템플릿 설정, 캔버스 편집이 가능하다. 닉네임/비밀번호·익명 옵션을 설정한다. **편집자는 로그인 필요 역할**이므로, 편집자 QR/URL로 초대받아 들어와도 로그인해야 편집자로 입장한다(F5).
- 참여자: 템플릿 수정·페이지 추가는 **불가**. 편집자가 설정해 놓은 페이지에서 캔버스 작업(객체 생성·이동·편집)과 포스트잇 감정표현(1인 1감정/포스트잇)이 가능하다. 비로그인 입장 시 닉네임(최대 15자)+비밀번호4로 식별.
- 뷰어: **보기 전용**. 객체 생성·이동·편집 불가, **페이지 추가·구성 불가**, **감정표현 불가**. **단 좌우 페이지 넘김(자유 열람)은 가능하다** — 편집자가 짜둔 여러 페이지를 스스로 앞뒤로 넘겨 볼 수 있다(페이지를 추가하거나 구성을 바꾸지는 못한다). 참여자와의 차이는 — 참여자는 객체를 만들고 옮기고 편집하며 감정표현도 하지만, 뷰어는 캔버스에 들어와 **보기만** 한다. **뷰어는 포스트잇 감정표현도 할 수 없다.** 비로그인 입장 시 닉네임(최대 15자)+비밀번호4로 식별.

**객체 컨트롤·잠금 권한 규칙:**

- **소유자·편집자**: 누가 만든 객체든 **모든 객체를 컨트롤**(수정·이동·삭제)할 수 있고, **모든 객체를 잠글 수 있다**(F13 객체 잠금). (단 캔버스가 잠긴 동안은 소유자·편집자도 캔버스 작업이 멈춘다 — 아래 캔버스 잠금 규칙.)
- **참여자**: **본인이 작성한 객체만** 컨트롤(수정·이동·삭제)할 수 있다. 다른 사람이 만든 객체는 보기·감정표현만 되고 옮기거나 지울 수 없다. **객체 잠금은 본인 객체라도 할 수 없다 — 잠금은 소유자·편집자 전용**이다.
- **뷰어**: 객체 컨트롤·잠금 불가(보기 전용 — 위 권한 경계와 동일).
- 소유자·편집자는 각 객체의 **작성자·수정자가 누구인지 객체 단위로 식별**할 수 있어야 한다(누가 만들고 누가 고쳤는지). 표기·동작 방식은 아래 작성자 가시성 규칙과 F13(객체 클릭 시 작성자 표시·객체 잠금)로 구현한다.

**캔버스 잠금(F8) 시 작업 권한 규칙(확정):**

- **캔버스 잠금은 '전원 작업 동결'이다.** 캔버스를 잠그면 **소유자·편집자·참여자 모두** 캔버스 작업이 멈춘다 — 객체 생성·수정·이동·삭제가 모두 불가하고, **참여자 커서는 숨겨진다**. 누구도(운영자 포함) 잠긴 캔버스 위에서 객체를 손대지 못한다.
- **단 잠금 해제(토글)는 소유자·편집자가 할 수 있다.** 잠금 해제는 '캔버스 작업'이 아니라 운영 컨트롤이므로, 소유자·편집자는 잠긴 상태에서도 상단바 잠금 버튼으로 잠금을 풀 수 있다(B 상단바 토글 — F7·F8). 잠금을 풀면 즉시 평소처럼 편집 가능 상태로 돌아간다.
- 이 규칙의 맥락 = 잠금은 "지금은 아무도 손대지 말자"고 캔버스 전체를 얼리는 장치다. 운영자가 잠근 동안 들여다보고 정리하고 싶으면 잠금을 풀고 작업한 뒤 다시 잠그면 된다.
- **입장·열람은 모두 그대로다** — 잠금은 사람의 입장을 막지 않는다(F8·F5). 잠긴 캔버스에 들어와도 안내 화면(F)이 아니라 보기·페이지 넘김이 가능한 캔버스 화면으로 들어간다.

**작성자 가시성 규칙:**

- 객체(포스트잇 등)의 작성자는 **객체를 클릭했을 때** 드러난다. 객체를 클릭하면 **객체 수정 옵션 바가 객체 위(상단)에 노출**되고, 그 **옵션 바의 제일 우측 끝에 작성자 아바타**가 표시된다. **아바타를 클릭하면 작성자의 풀네임**이 드러난다. (상시 표시·호버가 아니라 '클릭 시 수정 옵션 바 + 우측 아바타, 아바타 클릭 시 풀네임'으로 확정.)
- 소유자·편집자는 익명 옵션과 무관하게 이 방식으로 작성자를 **항상** 확인할 수 있다.
- **익명 옵션**을 켜면 참여자 간에는 작성자 비공개 — 그래도 소유자·편집자에게는(옵션 바 우측 아바타·풀네임으로) 보인다. 옵션은 소유자·편집자가 조정.

**내보내기(강제 퇴장) 권한 규칙 (F13):**

- **내보내기 = 소유자·편집자가 특정 접속자를 그 캔버스에서 강제로 내보내는(퇴장시키는) 동작**이다. 접속자 모달의 각 사람 행에서 실행한다(F13).
- **권한 경계:** **편집자는 참여자·뷰어까지** 내보낼 수 있고, **소유자는 편집자까지** 내보낼 수 있다(소유자가 가장 넓다). **소유자 자신과 '본인'은 내보내기 대상이 아니다** — 편집자는 다른 편집자·소유자를 내보낼 수 없고, 누구도 자기 자신을 내보낼 수 없다.
- **재입장:** 내보낸 사람은 다시 들어오려면 **승인이 필요**하다(자동 승인 옵션이 켜져 있더라도 내보낸 사람의 재입장은 승인을 거친다 — 내보내기의 의미가 자동 입장으로 무력화되지 않도록). 재입장 승인은 입장 승인과 같은 표면에서 처리한다(F5·F13).

**편집자 임명·초대 규칙:** 임명·초대 단위 = **캔버스**. 편집자가 되는 경로는 둘이다 — ① 소유자·편집자가 **접속자(participant) 패널에서 로그인 사용자를 편집자로 지정**(F6), ② **편집자 권한으로 발급한 QR/URL로 초대**(공유 패널에서 '편집자'를 선택해 발급한 링크). 어느 경로든 편집자로 입장하려면 **반드시 로그인**한다(편집자 = 로그인 사용자만). 임명·초대되면 그 캔버스의 페이지 추가·템플릿 설정·편집이 가능하다.

- **편집자 QR 초대 링크는 발급 후 10분에 만료**된다(만료 후 그 링크로는 편집자 입장 불가, 재발급 필요).
- **편집자 임명 해제 기능이 있다** — 소유자·편집자는 접속자 패널에서 임명한 편집자를 다시 해제할 수 있다.

**공유·입장 권한 규칙:**

- 캔버스 공유 시(QR/URL 발급, F2·F9) **초대할 권한을 먼저 선택**한다 — 편집자 / 참여자 / 뷰어. 선택한 권한에 해당하는 **별개의 QR·URL**이 발급·표시된다(권한별로 링크가 다르다). 공유 패널은 **권한별 QR 발급만** 담당한다(편집자 임명·해제는 접속자 패널 — F6).
- **편집자 링크**로 들어오면 로그인 게이트를 거쳐 편집자로 입장한다(로그인 필수). 편집자 초대 링크는 발급 후 **10분에 만료**된다.
- **참여자 링크 / 뷰어 링크**로 들어오면 로그인 없이 입장 가능하며, 비로그인 시 닉네임(최대 15자)+비밀번호4로 식별한다.
- **뷰어도 캔버스 정원(20명)에 포함**한다 — 정원은 편집자·참여자·뷰어 모두를 센다. 정원 초과 시 뷰어도 입장 불가 안내를 받는다(F5·F 안내).
- **참여자 입장 승인(F13):** 참여자가 캔버스에 접근할 때 **기본은 수동 승인**이며, 소유자·편집자가 **입장 자동 승인 옵션(기본 OFF)**을 켜면 새 참여자가 승인 없이 입장한다. 승인은 대기자별 승인/거절 + **'모두 승인'** 일괄 처리로 한다. **편집자도 입장 승인 대상이 될 수 있다.** **단 내보낸(강제 퇴장한) 사람의 재입장은 자동 승인이 켜져 있어도 승인을 거친다**(위 내보내기 권한 규칙).

**학생 대상:** **초·중·고** 학생, 태블릿·크롬북·스마트폰.

**범위 밖 사용자:** 대학·기업, 퀴즈앤 미사용 신규 교사(PoC 기간은 기존 퀴즈앤 채널로 한정).

## 2. 핵심 개념 모델 및 용어

```
프로젝트 (최상위 컨테이너 — 소유자가 생성. 권한·관리의 큰 틀)
 └── 캔버스 (핵심 단위 — 프로젝트 바로 아래. 권한별 QR/URL 발급, 학생 진입 단위, 진입자 최대 20명(뷰어 포함), 최대 30페이지)
      └── 페이지 1~30장 (캔버스의 작업 면 — 편집자가 템플릿을 붙여 구성, 좌우로 추가·이동, 유한. 뷰어도 좌우로 넘겨 자유 열람)
           └── 객체 (텍스트·그리기·포스트잇·도형 등 기본 도구로 만드는 것 — 클릭 시 수정 옵션 바·작성자 아바타·잠금)

  그룹 (선택 — 프로젝트 안에서 캔버스들을 묶는 폴더/라벨. 캔버스와 같은 레벨이며 상위 계층이 아니다. 안 묶어도 됨)
```

- **프로젝트**: 최상위 컨테이너. 소유자가 생성하며, 그 안에서 캔버스들을 만든다. 권한·관리가 걸리는 큰 틀이다. **PoC에선 사실상 단일 프로젝트로 운영**한다 — 다중 프로젝트 생성·전환·관리 UI는 범위 밖이며, 여기서는 개념적 분리만 둔다.
- **캔버스(핵심 단위)**: 프로젝트 바로 아래의 핵심 단위다. 여러 페이지로 구성된 하나의 작업 공간이자 학생이 들어오는 진입 단위. 캔버스는 프로젝트 안에 바로 존재하며 **그룹에 묶이는 것은 선택**이다(그룹 없이도 존재 가능). **새로 만든 캔버스의 기본 이름 = "새 캔버스"**이며 사용자가 바꾼다. 캔버스 이름은 **B 상단바에서 더블클릭으로 그 자리에서 수정**한다(인라인 편집 — F1). 공유 시 **초대 권한(편집자/참여자/뷰어)을 선택해 권한별 QR·URL을 발급**한다. **진입자 최대 20명(편집자·참여자·뷰어 모두 정원에 포함)**이다. 편집자 임명·닉네임/비밀번호·익명 옵션이 이 단위에 묶인다. **상태 = 진행 중 / 잠금**(공유 여부는 별도 구분). 잠금은 '전원 작업 동결'이며 입장을 막지 않는다(F8). **'정원 임박'은 상태·배지로 두지 않는다** — 정원이 차면 신규 입장만 막힐 뿐(F5·F 안내), 근접 경고 상태 표시는 없다. **PoC에는 '보관(archive)' 상태가 없다** — 캔버스는 사용자가 명시적으로 삭제(확인 모달 경유 — F1)하기 전까지 존속하며, 보관해 따로 갈무리하는 개념·동선은 PoC 범위 밖이다.
- **그룹(선택 — 캔버스를 묶는 단위, 상위 계층 아님)**: 프로젝트 안에서 **캔버스들을 묶는 선택적 폴더/라벨**이다(예: "1차시", "모둠 발표", "2026 1학기"). 캔버스 목록을 정리하는 수단일 뿐이며, **캔버스의 상위 계층이 아니라 캔버스와 같은 레벨**이다 — 캔버스는 그룹에 들어가 있을 수도, 어느 그룹에도 묶이지 않고 프로젝트 바로 아래 놓일 수도 있다. 그룹 카드 안에 속한 캔버스 썸네일들이 보이고 그룹명 라벨이 붙는다. **그룹 이름은 A 화면의 그룹 헤더에서 더블클릭으로 그 자리에서 수정**한다(인라인 편집 — F11).
- **페이지**: 캔버스의 작업 면. 편집자가 템플릿을 붙여 구성한다(예: 템플릿을 이것저것 붙여 10페이지짜리 캔버스를 완성). 좌우로 페이지를 추가하고 이동할 수 있다(미리캔버스·알로 참고). 무한 줌 월드가 아니라 **유한한 페이지**다. **캔버스당 페이지는 최대 30개(고정)** — 그 이상은 추가할 수 없다. **페이지의 추가·구성은 편집자 이상만**, 단 **뷰어를 포함한 모든 입장자는 좌우로 페이지를 넘겨 자유 열람**할 수 있다(넘겨 보기는 누구나, 추가·구성은 편집자 이상).
- **템플릿**: 페이지에 깔리는 객체 묶음(캔버스 기본 도구들을 조합해 만든 틀이며 별도 엔진이 아니다). 편집자가 페이지별로 설정한다. **기본값 = 빈 캔버스**. 선택지 = **빈 캔버스 · 브레인스토밍 · 마인드맵**.
- **객체**: 페이지 위에 만들고 옮기는 것 — 텍스트, 그리기(펜), 포스트잇, 도형 등 일반 캔버스 도구가 만드는 요소(FigJam·Miro·알로·미리캔버스 참고). 모든 객체에는 작성자(및 수정자) 정보가 남는다(1절 가시성·객체 컨트롤 권한 규칙). 객체를 클릭하면 **객체 상단에 수정 옵션 바**가 뜨고(제일 우측에 작성자 아바타, 클릭 시 풀네임), 그 바에서 **객체 잠금**(소유자·편집자 전용)도 할 수 있다(F13). **포스트잇 객체에는 감정표현(다종 이모지)을 남길 수 있다(다른 객체엔 없음).**
- **복제**: 잘 만든 **캔버스** 하나를 통째로 복제해 재사용하는 것(예: 1반·2반·3반용으로 복제). 복제본은 같은 프로젝트 안의 독립된 캔버스이고 각자 다른 QR을 가진다. 복제본끼리 내용은 동기화되지 않는다.

**용어 금지/통일:** 이 문서에서 "보드" 단독 사용 금지 — 사내에서 '보드'는 퀴즈앤 기존 wall 제품을 가리킨다. **프로젝트 / 그룹 / 캔버스 / 페이지 / 객체**로 통일. 단 위계상 **캔버스가 핵심 단위**이고 **그룹은 캔버스를 묶는 선택 수단**임을 혼동하지 말 것(그룹은 캔버스의 상위 통이 아니다).

**잠금 용어 구분(혼동 금지):** 이 문서에는 '잠금'이 두 개 있다 — ① **캔버스 잠금(F8)** = **전원**의 작업을 동결(소유자·편집자·참여자 모두 객체 생성·수정·이동·삭제 멈춤 + 참여자 커서 숨김). **입장·열람은 모두에게 계속 허용**하고, **잠금 해제만 소유자·편집자가 할 수 있다**(운영 컨트롤). ② **객체 잠금(F12·F13)** = 객체 하나만 잠가 다른 사람이 못 건드리게 함(편집자 이상 전용). 둘은 다른 동작이다.

**학생 대면 카피 — '관리자'로 통일:** 학생(참여자·뷰어)이 보는 화면 문구에서는 소유자·편집자를 **'관리자'**로 부른다('선생님'이 아니라). 캔버스를 운영하는 주체가 꼭 교사가 아닐 수 있고(편집자는 다른 학생일 수도 있다), 역할이 일반형이기 때문이다. 예: 입장 승인 대기 안내, 입장 거절 안내, 잠금 안내 배너 등에서 "관리자 승인을 기다리는 중", "관리자가 입장 요청을 받지 않았어요", "관리자가 캔버스를 잠갔어요" 식으로 쓴다. (교사 area 내부 문구는 영향 없음 — 학생 대면 문구만.)

### 기능 상세

**F1. 캔버스 생성·관리** *(소유자·편집자)*

- **필수 정보**: 프로젝트 안 캔버스 목록(이름·생성일·페이지 수, 그룹 소속 여부, 상태 배지), 캔버스별 설정(편집자 목록, 닉네임/비밀번호·익명 옵션), 캔버스별 **권한별 진입 QR·URL**(편집자/참여자/뷰어)
- **목록 탭 체계(확정)**: 캔버스 목록 상단 탭 = **전체 / 진행 중 / 공유된 캔버스**.
  - **전체** = 모든 상태(진행 중·잠금)의 캔버스 + 그룹 폴더를 함께 표시.
  - **진행 중** = active 상태 캔버스.
  - **공유된 캔버스** = 내가 **'편집자'로 들어가 있는(소유자가 아닌)** 캔버스 — 다른 사람이 소유한 캔버스에 편집자로 합류한 것을 모아 본다.
  - **잠금은 별도 탭·필터가 아니라 카드의 상태 배지로만** 표시한다.
  - **'보관함' 탭은 두지 않는다(PoC 범위 밖)** — 보관 개념 자체가 없으므로 보관 상태 캔버스를 모아 보는 탭도 없다.
- **잠금 상태 노티(A 캔버스 목록 — 확정)**: 잠긴 캔버스 카드에는 **`🔒 잠금` 상태 배지**가 붙고, 카드에 **"작업이 불가능해요"** 안내 텍스트를 함께 보여준다(이 캔버스가 작업 동결 상태임을 목록에서 바로 알린다).
- **'정원 임박' 표시 없음(확정)**: 카드 배지·캔버스 상태에 **정원 근접(임박) 경고는 두지 않는다**. 정원이 차면 신규 입장만 막힐 뿐(F5·F 안내), 목록 카드에 정원 임박 상태를 표시하지 않는다.
- **필수 행동**:
  - **캔버스 생성**: 두 경로 모두 같은 생성 흐름을 탄다 — ① 퀴즈앤 링크로 처음 진입할 때 템플릿 선택 모달이 자동으로 뜨는 경로(F9), ② 캔버스 목록 화면에서 명시적인 **'캔버스 생성'** 액션을 눌러 시작하는 경로. 어느 쪽이든 **템플릿 선택 화면에서 빈 캔버스(기본값) 또는 템플릿 3종 중 선택**(F3)해 캔버스를 만든다. **새 캔버스의 기본 이름은 "새 캔버스"**로 붙고, 사용자가 곧바로 바꿀 수 있다.
  - **캔버스 이름 수정(인라인 편집)**: 캔버스 이름은 **B 상단바에서 이름을 더블클릭하면 그 자리에서 바로 고친다**(별도 설정 화면·모달 없이). 입력을 마치면 즉시 반영된다.
  - 캔버스 이름 지정, 페이지 구성(F4, 최대 30페이지), 편집자 임명·초대·해제(F6), 옵션 설정, 공유 패널에서 **권한(편집자/참여자/뷰어) 선택 후 QR/URL 발급·재표시**(F6)
  - **카드 더보기(⋮) 메뉴**: 내 소유 캔버스 = **복사하기 · 잠금**(잠긴 카드는 '잠금 해제') **· 삭제**(보관·보관 해제 없음). 공유된(내가 편집자로 들어간) 캔버스 = **복사하기 · 잠금**(잠긴 카드는 '잠금 해제') **· 나가기**. — 잠금/잠금 해제는 목록에서 캔버스에 들어가지 않고도 잠금 상태를 토글하는 진입점이다(잠금 자체는 F8).
  - **캔버스 삭제 = 재확인 단계(확정)**: 삭제는 곧장 지워지지 않고 **'완전 삭제·되돌릴 수 없음'을 알리는 확인 모달**을 거친다(오삭제 방지). 모달에서 명시적으로 확인해야 삭제된다.
- **품질 기준**: 진입 후 첫 캔버스 QR 발급까지 3액션·30초 이내(실측 동선은 F9 참조)
- **하지 않는 것**: 캔버스 간 내용 동기화(복제본은 서로 독립 — F2) · 다중 프로젝트 생성·전환·관리 UI(2절 — PoC 범위 밖) · 확인 없는 즉시 삭제 · **캔버스 보관(archive)·보관함(PoC 범위 밖)** · **'정원 임박' 상태·배지 표시**

**F2. 캔버스 복제** *(소유자·편집자)*

- **필수 정보**: 복제 대상 캔버스, 복제본 목록(각자 권한별 QR·URL)
- **필수 행동**: 잘 만든 캔버스 하나를 통째로 복제(예: 1반·2반·3반용) · 복제본은 같은 프로젝트 안에서 각각 독립된 캔버스로 생성되고 각자 다른 QR을 가짐 · 복제본도 공유 시 권한(편집자/참여자/뷰어)을 선택해 권한별 QR/URL을 발급(F6)
- **플로우/상태**: 복제 즉시 진행 중 + (권한 선택 시) QR 표시 · 복제 횟수·프로젝트당 캔버스 수 상한은 미정(OQ-5) — 정해지기 전에는 임의 제한을 두지 않음
- **하지 않는 것**: 복제본 간 내용 동기화(복제본은 서로 독립) · 프로젝트 통째 복제(별도 컨테이너 단위 복제 수요는 OQ-5에서 함께 판단 — 임의로 만들지 않음)

**F3. 템플릿 3종** *(편집자가 페이지별 설정)*

- **필수 정보**: 템플릿 선택지 3종과 미리보기 — **빈 캔버스(기본값) / 브레인스토밍 / 마인드맵**. 새 캔버스·새 페이지 모달의 **기본 선택값 = 빈 캔버스**. 새 캔버스 생성 시 **기본 이름은 "새 캔버스"**(F1)
- **필수 행동**: 캔버스 생성 시(F1) 그리고 편집자가 페이지를 추가할 때 템플릿을 설정하면 해당 객체 묶음이 그 페이지에 깔린다. 템플릿 = 캔버스 기본 도구들을 조합해 만든 틀(별도 엔진 아님)
- **하지 않는 것**: 사용자 커스텀 템플릿 제작·저장(후속). ⚠️ 브레인스토밍 템플릿은 '전원 동시 발산'이 아니라 공동 분류·구조화 단계에 초점(동시 자유발산의 열위 근거 — `working-notes.md`)

**F4. 캔버스 공동 편집** *(참여자·소유자·편집자 / 뷰어는 보기 전용)*

- **필수 정보**: 같은 캔버스 사용자들의 편집이 실시간 반영된 페이지, 페이지 위치(N/M, M ≤ 30), 잔여 시간(F8), 객체 작성자 정보(1절 가시성 규칙에 따라 권한별 노출 — 객체 클릭 시 옵션 바 우측 아바타, 클릭 시 풀네임)
- **필수 행동**:
  - 참여자: 편집자가 설정해 둔 페이지에서 기본 도구로 객체 생성·이동·편집 — **텍스트 입력, 그리기(펜), 포스트잇, 도형** 등 일반 캔버스 기본 기능(FigJam·Miro·알로·미리캔버스 수준 참고). **단 수정·이동·삭제는 본인이 작성한 객체만 가능(F12), 객체 잠금은 불가**. 페이지 추가·템플릿 수정은 불가.
  - 편집자: 위에 더해 페이지 좌우 추가·이동(최대 30페이지까지), 각 페이지에 템플릿 설정. **누가 만든 객체든 모두 컨트롤·잠금 가능**(F12·F13). 캔버스 이름은 상단바에서 더블클릭으로 인라인 수정(F1). **단 캔버스 잠금 상태에서는 소유자·편집자도 캔버스 작업이 멈춘다(전원 동결 — F8). 잠금 해제는 상단바 잠금 버튼으로 할 수 있다.**
  - 뷰어: **보기 전용** — 같은 캔버스의 편집이 실시간 반영되는 화면을 보기만 한다. 객체 생성·이동·편집 불가, **페이지 추가·구성 불가**, **감정표현 불가**. **단 좌우 페이지 넘김(자유 열람)은 가능** — 편집자가 짜둔 여러 페이지를 스스로 앞뒤로 넘겨 본다(페이지를 추가·구성하지는 못한다).
- **플로우/상태**: 진행 중(편집 가능) / 잠금. **잠금 = 전원 작업 동결**(소유자·편집자·참여자 모두 객체 변경 멈춤, 참여자 커서 숨김, 단 보기·페이지 넘김 열람은 계속 가능). **잠금 해제는 소유자·편집자가 상단바 잠금 버튼으로 한다(운영 컨트롤).** 입장은 어느 상태에서도 막지 않는다. 뷰어는 진행 중·잠금 어느 상태에서도 보기만 한다.
- **품질 기준**: 한 사용자의 편집이 같은 캔버스 전원(뷰어 포함) 화면에 **2초 이내** 반영 · 동시 조작에서 객체 유실·중복 0건
- **하지 않는 것**: 다른 캔버스 열람·편집(참여자·뷰어 기준 — 소유자·편집자는 F7로 접근)

**F5. 입장·사용자 식별** *(편집자·참여자·뷰어)*

- **필수 정보**: 입장 화면 — 캔버스 이름·들어온 QR의 권한 배지. 권한별로 분기:
  - 편집자 링크: 로그인 화면(또는 로그인 유도). 만료(10분 초과)된 링크면 만료 안내.
  - 참여자·뷰어 링크: 닉네임(최대 15자) 입력란 + 비밀번호 4글자 입력란(비로그인 입장 시 필수)
- **필수 행동**: QR 스캔 또는 URL 접속 → **링크에 담긴 권한에 따라 분기**:
  - **편집자 QR/URL**: 로그인 → 편집자로 입장(로그인 필수 — 비로그인은 편집자로 입장 불가). 링크는 발급 후 **10분에 만료**, 만료 시 입장 불가 안내(재발급 필요).
  - **참여자·뷰어 QR/URL**: 로그인 없이 입장 가능. 비로그인 시 **닉네임(최대 15자) + 비밀번호 4글자 입력** → 해당 권한으로 입장. 로그인 사용자(참여자·뷰어 모두)는 닉네임/비밀번호 없이 입장하고 참여했던 캔버스에 언제든 재진입한다(뷰어로 참여했던 캔버스는 보기 전용으로 재진입). **QR이 권한을 결정하므로 입장자는 권한을 못 바꾼다(D에 권한 선택 토글 없음).**
  - **중복 닉네임 차단**: 같은 캔버스 안에 같은 닉네임이 이미 있으면 **입장(D) 화면에서 "이미 같은 닉네임을 쓰는 사람이 있어요" 식 안내를 띄우고 입장을 막아** 다른 닉네임을 입력하게 한다(닉네임은 캔버스 내 유일). **이 차단은 D에서 미리 일어나므로 입장 거절(F)의 사유가 아니다.**
  - **참여자 입장 승인(F13)**: 참여자가 입장하려 하면 기본은 **수동 승인 대기** — 소유자·편집자가 승인해야 입장. **입장 자동 승인 옵션(기본 OFF)**을 켜면 바로 입장하고, '모두 승인' 버튼으로 대기자를 한 번에 받는다(대기자별 승인/거절도 가능). 편집자도 입장 승인 대상이 될 수 있다. **내보낸(강제 퇴장당한) 사람이 다시 들어오려 하면 자동 승인이 켜져 있어도 승인을 거친다(F13 내보내기).**
  - **신원 연속성 한계(OQ-3)**: 비로그인 닉네임으로 입장한 사람이 이후 로그인해도 같은 사람으로 이어지지 않고 다른 사용자로 인식된다(미해결 — 9절 OQ-3·OQ-2 직결).
- **입장 후 잠금 캔버스 화면(E) — 잠금 노티(확정)**: 잠긴 캔버스에 들어온 입장자는 안내 화면(F)이 아니라 보기·열람이 되는 캔버스 화면(E)으로 들어가되, 현재 잠금 상태임을 다음과 같이 알린다(학생 대면 문구는 '관리자' 표기):
  - **E-참여자**: 화면 **상단 배너 — "관리자가 캔버스를 잠갔어요 · 지금은 보기만 할 수 있어요"** + **하단 도구 팔레트 비활성**(작업 동결이므로 도구를 쓸 수 없음을 시각적으로 함께 알림). 잠금 해제되면 배너가 사라지고 팔레트가 다시 활성화된다.
  - **E-뷰어**: 화면 **상단 배너 — "관리자가 캔버스를 잠갔어요 · 지금은 아무도 편집할 수 없어요"**. (뷰어는 원래 도구가 없으므로 팔레트 비활성 표시는 없다.)
- **입장 안내 화면(F 안내) — 변형 2종(확정):** 입장이 막혔을 때 단일 메시지 카드 한 화면의 변형으로 보여준다. 카드에는 어느 캔버스에서 막혔는지 요약(캔버스명·상태·역할)이 붙는다.

  | 변형 | 언제 | 메시지 골자 | 행동 버튼 |
  |---|---|---|---|
  | **정원 초과** | 진입자 20명(뷰어 포함) 꽉 참 | "지금은 정원이 모두 차 있어요 · 잠시 뒤 다시 시도" | [다시 시도] · [돌아가기]\* |
  | **입장 거절** | D-승인 대기에서 관리자가 수동 거절 | "관리자가 입장 요청을 받지 않았어요 · 이름 확인하고 다시 요청" | [다시 요청하기] · [돌아가기]\* |

  - **\* '돌아가기'는 로그인 사용자에게만** 제공한다 — 비로그인 입장자는 돌아갈 홈이 없으므로 '돌아가기'를 노출하지 않는다.
  - **'잠금' 변형은 두지 않는다(확정).** 캔버스 잠금(F8)은 **전원 작업을 동결할 뿐 입장을 막지 않으므로**, 잠긴 캔버스에 들어와도 안내 화면이 아니라 보기·열람이 가능한 E 화면으로 들어가고, 그 화면 안에서 위 잠금 배너로 상태를 알린다.
  - **'보관됨' 변형은 두지 않는다(PoC 범위 밖).** 보관 개념 자체가 PoC에 없으므로 보관된 캔버스 입장 시도라는 상황이 발생하지 않는다.
  - **'닉네임 중복'은 입장 거절 사유가 아니다** — 닉네임 중복은 D(입장) 화면에서 미리 차단되므로 F의 '입장 거절'에 해당하지 않는다. 입장 거절 = 관리자의 수동 거절뿐.
- **플로우/상태**: 비로그인 + 동일 닉네임/비밀번호4 = 동일 사용자로 복원 / 그 외 = 브라우저 세션별 별도 사용자 · **정원(20명, 뷰어 포함) 초과 시 입장 불가 안내**(편집자·참여자·뷰어 모두 정원에 셈) · 입장 거절·내보내기 후 재입장 거절 시 안내 화면(F) · **잠금 캔버스는 안내 화면 없이 입장·열람 가능(E 화면에서 잠금 배너로 안내)**
- **비밀번호 분실**: 비회원(참여자·뷰어) **비밀번호 찾기·재설정 장치는 없다**. 분실 시 동일 식별로의 복구 수단은 제공하지 않는다.
- **품질 기준**: 스캔→진입 로딩 3초 이내(4G+) · 참여자·뷰어는 무설치·무가입(로그인은 선택) · 편집자는 로그인 필수
- **하지 않는 것**: 참여자·뷰어에게 가입·로그인 강제, 닉네임(최대 15자)·비밀번호4 외 개인정보 수집 · 잠금 캔버스에 대한 입장 차단 안내(입장은 허용, E에서 배너로만 안내) · 보관 캔버스에 대한 입장 안내(보관 개념 없음)

**F6. 권한 관리** *(소유자·편집자)*

- **개념**: 권한 관리는 **두 패널**로 나뉜다 — ① **공유 패널** = 권한별 QR/URL 발급만, ② **접속자(participant) 패널** = 현재 접속자 목록에서 역할 지정(편집자 임명·해제)과 **내보내기(강제 퇴장)**. 캔버스 설정은 별도 화면이 아니라 **설정 묶음**으로 둔다(아래).
- **필수 정보**: 캔버스별 편집자 목록(접속자 패널), 닉네임/비밀번호·익명 옵션 상태, **공유 패널의 권한 선택 상태와 권한별(편집자/참여자/뷰어) QR·URL**, 참여자 입장 승인 모드(수동/자동) 상태
- **필수 행동**:
  - **공유·권한별 QR/URL 발급(공유 패널)**: 캔버스 공유 시 **초대할 권한을 먼저 선택(편집자/참여자/뷰어)**하면 그 권한에 해당하는 QR·URL이 발급·표시된다(권한별 별개 링크). **편집자 초대 링크는 발급 후 10분에 만료**된다. 공유 패널은 QR 발급만 담당한다(편집자 임명은 접속자 패널).
  - **편집자 임명·해제(접속자 패널)**: 소유자·편집자가 **접속자 패널에서 로그인 사용자를 그 캔버스의 편집자로 지정**하거나, 지정한 편집자를 다시 **해제**한다. 편집자로 입장하려면 반드시 로그인(편집자 = 로그인 사용자만, F5). 또는 위 **'편집자' 권한 QR/URL 초대**로도 편집자가 된다. 임명·초대된 편집자는 해당 캔버스의 페이지 추가·템플릿 설정·편집·다른 편집자 임명 가능.
  - **내보내기(강제 퇴장)(접속자 패널)**: 소유자·편집자가 접속자 패널의 각 사람 행에서 그 사람을 캔버스에서 **강제 퇴장**시킨다. **편집자는 참여자·뷰어까지, 소유자는 편집자까지** 내보낼 수 있다(소유자 자신·본인 제외). 내보낸 사람은 **재입장 시 승인 필요**(자동 승인이 켜져 있어도 승인을 거침). 동작·권한 경계 상세는 F13·1절 내보내기 권한 규칙.
  - **캔버스 설정 묶음**: 별도 설정 화면 없이 다음을 한 묶음으로 둔다 — **익명 옵션 on/off**(켜면 참여자 간 작성자 비공개, 소유자·편집자는 항상 확인) / **참여자 커서 숨기기**(F13) / **입장 자동 승인 옵션(기본 OFF)**(F13) / **닉네임+비밀번호4**(비로그인 입장 식별 — **끌 수 없는 고정**, 토글 아님) / **캔버스 삭제(확인 모달 경유 — F1)**.
- **뷰어 관리**: 뷰어(보기 전용) 입장은 **'뷰어' 권한으로 발급한 QR/URL**로 들어오는 경로로 확정. 뷰어도 정원(20명)에 포함된다.
- **하지 않는 것**: 페이지 단위 개별 권한, 읽기 전용 공유 링크의 별도 체계화(뷰어 링크가 사실상 보기 전용 입장 경로를 담당 — 추가 공유 링크 유형 수요는 후속) · 소유자 권한 이전(잔여 — OQ-6)

**F7. 캔버스 드릴다운 (지휘)** *(소유자·편집자)*

- **개념**: 별도의 전체 공유 화면이나 broadcast는 없다. 교사는 캔버스 목록에서 보고 싶은 캔버스를 **자기 화면에 크게 열어(드릴다운)** 들여다보는 것으로 지휘한다. 캔버스 목록 화면이 곧 지휘의 출발점이다.
- **드릴다운 = 편집 화면(B) 진입 (확정)**: 교사가 막힌 모둠을 '들여다보는 것'은 그 캔버스의 **편집 화면(B)에 들어가는 것과 같은 동작**이다. 캔버스 목록에서 카드를 누르면 그 캔버스가 자기 화면에 크게 열리고, 그것이 곧 편집 진입이다. **별도의 읽기 전용 확대 열람 모드는 두지 않는다** — 드릴다운과 편집은 하나의 동작이다.
  - '끼어들지 않고 보기만' 하고 싶을 때(학생 작업을 멈추고 들여다보기)는 **캔버스 잠금(F8 — 전원 작업 동결)**으로 흡수한다. 캔버스를 잠그면 **전원(학생·교사 모두) 편집이 멈춘 상태**가 되어 손대지 않고 상태를 들여다볼 수 있다(잠가도 입장·열람은 계속 허용). 들여다보다 정리가 필요하면 상단바 잠금 버튼으로 잠금을 풀고 작업한 뒤 다시 잠그면 된다. 보기 전용 드릴다운을 위한 별도 장치는 불필요하다.
  - **지휘 대상 범위는 교사가 보고 있는 목록 화면의 범위로 자연 결정된다** — 프로젝트 전체 목록에서 들어가면 프로젝트 전체가, 한 그룹 안 목록에서 들어가면 그 그룹이 지휘 대상이다. 지휘 대상을 따로 고르는 장치는 PoC 범위 밖이다.
- **필수 정보**: 캔버스 목록(썸네일, 편집 후 2초 이내 갱신, 잠긴 캔버스는 🔒 잠금 배지·"작업이 불가능해요" 텍스트 — F1), 캔버스별 참여 인원·객체 수, 드릴다운(편집 화면 B)으로 연 캔버스의 **객체별 작성자**(객체 클릭 시 옵션 바 우측 아바타·클릭 시 풀네임, 익명 옵션과 무관하게 소유자·편집자에게 항상 표시)
- **필수 행동**: 목록에서 특정 캔버스를 크게 열어 보기(= 편집 화면 B 진입), 타이머·잠금 제어, **객체 클릭 시 수정 옵션 바로 작성자 확인**(F13). B 편집 화면에서 **상단바 잠금 버튼 토글**로 잠금/해제(F8 — 잠금 중에는 본인도 작업이 멈추지만 잠금 해제는 가능). 캔버스 이름은 상단바에서 더블클릭으로 인라인 수정(F1).
- **품질 기준**: 캔버스 6개 기준 목록 한 화면에서 스크롤 없이 전체 파악 가능
- **하지 않는 것**: 선택 캔버스를 별도 전체 공유 화면으로 송출(broadcast — 범위 밖) · 채점·참여자별 기여도 표시(읽기 = 분포 파악·막힌 곳 돕기·좋은 사례 하이라이트) · 편집과 구분되는 별도의 읽기 전용 확대 열람 모드(드릴다운 = 편집 진입으로 확정)

**F8. 타이머·작업 동결(잠금)** *(소유자·편집자)*

- **필수 행동**: 캔버스에 제한 시간 설정, 시작·일시정지·연장·종료 · 만료 시 해당 캔버스 편집 자동 잠금(객체 유지), 재개방 가능
- **잠금 = '전원 작업 동결'의 정의(확정)**: 캔버스를 잠그면 **소유자·편집자·참여자 모두** (a) **캔버스 작업이 멈춘다**(객체 생성·수정·이동·삭제 모두 불가) — 운영자도 잠긴 캔버스 위에서는 객체를 손대지 못한다. 추가로 (b) **참여자 커서는 더는 보이지 않는다**. **입장·열람은 모두에게 계속 가능하다** — 잠금은 사람의 입장을 막는 것이 아니라 캔버스 위 '작업'을 멈추는 것이다. 따라서 잠긴 캔버스에 들어오는 사람은 입장 안내(F)가 아니라 보기·페이지 넘김이 가능한 캔버스 화면으로 들어간다(잠금 상태는 E 화면 배너로 알림 — F5).
  - **잠금 해제는 소유자·편집자가 한다(운영 컨트롤).** 잠금 해제는 '캔버스 작업'이 아니므로, 소유자·편집자는 잠긴 상태에서도 상단바 잠금 버튼으로 잠금을 풀 수 있다. 잠금을 풀면 즉시 평소 편집 가능 상태로 돌아간다(객체는 유지). 참여자는 잠금을 풀 수 없다.
- **잠금 사용자 노티 표면(화면별 — 확정)**: 각 화면이 잠금 상태를 다음처럼 알린다.
  - **A 캔버스 목록**: 잠긴 캔버스 카드에 **`🔒 잠금` 상태 배지** + **"작업이 불가능해요"** 안내 텍스트. 카드 더보기(⋮)에 **'잠금'**(잠긴 카드는 '잠금 해제') 항목(F1).
  - **B 편집자 뷰(교사 area)**: **상단바 잠금 버튼이 토글**된다 — 해제 상태 = 열린 자물쇠, 잠금 상태 = 닫힌 자물쇠. 잠금 시 버튼이 활성 표시되고 그 **우측에 "캔버스 잠김" 텍스트**가 뜬다. **잠금 중에는 편집자 도구 팔레트도 비활성**(소유자·편집자도 작업 동결이므로). 단 잠금 버튼 자체는 활성으로 남아 소유자·편집자가 잠금을 풀 수 있다.
  - **E-참여자(학생 area)**: **상단 배너 "관리자가 캔버스를 잠갔어요 · 지금은 보기만 할 수 있어요"** + **하단 도구 팔레트 비활성**(F5).
  - **E-뷰어(학생 area)**: **상단 배너 "관리자가 캔버스를 잠갔어요 · 지금은 아무도 편집할 수 없어요"**(뷰어는 원래 도구 없음 — F5).
- **품질 기준**: 잔여 시간이 교사 화면·참여자 화면에 동기 표시, 오차 1초 이내 · 잠금/해제가 같은 캔버스 전원 화면에 2초 이내 반영(전원 작업 동결·도구 비활성·노티 표면 포함)

**F9. 퀴즈앤 진입 동선·퍼널 계측** *(distribution)*

- **진입 방식**: 퀴즈앤 서비스에서 **링크 이동**으로 board-v2에 진입한다. board-v2는 **데이터 관리·도메인이 퀴즈앤과 완전 별개**이므로, v2 내부의 진입점 위치 설계는 우리 범위 밖이며 퀴즈앤은 링크만 연결한다.
- **첫 사용 동선**: 링크 진입 → board-v2 화면으로 이동하면서 동시에 '템플릿 생성' 모달 자동 오픈 → 빈 캔버스 또는 템플릿 3종 중 선택 시 즉시 캔버스 진입(기본 이름 "새 캔버스") → 캔버스 안에서 **공유 패널을 열어 초대 권한(편집자/참여자/뷰어)을 선택하고 그 권한의 QR을 발급**(실측 2액션 + 권한 선택 1액션). 이후 캔버스 목록으로 돌아오면 명시적 '캔버스 생성' 액션으로 같은 흐름을 다시 탈 수 있다(F1).
- **필수 행동**: 진입 → 캔버스 생성/복제 → (권한 선택 후 QR 발급) → 참여자 입장 → 재사용, 각 단계 이벤트 계측(소유자 계정+캔버스 ID 키, 비로그인 참여자는 디바이스 근사)
- **품질 기준**: 6절 지표 산출에 필요한 전 이벤트가 빠짐없이 수집될 것

**F10. 포스트잇 감정표현** *(참여자·소유자·편집자 / 뷰어 불가)*

- **적용 범위**: 감정표현은 **포스트잇 객체에만** 적용된다. 텍스트·도형·그리기 등 다른 객체에는 감정표현이 없다.
- **다종 이모지**: 공감 1종이 아니라 **여러 종류의 감정 이모지**를 제공한다(예: ❤️ 👍 😮 😂 👏). 참여자는 포스트잇에 그중 하나를 남긴다.
- **필수 정보**: 포스트잇별 감정 이모지 **종류별 카운트**(어느 이모지가 몇 개인지)
- **필수 행동**: 다른 사람의 포스트잇에 감정 이모지를 남김·교체·취소. **한 포스트잇에 1인 1감정** — 한 사람은 같은 포스트잇에 이모지 1개만 남길 수 있고, **다른 감정을 누르면 기존 감정이 교체**된다(추가가 아니라 교체). 자기 감정은 다시 눌러 **취소**할 수 있다(0으로). 집계는 **종류별 카운트**로 보여준다.
- **뷰어**: 뷰어는 **감정표현 불가**(보기 전용). 확정.
- **품질 기준**: 감정 반영 2초 이내(같은 캔버스 전원)
- **하지 않는 것**: 댓글(영구 제외 — 7절 비목표 1) · 포스트잇 외 객체에 대한 감정표현(포스트잇 한정) · 한 포스트잇에 한 사람이 2개 이상 감정 동시 부여(1인 1감정 — 교체만)

**F11. 그룹 관리** *(소유자·편집자)*

- **개념**: 그룹은 **캔버스의 상위 계층이 아니라**, 프로젝트 안에서 **캔버스들을 묶는 선택적 폴더/라벨**이다(캔버스와 같은 레벨 — 2절). 캔버스는 그룹에 넣지 않아도 프로젝트 바로 아래 존재한다.
- **필수 정보**: 캔버스 목록 화면의 그룹 카드(그룹명 라벨, 소속 캔버스 썸네일들)
- **필수 행동**: 프로젝트 안에서 그룹 생성·이름 지정(예: "1차시", "모둠 발표"), 캔버스를 그룹에 넣고 빼기, 그룹 단위 정리. 캔버스를 어느 그룹에도 넣지 않은 채 둘 수도 있다. **그룹 이름은 A 화면의 그룹 헤더에서 더블클릭으로 그 자리에서 수정**한다(인라인 편집 — 별도 모달 없이).
- **하지 않는 것**: 그룹 단위 일괄 권한·일괄 설정(후속) · 프로젝트당 캔버스 수·그룹당 캔버스 수 상한은 미정(OQ-5)

**F12. 객체 컨트롤 권한** *(전 역할 — 객체 단위)*

- **개념**: 객체(포스트잇·텍스트·도형·그리기)를 **누가 수정·이동·삭제할 수 있고 누가 잠글 수 있는가**의 규칙(1절 객체 컨트롤·잠금 권한 규칙). 객체 생성 권한(참여자가 새 객체를 만드는 것)과 별개로, 이미 놓인 객체를 **컨트롤(수정·이동·삭제)**하는 권한과 **잠금** 권한을 정한다. **컨트롤과 잠금은 권한 범위가 다르다** — 잠금은 편집자 이상 전용이다.
- **필수 행동**:
  - **소유자·편집자**: 작성자가 누구든 모든 객체를 수정·이동·삭제할 수 있고, 모든 객체를 **잠글 수 있다**. (단 캔버스가 잠긴 동안은 소유자·편집자도 객체 작업이 멈춘다 — F8 전원 동결.)
  - **참여자**: **본인이 작성한 객체만** 수정·이동·삭제할 수 있다. 다른 사람 객체는 보기·감정표현만 가능, 옮기거나 지울 수 없다. **객체 잠금은 본인 객체라도 할 수 없다(잠금 = 편집자 이상 전용).**
  - **뷰어**: 객체 컨트롤·잠금 불가(보기 전용).
- **필수 정보**: 각 객체의 작성자(및 수정자)가 누구인지 — **소유자·편집자는 객체 단위로 식별 가능해야 한다**. 식별은 객체 클릭 시 수정 옵션 바 우측 작성자 아바타(클릭 시 풀네임)로 드러난다(F13).
- **하지 않는 것**: 참여자가 다른 참여자 객체를 컨트롤(수정·이동·삭제) · 참여자의 객체 잠금(본인 객체 포함 — 잠금은 편집자 이상) · 객체별 개별 권한 부여(역할 기준으로만 결정)

**F13. 편집·운영 통제 (카오스 가드레일)** *(소유자·편집자, 일부 전 역할)*

- **개념**: 실시간 동시 편집의 카오스를 교사가 통제하기 위한 묶음(A3 가드레일의 구체화). 아래 동작들로 구성한다.
- **필수 행동**:
  - **참여자 커서 끄기(숨기기)** *(P0)*: 참여자의 마우스 커서(화살표)는 기본적으로 캔버스에 실시간 표시된다. 소유자·편집자는 이를 **끌 수 있다**(커서가 많아 화면이 어지러울 때 카오스를 줄인다 — 캔버스 설정 묶음, F6). 커서 표시 on/off는 소유자·편집자가 제어한다. (※ 캔버스 잠금 시에도 참여자 커서는 더는 보이지 않는다 — F8 전원 작업 동결의 일부.) — **P0 근거**: 동시 편집 카오스 억제가 A3 핵심 가정의 검증 조건이고, 다수 참여자 환경에서 가장 먼저 부딪히는 산만함이라 PoC에서 바로 필요.
  - **객체 클릭 시 수정 옵션 바 — 작성자 표시·객체 잠금** *(P0)*: 객체를 클릭하면 **객체 위(상단)에 수정 옵션 바**가 뜬다. 이 옵션 바의 **제일 우측 끝에 작성자 아바타**가 표시되어 어느 사용자가 만든 것인지 확인할 수 있고, **아바타를 클릭하면 작성자의 풀네임**이 드러난다. 소유자·편집자에게는 익명 옵션과 무관하게 항상 보인다(F12의 작성자 식별을 동작으로 구현). — **P0 근거**: 객체 컨트롤 권한(F12)과 교사의 '막힌 곳 돕기·하이라이트' 동선이 작성자 식별에 직접 의존하므로 함께 들어가야 한다.
    - **객체 잠금** *(P0)*: 같은 수정 옵션 바에서 객체를 **잠글 수 있다 — 단 소유자·편집자만 가능하다(참여자는 본인 객체도 잠글 수 없다)**. **잠긴 객체는 잠근 사람 외에는 수정·이동·삭제할 수 없다.** **이 '객체 잠금'은 객체 하나만 잠그는 것이고, 캔버스 전체를 동결하는 '캔버스 잠금'(F8)과는 다른 동작이다.** — 객체 잠금은 OQ-4(같은 객체 동시 조작 충돌)를 부분 완화하는 장치다(잠가 두면 그 객체에 대한 동시 조작 충돌 자체가 줄어든다). 단 동시 조작 충돌 처리 일반(두 사람이 잠기지 않은 같은 객체를 동시에 건드릴 때 무엇을 보여줄지)은 별개로 OQ-4에서 계속 다룬다. — **P0 근거**: 작성자 표시와 같은 옵션 바에서 제공되는 통제 동작이고, 교사가 완성·중요 객체를 학생 실수로부터 보호하는 데 즉시 쓰인다.
  - **참여자 입장 승인** *(P1)*: 참여자가 캔버스에 접근할 때 **기본 = 수동 승인**(교사가 승인해야 입장), **입장 자동 승인 옵션(기본 OFF)** 제공(켜면 새 참여자가 승인 없이 입장), 승인은 대기자별 승인/거절 + 다수 대기 시 **'모두 승인' 버튼**(F5·F6). 편집자도 입장 승인 대상이 될 수 있다. **내보낸 사람의 재입장은 자동 승인 ON이어도 승인을 거친다**(아래 내보내기). — **P1 근거**: 카오스 억제에는 도움이 되나, 기본을 '수동 승인'으로 두면 진입 마찰이 늘어 입장 무마찰(핵심 가치)과 긴장 관계다. 동선이 자리잡은 뒤 자동 승인을 켜도 되므로 커서 끄기·작성자 표시·객체 잠금보다 한 단계 뒤로 둔다.
  - **내보내기(강제 퇴장)** *(P1)*: 소유자·편집자가 **접속자 모달의 각 사람 행에서 그 사람을 캔버스에서 강제로 내보낸다**. 내보내면 그 사람은 즉시 캔버스에서 빠지고, **다시 들어오려면 승인이 필요**하다 — 자동 승인 옵션이 켜져 있어도 내보낸 사람의 재입장은 승인을 거친다(내보내기가 자동 입장으로 무력화되지 않게).
    - **권한 경계**: **편집자는 참여자·뷰어까지** 내보낼 수 있고, **소유자는 편집자까지** 내보낼 수 있다. **소유자 자신·본인은 대상이 아니다**(편집자가 다른 편집자·소유자를 내보낼 수 없고, 누구도 자신을 내보낼 수 없다). 동작 = 접속자 모달 각 행 우측의 [내보내기] 버튼 → 확인 모달("OO님을 이 캔버스에서 내보냅니다. 다시 들어오려면 승인이 필요해요.").
    - **P1 근거**: 입장 승인·커서 끄기와 같은 운영 통제 묶음이며, 방해하는 참여자를 즉시 빼는 카오스 대응 수단이다. 다만 핵심 입장 동선(P0)보다 한 단계 뒤의 운영 보조라 P1로 둔다.
- **품질 기준**: 커서 표시 on/off 전환이 같은 캔버스 전원 화면에 2초 이내 반영 · 객체 잠금/해제가 같은 캔버스 전원 화면에 2초 이내 반영 · 입장 승인 후 참여자 입장까지 추가 마찰 없음(승인 즉시 입장) · 내보내기 실행 후 대상 사용자가 캔버스에서 빠지는 데 2초 이내 반영
- **하지 않는 것**: 페이지 고정 등 추가 가드레일(수요 확인 후) · 참여자별 권한 개별 조정(역할 기준만) · 소유자·본인을 대상으로 한 내보내기 · **잠긴 객체에 대한 편집자 전권 우선순위(잠금이 편집자 권한을 이기는지 = 편집자가 다른 편집자/소유자가 잠근 객체를 풀 수 있는지)는 잔여 — OQ-7에서 결정**

## 6. 성공 지표 및 기각 기준

> 사전 벤치마크가 없는 첫 측정이므로 목표치는 [잠정]이며, 검증 모수·기간(OQ-1) 확정 후 고정한다.

**선행 지표 (일~주 단위):**

- 첫 캔버스 생성률 = 퀴즈앤 링크로 v2 진입한 교사 중 첫 캔버스 생성 비율 · 검증 대상 A1 · 목표 **5% [잠정 — OQ-1 확정 후 고정]** · 측정 = 오픈 후 4주, F9 퍼널 로그
- 캔버스당 참여율 = 입장 참여자 중 객체 1개+ 생성 비율 · 검증 대상 A3 · 목표 TBD — OQ-1 확정 후 고정 · 측정 = F4 편집 로그

**후행 지표 (주~월 단위):**

- 반복 사용률 = 첫 생성 교사 중 **2회 이상 활동** 비율("재사용" = 새 캔버스 생성 또는 기존 캔버스 복제 [잠정]) · 검증 대상 A2 — **이 PoC의 핵심 신호** · 목표 TBD — OQ-1 확정 후 고정

**기각(킬) 기준:** 오픈 4주 후 첫 캔버스 생성률 2% 미만 **AND** 반복 사용 교사 한 자릿수 → distribution 가설(A1)·수요 가설(A2) 재검토, 빌드 확장 중단.

- **데이터·도메인 분리**: board-v2는 퀴즈앤과 **데이터 관리·도메인이 완전 별개**다. 퀴즈앤과의 연결은 링크 이동뿐(F9). 단 로그인 계정 체계는 보류 상태로, **퀴즈앤 계정계 기반 SSO 연동을 검토 중**(미확정 — OQ-2). 비로그인 닉네임 세션과 로그인 계정을 잇는 신원 연속성도 함께 미해결(OQ-3)
- **플랫폼**: 태블릿·크롬북·스마트폰의 브라우저에서 동작, 특정 OS·브라우저 종속 금지, 앱 설치 없음
- **언어**: 한국어 전용(PoC 한정). 학생 대면 문구는 운영 주체를 **'관리자'**로 표기('선생님' 아님 — 2절 카피 가이드), 잠금 안내 배너도 '관리자' 표기("관리자가 캔버스를 잠갔어요 …" — F5·F8)
- **보존·삭제 정책**: 캔버스는 사용자가 명시적으로 '삭제'하기 전까지 존속한다(자동 만료·자동 삭제 없음). 캔버스 삭제는 사용자의 명시적 동작으로만 일어나며, **'완전 삭제·되돌릴 수 없음'을 알리는 확인 모달을 거친다**(오삭제 방지 — F1·F6). **PoC에는 보관(archive) 상태·보관함이 없다**(2절 캔버스 상태, 7절 비목표 18).
- **개인정보**: 비로그인 참여자·뷰어 수집 항목 = **닉네임(최대 15자) + 비밀번호 4글자**뿐, 이름·학번·연락처 미수집 · 단 객체 작성자 기록이 닉네임과 연결되므로 익명 옵션 시 노출 범위 준수(1절 가시성 규칙) · **비회원 비밀번호 찾기 장치 없음**(분실 시 복구 미제공 — 7절 비목표 15) · **초등 포함으로 만 14세 미만 해당 확정** → 아동 개인정보 규정 적합 여부·미성년 개인정보 보존/삭제 책임은 설계 단계 필수 확인(보관 개념이 없으므로 '보관 캔버스의 개인정보 보존' 항목은 별도 쟁점이 아니며, 일반 캔버스의 개인정보 보존·삭제 정책에 통합)
- **비용**: PoC 운영비 최소화 원칙 · 상한 TBD — OQ-1과 함께 산정
- **자기잠식 경계**: 설계 산출물마다 퀴즈앤 기존 '보드'(wall)와의 중복 여부 점검 — canvas 차별점(동시 공동 조작) 유지
- **제품 명칭**: "board-v2"는 가명(코드네임), 정식 명칭 미정 — 로고 영역은 우선 'board-v2' 텍스트로 표기(0절)
