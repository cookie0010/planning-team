# Canvas — 사용자 유형별 워크플로우

- **Version:** 0.1 (프리릴리스 — 사용자 검증 전)
- **작성일:** 2026-06-18
- **기준:** `canvas/wireframe/` (완성된 와이어프레임이 진실의 원천). 권한·규칙은 `prd/prd-v1.0.md` 참조.
- **읽는 법:** 디자인·개발이 "누가, 어떤 화면에서, 무엇을 해서, 어디로 가는가"를 한눈에 보기 위한 흐름 문서다. 화면 디테일은 와이어프레임을, 무엇을·왜는 PRD를 본다. 여기선 *흐름*만.

> 이 문서는 IA 문서를 대체한다. 와이어프레임이 이미 화면·인터랙션을 확정했으므로, 남은 빈칸은 "화면을 잇는 흐름"뿐이라 그것만 정리했다.

---

## 역할 한눈에

| 역할 | 누구 | 계정 | 한 줄 |
|---|---|---|---|
| **소유자** | 캔버스를 만든 교사 | 로그인 필수 | 캔버스 주인. 모든 권한 + 편집자까지 내보낼 수 있음 |
| **편집자** | 같이 운영하는 교사 | 로그인 필수 | 소유자와 거의 동일 운영 권한. 단 다른 편집자·소유자는 못 내보냄 |
| **참여자** | 학생 | 불필요 (닉네임+비번4) | 본인 객체만 만들고 편집. 감정표현 가능 |
| **뷰어** | 학생 | 불필요 (닉네임+비번4) | 보기만. 생성·편집·감정 전부 불가, 페이지 넘김만 |

화면은 크게 둘로 나뉜다 — **교사(소유자·편집자)는 같은 화면(A·B)을 쓰고 권한만 다르고**, **학생(참여자·뷰어)은 같은 입장 흐름(D)을 타고 협업 화면(E)에서 갈린다.**

---

## 1. 교사 — 소유자 워크플로우

캔버스를 만들고, 학생을 들이고, 수업 중 운영하고, 마무리한다.

### 1-1. 준비 (캔버스 만들기)

```
[로그인] → A (캔버스 목록)
   │  ※ 처음이면 A·빈(empty) — CTA 하나로 첫 캔버스 유도
   │
   ├─ [+ 캔버스 생성] → 템플릿 선택 모달 → B (새 캔버스 편집)
   │     · 기본명 '새 캔버스' (상단 제목 더블클릭으로 인라인 수정)
   │
   └─ 목록 관리(선택)
         · 카드 ⋮ → 복사 / 잠금·해제 / 삭제(재확인 모달)
         · 그룹 폴더 클릭 → A·그룹(드릴인) → 브레드크럼으로 복귀
         · 탭: 전체 / 진행 중 / 공유된 캔버스(내가 편집자로 들어간 것)
```

### 1-2. 초대 (학생·동료 들이기)

B 화면 상단에서:

```
[공유] → 공유 모달
   ├─ 권한 탭 [참여자] [편집자] [뷰어] — 탭마다 다른 QR/URL
   ├─ 참여자·뷰어용 QR: 학생에게 스캔시킴 → 학생은 D-입장으로
   └─ 편집자용 QR/URL: 10분 만료. 동료 교사 로그인 입장용

[설정 ▾]에서 입장 정책 결정
   ├─ 입장 자동 승인 ON/OFF  ← 승인 대기를 둘지 결정 (1-3에서 효과)
   ├─ 익명 옵션
   └─ 참여자 커서 숨기기
```

### 1-3. 운영 (수업 중)

```
B (캔버스 편집) — 운영 컨트롤
│
├─ 입장 승인        [입장 대기 N] → 승인 모달
│     · 개별 [승인]/[거절], [모두 승인], [자동 승인 토글]
│     · 승인 → 학생 E 진입 / 거절 → 학생 F(거절) 안내
│     · ※ 자동 승인 ON이면 이 단계 안 거치고 학생 직진입
│
├─ 사람 관리        [접속자 N명] → 접속자 모달
│     · 역할 셀렉트로 편집자 임명·해제
│     · [내보내기](강제 퇴장): 소유자는 편집자까지 / 본인·다른 소유자는 불가
│     · 내보낸 사람 재입장 시 → 자동 승인이어도 승인 강제
│
├─ 진행 제어        ⏱ 타이머(정지·리셋·+5/10/30/60분) · 🔒 잠금
│     · 잠금 = 전원 작업 동결(아래 §공통 규칙 참조). 입장은 안 막음
│
├─ 캔버스 작업      페이지 추가(≤30) · 객체 생성/편집/이동/삭제(모든 객체)
│     · 객체 클릭 → 수정 바(색·글자·잠금·삭제 + 작성자 아바타)
│     · 객체 개별 잠금 / 포스트잇 감정표현
│
└─ [← 돌아가기] → A
```

### 1-4. 마무리

- 잠금(🔒)으로 전원 동결해 결과 고정 → 보기/회고
- 또는 그냥 종료. (보관 개념은 이번 PoC 제외 — §열린 점)

## 2. 교사 — 편집자 워크플로우

화면·흐름은 **소유자와 동일(A·B)**. 권한만 다음 한 군데에서 좁다.

| 항목 | 소유자 | 편집자 |
|---|---|---|
| 내보내기(강제 퇴장) 대상 | 편집자·참여자·뷰어 | 참여자·뷰어만 (다른 편집자·소유자 불가) |
| 편집자 임명 | 누구든 | 로그인 사용자만 |
| 그 외 운영(승인·잠금·타이머·설정·객체) | ✓ | ✓ (동일) |

진입: 편집자용 QR/URL(10분 만료) 스캔 → 로그인 → A/B 합류.

## 3. 학생 — 참여자 워크플로우

```
참여자용 QR 스캔
   │
   ▼
D-입장
   ├─ 닉네임(≤15자) — 같은 캔버스에 중복이면 차단("이미 같은 이름…")
   ├─ 비밀번호 4자리
   └─ [입장하기]
        │
        ├─ 정원(20) 초과? ──── YES ─→ F-안내(정원 초과) ─[다시 시도]→ D
        │
        ├─ 자동 승인 ON ─────────────────→ E-참여자 (직진입)
        │
        └─ 자동 승인 OFF ─→ D-승인대기 ─┬─ 승인 ─→ E-참여자
                                        ├─ 거절 ─→ F-안내(거절) ─[다시 요청]→ D
                                        └─ 타임아웃 ─→ F-안내(거절)
   ▼
E-참여자 (함께 만들기)
   ├─ 본인 객체: 클릭 → 수정 바(색·글자·삭제) — 본인 것만
   ├─ 남의 객체: 편집 불가, 감정표현만 (포스트잇당 1인 1감정, 누르면 교체)
   ├─ 잠긴 객체: 못 건드림. 감정만
   ├─ 도구 팔레트로 본인 객체 생성 / 페이지 넘김(‹ ›)
   ├─ 타이머·접속자: 읽기만
   └─ 캔버스 잠기면 → 배너 + 도구 비활성(작업 동결, 보기만)

   ※ 교사가 [내보내기] 하면 → D로 튕겨나감(재입장 시 승인 필요)
```

---

## 4. 학생 — 뷰어 워크플로우

입장 흐름은 참여자와 동일(D → 승인대기/정원/거절 분기). 협업 화면만 다르다.

```
뷰어용 QR 스캔 → D-입장(뷰어 배지) → (승인 분기 동일) → E-뷰어

E-뷰어 (보기 전용)
   ├─ 모든 객체 클릭 불가 — 작성자·감정·잠금배지 '읽기'만
   ├─ 감정표현 불가(⊕ 없음) · 도구 팔레트 없음("보기 전용" 안내)
   ├─ 페이지 넘김은 자유(‹ ›) — 본인 속도로 열람
   └─ 캔버스 잠겨도 보기 전용이라 변화 없음(배너만)
```

## 5. 공통 규칙 (워크플로우를 가로지름)

### 입장·승인
- 정원 **20명**. 초과 시 입장 자체 차단 → F-안내(정원 초과).
- 닉네임 중복은 같은 캔버스 내에서 차단.
- 자동 승인 OFF가 기본 — 교사가 승인 모달에서 통제. ON이면 대기 없이 직진입.

### 잠금(🔒) = 전원 작업 동결
- 입장은 막지 않음. **소유자·편집자 포함 전원**이 객체 생성/수정/감정 불가.
- 잠긴 동안에도 교사는 **잠금 해제·타이머·설정** 운영 컨트롤은 가능.
- 참여자: 도구 비활성 + 배너. 뷰어: 원래 보기 전용이라 배너만.
- 목록(A)에서는 카드 상태 배지로만 표시(별도 탭 아님).

### 내보내기(강제 퇴장)
- 위치: B 접속자 모달의 각 행.
- 권한: 소유자→편집자까지 / 편집자→참여자·뷰어까지 (본인·상위 제외).
- 내보낸 사람 재입장 시 자동 승인 무효 — **항상 승인 필요**.

### F-안내 변형 2종
- 정원 초과 / 입장 거절. ([돌아가기]는 로그인 사용자에게만 노출.)
- 잠금 변형은 폐기됨(잠금은 입장을 막지 않으므로).

## 7. 열린 점 (흐름에 영향 주는 것만)

- **신원 연속성 미해결(OQ-12·26):** 비로그인 닉네임 입장자가 나중에 로그인해도 '같은 사람'으로 안 이어짐 → 재진입(G) 워크플로우가 아직 못 굳음.
- **보관 개념 PoC 제외:** 종료한 캔버스를 보관·재개하는 흐름은 이번 범위 밖.
- **검증 모수(OQ-02, blocking):** 정원 20·페이지 30 등 수치는 검증 전 가정.
