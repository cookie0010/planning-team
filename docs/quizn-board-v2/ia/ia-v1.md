## 0. 이 문서가 무엇인가

이 제품에 **화면이 몇 개 있고, 각 화면이 무엇을 보여주고, 화면끼리 어떻게 이어지는지**를 적은 설계도다. 실제 그림(와이어프레임)을 그리기 전에, 그릴 대상의 목록과 연결을 먼저 확정하는 단계다.

전체는 두 세계로 나뉜다 — **교사가 쓰는 화면**(데스크톱)과 **학생이 쓰는 화면**(태블릿·폰). 둘이 만나는 유일한 접점은 캔버스다.

- 교사 화면 기준: 데스크톱 1440px
- 학생 화면 기준: 태블릿 1024px (+ 폰 390px 변형)

## 1. 화면 목록

### 교사 화면 (3개)

| 코드 | 화면 | 무엇을 보여주나 | 무엇을 하나 | PRD 기능 |
|---|---|---|---|---|
| **A** | 프로젝트 목록 (= 지휘석) | 내 프로젝트(=캔버스) 카드들, 그룹으로 묶여 정리됨 | 새 프로젝트 만들기·복제·삭제, 그룹 정리, 캔버스 열기. **수업 중엔 카드가 2초마다 갱신되는 라이브 썸네일**이 되어 지휘석 역할 | F1·F2·F7·F11 |
| **B** | 캔버스 편집 (편집자 뷰) | 페이지(좌우로 넘김), 도구바, 잔여 시간, 객체 작성자 | 객체 만들기·편집, **페이지 추가·템플릿 설정**, QR 띄우기, 타이머·잠금 | F4·F3·F8 |
| **C** | 프로젝트 설정 | 편집자 목록, 익명/닉네임·비번 옵션 상태 | 편집자 임명(캔버스 단위), 옵션 on/off | F6 |

**별도 화면이 아니라 위에 겹쳐 뜨는 것들:**
- **템플릿 생성 모달** — A에 진입하면 자동으로 뜸. 템플릿 3종 카드 중 선택 → 즉시 B로 진입
- **QR 표시 패널** — B 안에서 [QR] 버튼으로 띄움 (교실 프로젝터용 큰 화면)
- **스포트라이트 · 드릴다운** — A(지휘석)의 동작. 한 캔버스를 골라 전체 공유 화면에 띄우거나(스포트라이트), 그 캔버스로 들어감(드릴다운)

### 학생 화면 (4개)

| 코드 | 화면 | 무엇을 보여주나 | 무엇을 하나 | PRD 기능 |
|---|---|---|---|---|
| **D** | 입장 화면 | 캔버스 이름, 닉네임 입력란(옵션 시 비번란) | 닉네임 입력 → 입장 | F5 |
| **E** | 캔버스 편집 (참여자 뷰) | 편집자가 짜둔 페이지, 하단 도구바, 잔여 시간, 누가 들어왔나(presence) | 객체 만들기·편집, 공감 투표. **페이지 추가·템플릿 수정은 불가** | F4·F10 |
| **F** | 안내 화면 | 상황 메시지 | 없음(읽고 대기) — 정원 초과 / 잠금(보기 전용) / 보관, 3종 변형 | F5·F8 |
| **G** | 내 활동 (로그인 시) | 참여했던 캔버스 목록 | 재진입 | F5 |

> 교사가 캔버스에 직접 들어간 모습은 별도 화면이 아니라 **B(편집자 뷰)** 그 자체다 — 학생의 E에 타이머·잠금·페이지 편집 권한이 더해진 것.

## 2. 화면 연결 (네비게이션)

```
교사 세계 (데스크톱)
  A 프로젝트 목록(=지휘석)
   ├─(자동) 템플릿 생성 모달 ──선택──▶ B 캔버스 편집
   ├─(복제) ────────────────────────▶ 복제본 = 또 다른 B
   ├─(드릴다운) ─────────────────────▶ B 캔버스 편집
   ├─(설정) ─────────────────────────▶ C 프로젝트 설정
   └─(스포트라이트) ──── 전체 공유 화면에 한 캔버스 띄움

  B 캔버스 편집
   └─[QR] ─── QR 표시 패널 ─── (학생에게 전달)

학생 세계 (태블릿·폰)
  QR/URL ──▶ D 입장 ──▶ E 캔버스 편집
                         └─(정원초과·잠금·보관) ──▶ F 안내 화면
  로그인 학생: G 내 활동 ──▶ E 재진입
```

두 세계의 연결고리는 단 하나 — **B에서 띄운 QR이 학생의 D(입장)로 이어진다.**

## 3. 핵심 흐름 4개

**① 교사 첫 사용 (실측 2액션)**
```
퀴즈앤 링크 클릭 → A(목록)+템플릿 모달 자동 → [템플릿 선택] → B(캔버스) → [QR 버튼] → QR
```
PRD의 "3액션·30초" 기준을 충족(오히려 2액션). 첫 진입 시 빈 프로젝트 목록 위에 템플릿 모달이 바로 떠서, 고르는 순간 캔버스로 들어간다.

**② 교사 재사용 (복제)**
```
A(목록) → [프로젝트 복제] → 복제본(B) → QR
```
잘 만든 프로젝트 하나를 1반·2반·3반용으로 통째 복제. 복제본은 각각 독립이고 QR도 따로.

**③ 학생 입장**
```
QR 스캔 / URL → D(입장: 닉네임) → E(편집)
```
무설치·무가입. 옵션이 켜져 있으면 비번도 입력. 로딩 3초 이내 목표.

**④ 교사 지휘 (수업 중)**
```
A(=지휘석) 라이브 썸네일 → 드릴다운(→B) / 스포트라이트 / 타이머 제어
```
캔버스 6개를 한 화면에서 스크롤 없이 파악. 막힌 모둠 돕고, 좋은 사례를 전체에 띄운다.

## 4. 아직 안 정한 것 (그림 그리며 판단)

- **복제하면 자동으로 그룹이 되나?** 1반·2반·3반을 복제했을 때 자동으로 한 그룹으로 묶일지, 교사가 수동으로 묶을지. → 일단 **수동**으로 가정.
- **지휘석은 전체를 보나, 한 그룹만 보나?** 캔버스 6개 한 화면 기준이면 보통 한 그룹(한 활동의 복제본들) 단위로 모아 보는 게 자연스럽다. → FigJam 흐름도에서 눈으로 판단.
- **그룹 없이 시작할 때의 A 화면** — 첫 사용자는 그룹도 프로젝트도 없다. 빈 목록 + 템플릿 모달이 기본.
