## 0. 이 문서가 무엇인가

이 제품에 **화면이 몇 개 있고, 각 화면이 무엇을 보여주고, 화면끼리 어떻게 이어지는지**를 적은 설계도다. 실제 그림(와이어프레임)을 그리기 전에 그릴 대상의 목록과 연결을 먼저 확정하는 단계다. 이번 v10은 v9의 골격은 그대로 두고, **객체를 다루는 방식(공감 투표·작성자 확인·객체 잠금)과 몇몇 열린 점(뷰어 공감·보관 보존·편집자 QR 만료)을 확정값으로 못 박았다.** 화면 개수·코드·네비게이션 골격은 v9와 같고, 주로 B 화면의 객체 인터랙션과 E-참여자·E-뷰어 서술이 또렷해졌다.

전체는 두 area로 나뉜다 — **교사가 쓰는 화면**(데스크톱)과 **학생이 쓰는 화면**(태블릿·폰). 둘이 만나는 유일한 접점은 캔버스다.

- 교사 화면 기준: 데스크톱 1440px
- 학생 화면 기준: 태블릿 1024px (+ 폰 390px 변형)

### 먼저 짚는 용어: 역할 4단계

화면을 보기 전에 한 줄씩 정의한다. PRD v11 §1을 그대로 따른다.

- **소유자** = 프로젝트를 만든 사람(보통 교사). 로그인 필수. 모든 것을 제어한다.
- **편집자** = 소유자나 다른 편집자가 임명·초대한 사람. 로그인 필수. 임명된 캔버스의 페이지 추가·템플릿 설정·편집이 가능하다.
- **참여자** = 캔버스에 들어와 **함께 만드는** 사람(보통 학생). 무계정 입장 가능. 객체 생성·이동·편집 + 포스트잇 공감 투표. 단, **수정·이동·삭제·잠금은 본인이 만든 객체에만** 가능하다(F12).
- **뷰어** = 캔버스에 들어와 **보기만** 하는 사람. 무계정 입장 가능. 객체 편집 불가, 페이지 변경 불가, **공감 투표 불가**(v10 확정), 실시간 반영을 보기만.

> 화면 수준에서 핵심은: 소유자·편집자는 **교사 area**(데스크톱)에 산다. 참여자·뷰어는 **학생 area**(태블릿·폰)에 산다. 참여자와 뷰어는 **같은 화면(E)**을 쓰되, 뷰어는 그 화면의 '보기 전용' 변형으로 들어온다.

### 계층 (v9 정정 유지 — 그룹은 캔버스의 상위가 아니다)

**핵심 단위는 캔버스다.** 캔버스는 프로젝트 바로 아래에 놓이고, 그 안에 페이지와 객체가 담긴다. **그룹은 그 캔버스들을 묶어 정리하는 선택적 수단(폴더/라벨)** 일 뿐이라 캔버스와 같은 레벨에 있고, 안 묶어도 된다.

```
프로젝트 (최상위 통 — 소유자가 생성. PoC에선 사실상 1개. 다중 프로젝트 화면은 범위 밖, 개념상으로만 위에 존재)
 └ 캔버스 (★핵심 단위 — 권한별 QR · 학생 진입 단위 · 진입자 20명(편집자·참여자·뷰어 모두 포함) · 최대 30페이지)
     └ 페이지 (편집자가 템플릿을 붙여 구성 · 최대 30개 · 좌우 추가·이동)
        └ 객체 (텍스트·그리기·포스트잇·도형) — 작성자 정보가 남고, 잠글 수 있음(v10)

 (그룹 = 선택. 캔버스들을 묶는 폴더/라벨. 캔버스와 동급이지 상위 층이 아니다. 안 묶어도 됨)
```

쉽게 말하면: 캔버스가 '파일'이고, 그룹은 그 파일들을 넣어 두는 '폴더'다. 폴더가 파일의 '상위 종류'가 아니듯, 그룹도 캔버스의 '상위 계층'이 아니다.

> 화면 매핑상 '그룹'은 별도 층이 아니라 **A 안에서 폴더 카드로 드러나고, 그 폴더를 누르면 A의 한 단계 안쪽 뷰(A-그룹 내부)** 로 들어간다. 이 화면 동작은 v9와 같다.

> 화면 본체는 A·A-그룹 내부·B 셋이다(C 독립 화면은 v9에서 폐기). v10에서 화면 개수·코드 변화는 없고, B의 객체 인터랙션이 또렷해졌다.

#### 새 캔버스 만들기 모달 (독립 화면 아님 — v9 그대로)

'템플릿 선택'은 독립 화면이 아니라 **여러 화면에서 같은 코드로 뜨는 공용 모달**(`wf-screen.js`)이다. 화면 목록에 올리지 않고 오버레이로 둔다.

- 내용 = **빈 캔버스 + 템플릿 3종**(빈 캔버스·브레인스토밍·마인드맵, 개수는 OQ-03) 미리보기 그리드 + [취소]·[이 템플릿으로 시작].
- 뜨는 경로(트리거 = `[data-new-canvas]`) — 모두 같은 모달: A 헤더 [+ 캔버스 생성] / '새 캔버스 만들기' 타일 / A 빈 상태 CTA / A-그룹 내부 [+ 이 그룹에 캔버스] / **B의 [+ 페이지 추가]**(페이지별 템플릿 F3) / 링크 첫 진입 자동(F9).
- 한 가지 의미 차이: A 쪽에서 고르면 **새 캔버스**, B [+ 페이지 추가]에서 고르면 **그 캔버스의 새 페이지**. 화면(모달)은 같고 결과 단위만 다르다.

> **'빈 캔버스 기본 제공' = 확정(OQ-03 해소).** 템플릿 최종 개수만 OQ-03에 남아 있다.

#### B(캔버스 편집 = 드릴다운으로 연 화면) 구조

B는 캔버스 하나를 편집자가 짜고 수업 중 운영·지휘하는 화면이다. 교사가 A에서 카드를 누르면 곧 이 화면이 자기 화면에 크게 열린다(= 드릴다운). 와이어프레임(`t-b-edit.html`) 기준 네 영역이다.

- **상단바** — [← 목록 복귀], **캔버스명 + 상태 배지**, **타이머 제어**, **캔버스 잠금** 버튼, **참여자 커서 끄기 토글**, **참여자 입장 승인**, presence, **[설정]**(드롭다운), **[공유]**.
  - **타이머 제어** = 잔여시간 + **일시정지(⏸)** + **드롭다운(▾)**(큰 잔여시간·정지·리셋·+5/+10/+30/+60분, F8).
  - **캔버스 잠금** = 타이머와 별개 버튼. 캔버스 전체를 잠가 학생 편집을 멈춘다(상태 '잠금'으로). '끼어들지 않고 보기만' 하려면 이 잠금을 켠 채 드릴다운한다(F7·F8). **이 '캔버스 잠금'은 캔버스 전체를 멈추는 것**이고, 아래 '객체 잠금'은 **객체 하나만** 잠그는 별개 동작이다 — 둘을 혼동하지 않게 라벨을 구분한다.
  - **참여자 커서 끄기 토글(v9, F13·OQ-14)** — 참여자 커서 표시를 끈다(카오스 방지). 정확한 위치는 와이어프레임에서.
  - **참여자 입장 승인(v9, F13)** — 아래 별도 박스 참조.
- **페이지 레일** — 페이지 칩(현재 강조) + **[+ 페이지 추가]** + **N/30 표시**. 추가 버튼은 새 캔버스 모달을 띄워 페이지별 템플릿을 고른다(F3·F4).
- **캔버스(작업 영역)** — 격자 배경 위 객체(포스트잇·텍스트·도형·펜). **포스트잇은 작성자명을 좌하단에 상시 표시 + 공감(♥)을 우하단**에 둔다(아래 공감 박스). 좌우 페이지 넘김(‹ ›). **템플릿은 페이지별** 설정.
  - **객체 클릭 → 수정 옵션 바 + 우측 작성자 프로필(v10 확정, F12·F13·OQ-18)** — 객체를 클릭하면 **객체 위(상단)에 '수정 옵션 바'가 뜨고, 그 바의 제일 우측에 작성자 프로필**(누가 만들었는지)이 붙는다. 이게 v9의 '작성자·수정자 팝오버'를 대체하는 확정 형태다. 소유자·편집자는 익명 옵션과 무관하게 이 바의 우측에서 항상 작성자를 확인한다(PRD §1 가시성 규칙). 포스트잇 좌하단 작성자명 상시 표시와 정합하고, 텍스트·도형 등 작성자명이 상시로 안 붙는 객체는 이 바를 통해 확인한다.
  - **수정 옵션 바 안의 '객체 잠금'(v10 신규, F12 정합)** — 수정 옵션 바에 **객체 잠금** 동작을 둔다. 잠긴 객체는 **다른 사람이 수정·이동·삭제할 수 없다**. 잠글 수 있는 주체 = 그 객체를 컨트롤할 수 있는 사람 — **소유자·편집자는 모든 객체를, 참여자는 본인이 만든 객체를** 잠근다(F12와 같은 권한 축). 잠긴 객체에는 **시각 표시(자물쇠 배지 등)가 있다** — 정확한 형태는 와이어프레임에서. (이 '객체 잠금'은 객체 하나만 잠그는 것이라, 상단바의 '캔버스 잠금'[캔버스 전체 멈춤]과 다르다.)
  - **객체 컨트롤 권한(F12)** — B는 소유자·편집자 뷰이므로 **모든 객체를 옮기고·고치고·잠근다**. (참여자는 본인 객체만, 뷰어는 불가 — E에서 갈림. §3-⑤.)
- **하단 플로팅 도구 팔레트** — 선택·이동(손) | 텍스트·포스트잇·펜·도형 | + 더보기 | 실행취소·다시실행. → **도구 목록 자체는 미확정**(§4).

#### B 포스트잇: 작성자명 + 공감(♥) (v10 통일)

B의 포스트잇 객체는 본문 아래 발(footer)에 두 가지를 둔다 — **좌하단에 작성자명**(편집자 뷰에서 상시 표시), **우하단에 공감(♥)**. **이 공감은 학생 area의 E-참여자 공감 투표와 같은 것이다** — v9까지 둘(B의 '감정표현 ☺'과 E의 '공감 ♥')을 통일할지 미정이었으나, v10에서 **하나의 표현으로 수렴**시켰다:

- **포스트잇에만** 붙는다(다른 객체엔 공감 버튼이 없다, OQ-16 해소).
- **1인 1표**다 — 한 사람이 한 포스트잇에 한 번만 공감하고, **다시 누르면 취소**(토글). 카운트는 공감한 사람 수.
- B(교사 뷰)와 E(학생 뷰)가 **같은 아이콘·같은 카운트·같은 토글 규칙**을 쓴다(F10).

#### B의 공유 패널 (오버레이) — 편집자 임명도 여기 (v9 유지 + v10 한 줄)

B 상단바의 [공유]를 누르면 위에 겹쳐 뜨는 모달이다(교실 프로젝터용 큰 QR). **초대 권한 탭을 먼저 고른다 — 참여자 / 편집자 / 뷰어.** 고른 권한에 맞춰 **QR · URL · [링크 복사]**를 발급·표시한다. 권한마다 별개 링크이고, 이 권한 선택이 학생 입장(D)의 분기를 결정한다(F6). 와이어프레임 기본 선택 = 참여자.

- **'편집자' 탭이 임명+초대를 한곳에서 맡는다(v9 유지).** 이 탭에 (1) 소유자 행 + 임명된 편집자 행(각 행에 '임명 해제'), (2) [+ 편집자 임명](로그인 사용자 직접 임명), (3) 편집자 QR·URL·링크 복사가 함께 있다.
- **편집자 QR 초대 링크는 10분 만료(v10, OQ-17 해소).** '편집자' 탭의 QR/링크 초대는 발급 후 **10분 뒤 만료**된다(아무나 오래 떠 있는 링크로 편집자가 되지 못하게). 임명 해제도 이 탭에서 한다(위 (1)). 참여자·뷰어 QR의 만료 정책은 별개이며 여기 적용되지 않는다.

#### B [설정] = 드롭다운(팝오버) — v9 유지 (C 독립 화면 폐기)

B [설정]은 풀페이지 전환이 아니라 **상단바 버튼에 붙는 드롭다운(팝오버)** 이다. 항목 셋(F6):

- **익명 옵션 토글** — 켜면 참여자끼리 작성자 비공개(소유자·편집자는 수정 옵션 바 우측에서 항상 확인).
- **닉네임 + 비밀번호 4글자 안내** — "끌 수 없는 기본 설정" 안내만(토글 아님, F5).
- **캔버스 삭제(위험)** — 객체·페이지 모두 사라짐, 되돌릴 수 없음 안내.

> 편집자 임명·임명 해제는 여기 없다 — 공유 패널 '편집자' 탭이다. 설정 드롭다운은 캔버스 옵션(익명·식별·삭제)만 다룬다.

#### B의 참여자 입장 승인 (v9 유지, F13)

참여자가 입장하려 할 때 **기본은 교사의 수동 승인**이다. B에 이를 다루는 UI가 있다: **승인 대기 알림/영역**(대기자가 뜸, 한 명씩 승인/거절), **자동 승인 옵션**(켜면 이후 입장자는 바로), **[모두 승인]**(대기자 일괄). 이 단계는 학생 흐름에서 **D(입장) → '승인 대기' → E(캔버스)** 사이에 끼는 단계로 나타난다. 정확한 표면은 와이어프레임에서.

#### B의 오버레이/팝오버 정리 (별도 화면이 아닌 것들)

- **새 캔버스 만들기 모달** — A·A-빈 상태·A-그룹 내부·B(페이지 추가)·링크 첫 진입에서 같은 코드로 뜬다.
- **공유 패널** — 권한 탭 + 편집자 임명(QR 10분 만료 포함).
- **설정 드롭다운** — 익명·안내·삭제.
- **객체 수정 옵션 바(우측 작성자 프로필 포함)** — 객체 클릭 시 객체 위에 뜨는 바. 작성자 확인·객체 잠금·기타 수정 동작을 담는다. 화면이 아니다.

**A·A-빈 상태·A-그룹 내부의 관계:** 셋 다 같은 종류의 화면(캔버스 그리드)이고 담는 내용만 다르다. A의 그룹 폴더 카드를 클릭 → A-그룹 내부로 들어가고, 브레드크럼으로 A 복귀한다.

**별도 화면이 아니라 위에 겹쳐 뜨는 것들 (A 쪽):**

- **새 캔버스 만들기 모달** / **그룹 만들기 동작**(만드는 인터랙션은 §4 미정) / **캔버스 카드 더보기(⋮) 메뉴** — **복사하기 / 보관 / 삭제** 세 항목이 모든 캔버스 카드에 상시. '복사하기' = 통째 복제(F2). **'보관'으로 보낸 캔버스는 명시 삭제 전까지 사라지지 않는다(v10·OQ-15).**

> **상태와 필터.** 캔버스 상태 enum = **진행 중 / 잠금 / 보관** 셋(데이터). 목록 상단 상태 칩 = 전체 / 진행 중 / 잠금 세 보기(`t-a-list-v2.html`). 따라서 **(캔버스)잠금 = 카드 위 상태 배지 + 목록 필터 칩 둘 다.** **'보관'이 칩 줄에 끼는지 별도 보기로 빠지는지는 아직 미정(§4)** — 다만 보관해도 데이터가 사라지지 않는다는 점은 v10에서 확정됐다.

### 학생 화면 (4개 화면 + E의 권한 변형 2종)

> 학생 area는 아직 HTML 미작업이다(E-참여자 `s-e-participant.html`만 초기본). v10에서는 **정의·흐름 수준만** 건드린다 — 새로 그리지 않는다. 바뀐 곳: E-참여자의 공감 = '포스트잇 1인 1표', E-참여자의 객체 잠금(본인 객체), E-뷰어 공감 불가 확정. 화면 개수·코드는 그대로다.

| 코드 | 화면 | 무엇을 보여주나 | 무엇을 하나 | PRD 기능 |
|---|---|---|---|---|
| **D** | 입장 화면 | 캔버스 이름. **들어온 QR의 권한에 따라 입장 방식이 갈림** | 권한별 분기에 따라 로그인 또는 닉네임+4글자 비번 입력 → 입장 시도. **닉네임 중복이면 안내·차단**(다른 닉네임 유도, OQ-11) | F5 |
| **D-승인 대기** | 입장 후 승인 대기 (v9, D의 변형) | "선생님 승인을 기다리는 중" 안내 | 없음(대기). 교사가 B에서 승인하면 E로, 거절·정원초과·잠금이면 F로. **자동 승인이 켜져 있으면 이 단계를 건너뛴다** | F5 |
| **E** | 캔버스 편집 | 편집자가 짜둔 페이지, 하단 도구바, 잔여 시간, presence | **권한 변형 2종**(아래) | F4·F10·F12 |
| **F** | 안내 화면 | 상황 메시지 | 없음(읽고 대기) — 정원 초과 / 잠금(보기 전용) / 보관 / **입장 거절** 변형들 | F5·F8 |
| **G** | 내 활동 (로그인 시) | 참여했던 캔버스 목록 | **재진입 — 그때의 권한대로 E-참여자 또는 E-뷰어로** | F5 |

**E의 권한 변형 2종(v10 객체 권한·공감 반영):**

- **E-참여자** — 객체 작업 + 공감. 객체를 옮기고·고치고·잠그는 것은 **본인이 만든 객체에만** 가능(F12). 남의 객체는 보고 **공감만** 할 수 있지 옮기거나 고칠 수 없고, **잠긴 객체는 작성자 본인 외에는 못 건드린다**(v10). **공감은 포스트잇에만, 1인 1표(다시 누르면 취소)**다. (`s-e-participant.html` 서술에 '본인 객체만 편집·잠금 / 공감=포스트잇 1인 1표' 반영.)
- **E-뷰어** — 보기 전용(새 화면 아니라 E의 변형). 객체 생성·이동·편집 전부 불가, 페이지 변경 불가, **공감 투표 불가(v10 확정, OQ-21)**. 실시간 반영을 보기만 한다.

> 참고: 소유자·편집자는 **모든 객체**를 컨트롤·잠금(B). 참여자는 **본인 객체만**, 뷰어는 **불가**. 이 권한 3단계가 E-참여자/E-뷰어/B를 가르는 핵심 축이다. 공감은 **참여자만**(포스트잇·1인 1표), 뷰어는 못 한다.

**D의 권한 분기:** 편집자 QR=로그인 필수→B / 참여자 QR=로그인 or 닉네임+비번→(승인 대기)→E-참여자 / 뷰어 QR=로그인 or 닉네임+비번→(승인 대기)→E-뷰어. 정원 20명은 편집자·참여자·뷰어 모두 포함, 초과 시 F.

두 area의 연결고리는 **B의 공유 패널에서 발급한 권한별 QR**이다. **편집자 QR은 D를 거쳐 다시 교사 area(B)로 합류**(QR은 10분 만료)하고, 참여자·뷰어 QR은 학생 area(E)에 머문다. 참여자·뷰어는 D 다음 '승인 대기'를 거쳐 E로 들어가고(자동 승인이면 건너뜀). G(내 활동)는 학생 area 안의 재진입 통로다.

> **v10에서 달라진 곳 정리.** ① B 포스트잇 공감과 E 공감이 **하나의 표현(포스트잇·1인 1표·토글)**으로 합쳐졌다. ② 객체 클릭 시 뜨는 것이 '팝오버'에서 **'수정 옵션 바 + 우측 작성자 프로필'**로 확정됐고, ③ 그 바 안에 **객체 잠금**(본인/모든 객체 권한별)이 새로 들어갔다. ④ E-뷰어 **공감 불가** 확정. ⑤ 카드 ⋮ '보관' = **영구 보존**(자동 삭제 없음), 편집자 QR = **10분 만료**.

> 맨 위에 '프로젝트'가 한 층 더 있지만 PoC에선 1개만 쓰므로 그 화면은 그리지 않는다. A→A-그룹 내부는 '한 층 내려가기'가 아니라 '한 폴더를 펼쳐 보기'다.

## 3. 핵심 흐름

**① 교사 첫 사용 (실측 2액션)**
```
퀴즈앤 링크 클릭 → A + 새 캔버스 모달 자동 → [템플릿 선택] → B(캔버스) → [공유 버튼] → 권한 고르고 QR
```
PRD의 "3액션·30초" 기준 충족(2액션 = 템플릿 선택 1 + 공유 1). 권한 기본값 '참여자'가 미리 선택돼 별도 액션으로 세지 않는다(§4). 캔버스 0개면 A 대신 A-빈 상태가 뜨지만 모달이 자동으로 떠 같은 2액션이 된다.

**② 교사 재사용 — 새로 만들기 / 복사 두 갈래**
```
(새로)  A → [+ 캔버스 생성] / '새 캔버스' 타일 → 새 캔버스 모달(빈/3종) → B → 공유(권한)→QR
(복사)  A → 캔버스 카드 [⋮ → 복사하기] → 복사본(B) → 공유(권한)→QR
```
잘 만든 캔버스 하나를 1반·2반·3반용으로 통째 복사한다(F2). 복사하기는 잠금·진행 중 가리지 않고 모든 카드 ⋮ 메뉴에 있다. 묶어 두고 싶으면 A에서 그룹으로 묶을 수 있다(선택).

**③ 교사 — 그룹으로 정리하고 그 폴더를 펼쳐 작업**
```
A → [그룹 만들기] / '새 그룹' 타일 → 캔버스 담기 → 그룹 폴더 카드 생성
A → 그룹 폴더 [클릭] → A-그룹 내부 → 캔버스 [카드 클릭 = 드릴다운]→B / [+ 이 그룹에 캔버스]→새 캔버스 모달
A-그룹 내부 → [← 캔버스] 브레드크럼 → A 복귀
```
캔버스를 폴더로 묶고 폴더를 눌러 그 그룹만 모아 본다. 그룹은 정리용 폴더라 안 묶어도 캔버스는 A에 그냥 있다. 이 그룹 단위가 곧 지휘 범위가 된다(아래 ⑥, OQ-20 해소).

**④ 교사 — 캔버스 운영·지휘 (B 안에서) — v10: 객체 인터랙션 또렷해짐**
```
A(또는 A-그룹 내부) → 캔버스 [카드 클릭 = 드릴다운] → B (자기 화면에 크게 열림)
B → 타이머 ▾ (정지 / 리셋 / +5·+10·+30·+60분)        ← 활동 시간 조절
B → 페이지 레일 [+ 페이지 추가] → 새 캔버스 모달(페이지별 템플릿) → 새 페이지 (최대 30)
B → 도구 팔레트로 객체 만들기·편집 (소유자·편집자는 모든 객체 컨트롤·잠금)
B → 객체 [클릭] → 수정 옵션 바(객체 위) → 우측 작성자 프로필로 작성자 확인 / 바 안에서 객체 잠금  ← v10
B → 포스트잇 [♥] 공감 (포스트잇 한정·1인 1표·토글, E와 동일)
B → 참여자 커서 끄기 토글 / 참여자 입장 승인 (승인/거절 / 모두 승인 / 자동 승인)
B → [캔버스 잠금]  ← 캔버스 전체 멈춤(상태 '잠금'). '보기만'은 잠근 채 드릴다운
B → [공유] → 권한 탭 → 그 권한 QR (편집자 탭 = 임명+초대 한곳, 편집자 QR 10분 만료)
B → [설정] → 드롭다운(익명 토글 / 닉네임+비번4 안내 / 캔버스 삭제)   ← 팝오버
```
편집자가 수업 전 페이지를 짜고, 수업 중 타이머·잠금·승인·커서로 운영하며, 공유 패널로 권한별 QR을 띄운다. **v10에서 객체를 클릭하면 수정 옵션 바가 뜨고(우측 = 작성자 프로필), 그 바에서 객체 하나를 잠가 둘 수 있다**(예: 발표 기준이 되는 포스트잇을 학생이 못 건드리게). 도구 팔레트 목록은 예시(§4).

**⑤ 학생 입장 — 권한별 3갈래 + 승인 대기 (v10: 공감·잠금 반영)**
```
(참여자) 참여자 QR/URL → D(로그인 or 닉네임+4글자 비번 · 닉네임 중복 차단) → D-승인 대기 →[교사 승인]→ E-참여자
          · 본인 객체만 옮기고·고치고·잠금. 남의 객체는 보고 공감만. 잠긴 객체는 작성자 외 못 건드림.
          · 공감 = 포스트잇에만, 1인 1표(다시 누르면 취소)
(뷰어)   뷰어   QR/URL → D(로그인 or 닉네임+4글자 비번 · 닉네임 중복 차단) → D-승인 대기 →[교사 승인]→ E-뷰어(보기 전용 · 공감 불가)
(편집자) 편집자 QR/URL → D(로그인 필수) → B(편집자로 합류, 모든 객체 컨트롤·잠금)
         · 자동 승인이 켜져 있으면 참여자·뷰어는 'D-승인 대기'를 건너뛰고 바로 E
         · 정원 초과·잠금·보관·입장 거절 → F 안내 화면
```
참여자·뷰어는 무설치·무가입, 로딩 3초 이내 목표. 닉네임 중복이면 D에서 차단(OQ-11). 기본은 교사 수동 승인이라 D 다음 '승인 대기'를 거친다(자동 승인이면 건너뜀). 정원 20명은 편집자·참여자·뷰어 합산. **E-참여자는 본인 객체만 옮기고·고치고·잠그며, 공감은 포스트잇에 1인 1표. E-뷰어는 보기만 하고 공감도 못 한다(v10).**

**⑥ 교사 지휘 (수업 중) — 진입점 갭 해소(v9 유지)**
```
A(또는 A-그룹 내부) 라이브 썸네일 → 막힌 모둠 파악 → 캔버스 [카드 클릭 = 드릴다운]→B → 타이머·잠금·승인·커서·객체 잠금 제어
```
캔버스 6개를 한 화면에서 스크롤 없이 파악(F7 품질 기준)하고 막힌 모둠을 돕는다. 카드 클릭으로 캔버스를 크게 여는 것(드릴다운)이 곧 편집 화면(B) 진입이며 둘은 같은 동작이다. '끼어들지 않고 보기만'은 캔버스를 잠근 채(F8) 드릴다운한다. 지휘 대상 범위(OQ-20)는 보던 목록의 범위로 결정된다(A 전체=프로젝트 전체 / A-그룹 내부=그 그룹). **v10에서 교사는 B 안에서 개별 객체를 잠가(객체 잠금) 특정 객체를 고정해 둘 수도 있다.**

## 4. 아직 안 정한 것 (그림 그리며 판단)

이번 v10에서 닫힌 것 / 새로 또렷해진 것 먼저 정리한다.

**v10에서 해소·확정된 것 (더 이상 미정 아님):**

- **OQ-16 공감 투표 세부** — 공감은 **포스트잇에만, 1인 1표(취소=토글)**. B 포스트잇 공감과 E 참여자 공감은 **같은 표현·카운트·규칙**으로 통일. (v9에 남았던 'B 감정표현 ↔ E 공감 통일 여부' 닫힘.)
- **OQ-18 작성자 표기 형태** — 객체 클릭 시 **수정 옵션 바 + 그 바 우측의 작성자 프로필**로 확정. (v9의 '작성자·수정자 팝오버 형태 미정' 닫힘.)
- **OQ-21 뷰어 공감** — 뷰어 공감 **불가** 확정.
- **OQ-15 보관 캔버스 보존** — 보관해도 명시 삭제 전까지 **영구 보존**(자동 삭제 없음). (단, '보관' 상태 칩의 자리·표시 위치는 아래에 미정으로 남음.)
- **OQ-17 편집자 QR 만료** — 편집자 QR 초대 링크 **10분 만료** + 임명 해제 있음(공유 패널 '편집자' 탭).

**아직 남은 열린 점:**

- **객체 잠금의 표면 세부(v10 신규 미정).** 수정 옵션 바 안에서 잠금 동작을 어떤 토글/버튼으로 보일지, 잠긴 객체의 시각 표시(자물쇠 배지 등) 형태, '캔버스 잠금'과 라벨·아이콘이 헷갈리지 않게 하는 표현 — 와이어프레임에서 확정. IA는 'B·E의 수정 옵션 바에 객체 잠금이 있고, 잠긴 객체엔 시각 표시가 있다'까지 확정.
- **수정 옵션 바의 항목 구성.** 우측 작성자 프로필·객체 잠금 외에 어떤 수정 동작(복제·삭제·앞뒤 정렬 등)을 이 바에 담을지, 권한(참여자=본인 객체만)에 따라 바의 항목이 어떻게 줄어드는지 — 와이어프레임에서.
- **A 상태 칩 '잠금'과 '보관'의 자리.** 칩을 전체 / 진행 중 / 잠금으로 그렸다. '보관'이 칩 줄에 끼는지 별도 보기로 빠지는지, 보관 캔버스가 같은 그리드에 섞이는지 미정. (보관해도 사라지지 않는다는 점만 v10 확정.)
- **새 캔버스 모달의 '페이지 추가' 결과 표현.** A에서 띄우면 새 캔버스, B [+ 페이지 추가]에서 띄우면 새 페이지 — 카피·헤더 분기 여부 와이어프레임에서.
- **B 하단 도구 팔레트의 도구 목록.** 와이어프레임 항목은 예시(F4 객체 종류와 맞춤).
- **공유 패널의 권한 선택 UX.** 탭 전환식(참여자 기본)으로 그렸다 — 2액션 유지할지, 세 QR 동시 표시안을 둘지. 편집자 탭에 임명+초대+10분 만료 안내가 모여 무거워지지 않는지도 점검.
- **타이머 제어의 표면 분리.** 일시정지(⏸)는 노출, 정지·리셋·시간 조정은 드롭다운(▾). 일시정지/정지 의미 차이와 캔버스 잠금 버튼과의 관계 정리.
- **그룹 만들기·담기 인터랙션.** 폴더가 네비 대상인 건 확정. 만들고 담는 동작(드래그? 다중 선택 후 '그룹으로'?)은 와이어프레임에서.
- **그룹 폴더 카드의 라이브성.** 미니 썸네일 2×2를 라이브 갱신할지 정적으로 둘지.
- **참여자 커서 끄기 토글의 위치 / 입장 승인 UI의 표면.** B 상단바인지 도구 영역인지 / 상단바 배지·presence 영역·토스트 중 어디인지. IA는 'B에 있다'까지.

**상위 문서(PRD)와 함께 보는 잔여 OQ:**

- **객체 동시 조작 충돌 처리 (PRD 실제 OQ-13, 열림 — 닫지 않음).** '같은 객체를 두 사람이 같은 순간 다르게 바꿀 때 무엇을 최종으로 삼나'의 문제. **v10의 객체 잠금은 이를 부분적으로 완화한다** — 누군가 잠가 둔 객체는 다른 사람이 못 바꾸니 그 객체에서는 충돌이 안 난다. 하지만 **잠그지 않은 객체를 편집자↔편집자·편집자↔작성자(학생)가 동시에 건드릴 때의 충돌은 그대로 남으므로, OQ-13 자체는 계속 열어둔다**(마지막 입력 우선 등 실시간 동기화 정책은 구현 영역, PoC 제품 결정으로 닫지 않음 — 2026-06-17 사용자 결정 유지). 화면에는 영향 없음.
- **프로젝트당·그룹당 캔버스 수 상한 (OQ-19, 열림).** 임의 고정값 두지 않음. 복제본 자동 그룹 묶임 여부도 미정 → 수동 가정. (캔버스당 30페이지·진입 20명은 이미 고정.)
- **템플릿 선택지 개수 (OQ-03).** 빈 캔버스 기본 확정. 최종 개수는 와이어프레임에서(현재 빈 캔버스 + 브레인스토밍 + 마인드맵 3종).
- **참여자 로그인 계정 체계 (OQ-12, 보류).** 퀴즈앤 SSO 고려 중. G(내 활동) 재진입의 전제라 G 상세에 영향 가능.
- **OQ-02 등 산정/규모 항목.** 전사 동시 캔버스 수 등은 화면 그리기를 막지 않음.
- **비밀번호 분실·재입력 흐름(OQ-11 잔여) / F 정원 초과 문구** — 학생 area HTML 작업 시 확정.

## 5. PRD와의 정합

> **이번 v10의 결정은 전부 prd-v11 범위 안이고(객체 잠금만 prd-v12 확정 델타), 상위로 돌려보낼 갭은 없다.**
> - **공감 = 포스트잇·1인 1표(OQ-16)**: prd-v11 F10이 '포스트잇 공감, 공감 남김·취소'를 규정하고 OQ-16에 '1인당 투표 횟수 등 세부'를 남겨 둔 것을 닫았다 — B·E 공감을 한 표현으로 통일한 건 화면 결정이지 새 스펙이 아니다.
> - **작성자 = 수정 옵션 바 + 우측 작성자 프로필(OQ-18)**: prd-v11 §1 가시성 규칙·F12·F13(객체 클릭 시 작성자 확인)이 규정한 식별 동작의 **표면 형태**를 정한 것. 기능 정의는 그대로다.
> - **객체 잠금(신규)**: prd-v11에서는 F13 '하지 않는 것'에 '객체 단위 잠금'이 OQ-14 잔여 검토 대상으로, §7 비목표 14/16 근처에 v2 후보로 적혀 있었다. **prd-v12가 이를 확정 기능으로 끌어올리는 중**(병렬 작성)이며, 권한 축은 F12와 정합(소유자·편집자=전체 / 참여자=본인 객체). IA는 그 확정 델타를 화면(B·E 수정 옵션 바)에 옮긴 것 — IA가 새 제품 결정을 만든 게 아니다.
> - **뷰어 공감 불가(OQ-21)**: prd-v11 F10·E 뷰어 서술의 [잠정 — 기본 불가]를 확정으로 닫았다.
> - **보관 영구 보존(OQ-15) · 편집자 QR 10분 만료(OQ-17)**: 각각 캔버스 보관 정책·공유 링크 정책의 스펙 확정. 화면에는 A 한 줄·공유 패널 한 줄로만 닿는다.
>
> **남는 OQ는 와이어프레임 진행을 막지 않는다.** 특히 **PRD 실제 OQ-13(동시 조작 충돌)은 객체 잠금으로 부분 완화될 뿐 계속 열려 있고**(화면 무관), OQ-12(계정 체계)는 보류로 G 상세의 전제다.

## 부록. 흐름도 상태

> **FigJam 흐름도 갱신 필요(미반영 — 이번에는 그리지 않음).** 이 v10도 흐름도는 그리지 않았다. 기존 흐름도(`ux/ia/ia-flowchart-v5.png`, fileKey `OT6YlTAIzTNYMrf0m9yRoo`)는 v5 구조라 v6~v10과 어긋난다. 다음에 인증된 세션에서 갱신할 때 반영할 변경점:
>
> **(v6~v9 누적 미반영분 — 여전히 필요)**
> - 교사 area에 **A-그룹 내부** 노드 + 그룹 폴더→A-그룹 내부 엣지 + 브레드크럼 복귀 양방향 엣지.
> - **'T 템플릿 선택' 노드를 없애고 '새 캔버스 모달' 오버레이 노드(육각형)로 통합** — A·A-빈 상태·A-그룹 내부·B(페이지 추가)에서 이 모달로 향하는 엣지. 자동 진입은 점선.
> - **A-빈 상태 노드**(stadium) 추가. **C(캔버스 설정) 노드 삭제**(B 설정 드롭다운으로 흡수).
> - 카드 ⋮ 분기 = 복사하기 / 보관 / 삭제, A 상태 칩 = 전체 / 진행 중 / 잠금. **카드 엣지 라벨 = '카드 클릭 = 드릴다운 = B 편집'**(드릴다운 진입점 미정 주석은 삭제).
> - 편집자 임명을 공유 패널 '편집자' 탭으로(임명·임명 해제 + QR 초대 한 노드).
> - **B 내부 분기**에 커서 끄기 토글·참여자 입장 승인 추가.
> - 학생 area 입장 동선에 **'D-승인 대기' 노드** + 자동 승인 점선 + 입장 거절·정원초과·잠금·보관→F. D 노드에 '닉네임 중복 차단' 주석.
>
> **(v10 신규 반영분 — 다음 갱신 때 추가)**
> - **B 객체 클릭 엣지를 '수정 옵션 바(우측 작성자 프로필)'로 표기** — v9 부록의 '객체 클릭 → 작성자·수정자 팝오버' 표기를 이 형태로 교체. 그 바 안에 **객체 잠금** 항목 주석.
> - **포스트잇 공감을 B·E 공통 노드/라벨로 통일** — '공감 ♥ = 포스트잇 한정·1인 1표·토글'. B의 '감정표현 ☺'과 E의 '공감'을 별개로 그렸다면 하나로 합친다.
> - **편집자 탭 노드에 'QR 10분 만료' 주석**, 카드 ⋮ '보관'에 '영구 보존' 주석.
> - **E-뷰어 노드에서 공감 엣지 제거**(뷰어 공감 불가 확정) — E-참여자에만 공감 엣지.
> - **'캔버스 잠금'(상단바, 캔버스 전체)과 '객체 잠금'(수정 옵션 바, 객체 하나)을 라벨로 명확히 구분** — 같은 '잠금'이라도 대상이 다름을 흐름도에서 헷갈리지 않게.
>
> **(학생 area)** 나머지 학생 area는 HTML 미작업이므로 흐름도에서도 정의/흐름 수준으로만 둔다.
