> Project: `quizn-board-v2` / Date: 2026-06-15 / Version: v2

> **v1→v2 바뀐 점(한 줄 요약):** ① 스포트라이트(전체 공유) 화면 삭제 — 교사가 자기 화면에서 캔버스를 크게 보는 드릴다운으로 충분. ② 캔버스 '생성' 동선 추가 — 목록에 [+ 캔버스 생성] 버튼 → 템플릿 선택 화면(빈 캔버스 + 3종) → 캔버스. 링크 진입 시 자동으로 뜨는 모달도 같은 선택 화면. ③ 계층 변경 — 프로젝트와 캔버스를 다시 분리(프로젝트 > 그룹 > 캔버스 > 페이지 > 객체), landing 화면 = 그 프로젝트의 **캔버스 목록(=지휘석)**.

## 0. 이 문서가 무엇인가

이 제품에 **화면이 몇 개 있고, 각 화면이 무엇을 보여주고, 화면끼리 어떻게 이어지는지**를 적은 설계도다. 실제 그림(와이어프레임)을 그리기 전에, 그릴 대상의 목록과 연결을 먼저 확정하는 단계다.

전체는 두 세계로 나뉜다 — **교사가 쓰는 화면**(데스크톱)과 **학생이 쓰는 화면**(태블릿·폰). 둘이 만나는 유일한 접점은 캔버스다.

- 교사 화면 기준: 데스크톱 1440px
- 학생 화면 기준: 태블릿 1024px (+ 폰 390px 변형)

### 이번에 바뀐 계층부터 한 줄씩

먼저 용어를 정의한다. 위에서 아래로 담는 관계다.

- **프로젝트** = 가장 바깥 컨테이너. 한 학기·한 과목처럼 캔버스들을 담는 가장 큰 통이다. PoC에서는 사실상 **프로젝트 1개만** 쓴다(교사 한 명에 통 하나). 여러 프로젝트를 오가는 화면은 PoC 대상이 아니므로 화면으로 그리지 않고, 개념적으로만 맨 위에 있다고 둔다.
- **그룹**(선택) = 프로젝트 안에서 **캔버스들을 묶는 단위**다. 예: "1단원 활동", "2026 1학기 1반들". v1에서 그룹은 '프로젝트를 담는 폴더'였는데, v2에서는 **프로젝트 안에서 캔버스끼리 묶는 묶음**으로 위치가 바뀌었다. 없어도 된다(선택).
- **캔버스** = 학생이 들어오는 진입 단위. **QR 1개**가 붙고, **진입자 최대 20명**, 여러 **페이지**로 구성된다. 예: "3반 브레인스토밍". 잘 만든 캔버스 하나를 1반·2반·3반용으로 **복제**한다.
- **페이지** = 캔버스의 작업 면. 편집자가 템플릿을 붙여 구성한다(예: 10페이지).
- **객체** = 페이지 위에 만드는 것(텍스트·그리기·포스트잇·도형).

```
프로젝트 (최상위 통 — PoC에선 사실상 1개. 다중 프로젝트 화면은 PoC 비대상, 개념상으로만 위에 존재)
 └ 그룹 (선택 — 프로젝트 안에서 캔버스끼리 묶는 단위)
    └ 캔버스 (QR 1개 · 학생 진입 단위 · 진입자 20명 · 여러 페이지)
       └ 페이지 (편집자가 템플릿을 붙여 구성)
          └ 객체 (텍스트·그리기·포스트잇·도형)
```

> 이 계층은 prd-v5 §2의 "프로젝트=캔버스" 통합을 되돌린 것이다(이번 개정의 권위 있는 스펙). PRD는 @prd-writer가 v6에서 같은 방향으로 정리 중이다 — 어긋나는 부분은 §4 '아직 안 정한 것'에 표시한다.

## 1. 화면 목록

### 교사 화면 (4개)

| 코드 | 화면 | 무엇을 보여주나 | 무엇을 하나 | PRD 기능 |
|---|---|---|---|---|
| **A** | 캔버스 목록 (= 지휘석) | 이 프로젝트의 캔버스 카드들, 그룹으로 묶여 정리됨. 카드마다 참여 인원·객체 수 | **[+ 캔버스 생성]**, 복제·삭제, 그룹으로 묶기, 캔버스 열기. **수업 중엔 카드가 2초마다 갱신되는 라이브 썸네일**이 되어 지휘석 역할 | F1·F2·F7·F11 |
| **T** | 템플릿 선택 | 선택지 4장 — **빈 캔버스 / 브레인스토밍 / 마인드맵 / (예비)**, 각 미리보기 | 하나 고르면 그 템플릿이 깔린 새 캔버스가 만들어지고 즉시 B로 진입 | F3·F1 |
| **B** | 캔버스 편집 (편집자 뷰) | 페이지(좌우로 넘김), 도구바, 잔여 시간, 객체 작성자 | 객체 만들기·편집, **페이지 추가·템플릿 설정**, QR 띄우기, 타이머·잠금 | F4·F3·F8 |
| **C** | 캔버스 설정 | 편집자 목록, 익명/닉네임·비번 옵션 상태 | 편집자 임명(캔버스 단위), 옵션 on/off | F6 |

**별도 화면이 아니라 위에 겹쳐 뜨는 것들:**
- **템플릿 선택 모달** — A에서 [+ 캔버스 생성]을 누르거나, 퀴즈앤 링크로 처음 진입하면 자동으로 뜬다. 내용은 위 **T 화면과 같다**(빈 캔버스 + 템플릿 3종). 즉 수동 생성과 자동 진입이 **같은 선택 흐름**을 쓴다. → 그래서 목록에는 'T'로 한 번만 올리고, 자동/수동은 진입 경로 차이로만 본다.
- **QR 표시 패널** — B 안에서 [QR] 버튼으로 띄움 (교실 프로젝터용 큰 화면)
- **드릴다운** — A(지휘석)의 동작. 한 캔버스 카드를 골라 그 캔버스로 들어가거나(=B), 카드를 크게 확대해 자기 화면에서 본다. **(v1의 '스포트라이트=전체 공유 화면'은 삭제됨 — 별도 화면·기능 아님)**

### 학생 화면 (4개)

| 코드 | 화면 | 무엇을 보여주나 | 무엇을 하나 | PRD 기능 |
|---|---|---|---|---|
| **D** | 입장 화면 | 캔버스 이름, 닉네임 입력란(옵션 시 비번란) | 닉네임 입력 → 입장 | F5 |
| **E** | 캔버스 편집 (참여자 뷰) | 편집자가 짜둔 페이지, 하단 도구바, 잔여 시간, 누가 들어왔나(presence) | 객체 만들기·편집, 공감 투표. **페이지 추가·템플릿 수정은 불가** | F4·F10 |
| **F** | 안내 화면 | 상황 메시지 | 없음(읽고 대기) — 정원 초과 / 잠금(보기 전용) / 보관, 3종 변형 | F5·F8 |
| **G** | 내 활동 (로그인 시) | 참여했던 캔버스 목록 | 재진입 | F5 |

> 교사가 캔버스에 직접 들어간 모습은 별도 화면이 아니라 **B(편집자 뷰)** 그 자체다 — 학생의 E에 타이머·잠금·페이지 편집 권한이 더해진 것.

## 2. 화면 연결 (네비게이션)

```
교사 세계 (데스크톱)
  A 캔버스 목록(=지휘석)            ← 이 프로젝트의 landing/return 화면
   ├─[+ 캔버스 생성] ───────────────▶ T 템플릿 선택 ──[고름]──▶ B 캔버스 편집
   ├─(자동) 템플릿 선택 모달 ────────▶ (= T 내용) ──[고름]──▶ B 캔버스 편집   ← 링크 첫 진입 시
   ├─[복제] ────────────────────────▶ 캔버스 복제본 = 또 다른 B (QR 각자)
   ├─(드릴다운) ─────────────────────▶ B 캔버스 편집 / 카드 확대(자기 화면)
   └─[설정] ─────────────────────────▶ C 캔버스 설정

  B 캔버스 편집
   └─[QR] ─── QR 표시 패널 ─── (학생에게 전달)

학생 세계 (태블릿·폰)
  QR/URL ──▶ D 입장 ──▶ E 캔버스 편집
                         └─(정원초과·잠금·보관) ──▶ F 안내 화면
  로그인 학생: G 내 활동 ──▶ E 재진입
```

두 세계의 연결고리는 단 하나 — **B에서 띄운 캔버스 QR이 학생의 D(입장)로 이어진다.**

> 맨 위에 '프로젝트'가 한 층 더 있지만(여러 프로젝트 전환·목록), PoC에선 프로젝트 1개만 쓰므로 그 화면은 그리지 않는다. A(캔버스 목록)가 곧 그 단일 프로젝트의 안쪽이다.

## 3. 핵심 흐름

**① 교사 첫 사용 (실측 2액션)**
```
퀴즈앤 링크 클릭 → A(캔버스 목록)+템플릿 선택 모달 자동 → [템플릿 선택] → B(캔버스) → [QR 버튼] → QR
```
PRD의 "3액션·30초" 기준 충족(2액션). 첫 진입 시 빈 목록 위에 템플릿 선택이 바로 떠서, 고르는 순간 캔버스로 들어간다. 빈 캔버스도 선택지에 포함된다.

**② 교사 재사용 — 새로 만들기 / 복제 두 갈래**
```
(새로)  A → [+ 캔버스 생성] → T(빈 캔버스/3종 중) → B → QR
(복제)  A → [캔버스 복제] → 복제본(B) → QR
```
잘 만든 캔버스 하나를 1반·2반·3반용으로 통째 복제한다. 복제본은 각각 독립이고 QR도 따로. (v1은 '프로젝트 복제'였으나 계층 변경으로 복제 단위가 **캔버스**로 내려옴 — §4 참고.)

**③ 학생 입장**
```
QR 스캔 / URL → D(입장: 닉네임) → E(편집)
```
무설치·무가입. 옵션이 켜져 있으면 비번도 입력. 로딩 3초 이내 목표.

**④ 교사 지휘 (수업 중)**
```
A(=지휘석) 라이브 썸네일 → 드릴다운(→B) / 카드 확대 / 타이머 제어
```
캔버스 6개를 한 화면에서 스크롤 없이 파악. 막힌 모둠을 돕고, 한 캔버스를 골라 자기 화면에서 크게 본다. (전체 공유 화면=스포트라이트는 삭제 — 큰 화면 공유가 필요하면 교사가 자기 화면을 프로젝터에 그대로 띄우면 된다.)

## 4. 아직 안 정한 것 (그림 그리며 판단)

- **복제 단위가 캔버스로 내려옴.** 1반·2반·3반 = 캔버스 복제. 흐름도엔 '캔버스 복제'로 그린다. 복제하면 자동으로 한 **그룹**으로 묶일지(예: "3반들"), 교사가 수동으로 묶을지는 미정 → 일단 **수동** 가정. (PRD가 v6에서 병렬 정리 중 — 어긋나면 open point.)
- **지휘석 범위 — 프로젝트 안 전체 캔버스를 보나, 한 그룹만 보나?** 캔버스 6개 한 화면 기준이면 보통 한 그룹(한 활동의 복제본들) 단위로 모아 보는 게 자연스럽다. → 여전히 open point. FigJam에서 눈으로 판단.
- **템플릿 선택을 모달(T)로 띄우나, 별도 화면으로 가나?** 자동 진입은 모달이 자연스럽고, [+ 캔버스 생성] 수동 생성도 같은 모달로 통일하는 것을 기본으로 둔다. 와이어프레임에서 확정.
- **T 화면의 선택지 개수.** PRD는 빈 캔버스 + 3종(빈/브레인스토밍/마인드맵)인데 OQ-03에서 3종 자체가 검증 중. 흐름도엔 '빈 캔버스 + 템플릿들'로 두고 개수는 와이어프레임에서.
- **PRD 계층과의 정합.** 이번 v2는 "프로젝트와 캔버스 분리"를 권위 스펙으로 삼았다. prd-v5는 아직 "프로젝트=캔버스"이므로, PRD v6 확정 전까지는 이 IA가 앞서 있는 상태다. → @prd-writer 정리 결과와 맞춰 재확인 필요.
- (PRD 잔여 OQ는 prd-v5 §9 참조 — OQ-02 검증규모·OQ-03 템플릿·OQ-19 복제 규모 등. 와이어프레임 진행에는 막힘 없음.)
