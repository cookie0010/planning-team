## 0. 이 문서가 무엇인가

이 제품에 **화면이 몇 개 있고, 각 화면이 무엇을 보여주고, 화면끼리 어떻게 이어지는지**를 적은 설계도다. 실제 그림(와이어프레임)을 그리기 전에, 그릴 대상의 목록과 연결을 먼저 확정하는 단계다.

전체는 두 area로 나뉜다 — **교사가 쓰는 화면**(데스크톱)과 **학생이 쓰는 화면**(태블릿·폰). 둘이 만나는 유일한 접점은 캔버스다. (이전 버전에서 '두 세계'라고 부르던 것을 v4부터 'area'로 통일한다 — 같은 뜻이다.)

- 교사 화면 기준: 데스크톱 1440px
- 학생 화면 기준: 태블릿 1024px (+ 폰 390px 변형)

### 먼저 짚는 용어: 역할 4단계

화면을 보기 전에 한 줄씩 정의한다. PRD v7 §1을 그대로 따른다.

- **소유자** = 프로젝트를 만든 사람(보통 교사). 로그인 필수. 모든 것을 제어한다.
- **편집자** = 소유자나 다른 편집자가 임명한 사람. 로그인 필수. 임명된 캔버스의 페이지 추가·템플릿 설정·편집이 가능하다.
- **참여자** = 캔버스에 들어와 **함께 만드는** 사람(보통 학생). 무계정 입장 가능. 객체 생성·이동·편집 + 포스트잇 공감 투표.
- **뷰어** = 캔버스에 들어와 **보기만** 하는 사람. 무계정 입장 가능. 객체 편집 불가, 페이지 변경 불가, 실시간 반영을 보기만. 공감 투표 가능 여부는 미정(기본 불가) — PRD OQ-21.

> 화면 수준에서 핵심은: 소유자·편집자는 **교사 area**(데스크톱)에 산다. 참여자·뷰어는 **학생 area**(태블릿·폰)에 산다. 참여자와 뷰어는 **같은 화면(E)**을 쓰되, 뷰어는 그 화면의 '보기 전용' 변형으로 들어온다.

### 계층 (PRD v7 §2 그대로)

위에서 아래로 담는 관계다.

```
프로젝트 (최상위 통 — 소유자가 생성. PoC에선 사실상 1개. 다중 프로젝트 화면은 범위 밖, 개념상으로만 위에 존재)
 └ 그룹 (선택 — 프로젝트 안에서 캔버스끼리 묶는 단위. 예: "1차시", "모둠 발표". 필수가 아니라 안 묶어도 됨)
    └ 캔버스 (QR 1개 · 학생 진입 단위 · 진입자 20명 · 최대 30페이지)
       └ 페이지 (편집자가 템플릿을 붙여 구성 · 최대 30개 · 좌우 추가·이동)
          └ 객체 (텍스트·그리기·포스트잇·도형)
```

> v3·v4와 같은 계층이다. v5는 이 계층을 건드리지 않는다. **진입자 20명에는 참여자와 뷰어가 모두 포함된다**(v4는 뷰어를 이 정원에서 빼두었으나 v5에서 포함으로 정정 — 아래 §1·§4 참조).

## 1. 화면 목록

### 교사 화면 (4개)

| 코드 | 화면 | 무엇을 보여주나 | 무엇을 하나 | PRD 기능 |
|---|---|---|---|---|
| **A** | 캔버스 목록 (= 지휘석) | 이 프로젝트의 캔버스 카드들, 그룹으로 묶여 정리됨. 카드마다 참여 인원·객체 수 | **[+ 캔버스 생성]**, 복제·삭제, **그룹 만들기·묶기(선택)**, 캔버스 열기. **수업 중엔 카드가 2초마다 갱신되는 라이브 썸네일**이 되어 지휘석 역할(드릴다운 + 타이머 제어) | F1·F2·F7·F11 |
| **T** | 템플릿 선택 | 선택지 — **빈 캔버스 + 템플릿 3종**(브레인스토밍·마인드맵·예비), 각 미리보기 | 하나 고르면 그 템플릿이 깔린 새 캔버스가 만들어지고 즉시 B로 진입 | F3·F1 |
| **B** | 캔버스 편집 (편집자 뷰) | 페이지(좌우로 넘김, **N/30 표시**), 도구바, 잔여 시간, 객체 작성자 | 객체 만들기·편집, **페이지 추가·템플릿 설정**(최대 30), QR 공유, 타이머·잠금 | F4·F3·F8 |
| **C** | 캔버스 설정 | 편집자 목록, 익명/닉네임·비번 옵션 상태 | 편집자 임명(캔버스 단위), 옵션 on/off | F6 |

**별도 화면이 아니라 위에 겹쳐 뜨는 것들:**

- **템플릿 선택 모달** — A에서 [+ 캔버스 생성]을 누르거나, 퀴즈앤 링크로 처음 진입하면 자동으로 뜬다. 내용은 위 **T 화면과 같다**. 즉 수동 생성과 자동 진입이 **같은 선택 흐름**을 쓴다. → 목록에는 'T'로 한 번만 올리고, 자동/수동은 진입 경로 차이로만 본다.
- **그룹 만들기·묶기 (선택)** — A 안에서 캔버스 카드를 그룹으로 묶는 동작이다. **필수가 아니라 선택** — 묶지 않아도 캔버스는 그냥 목록에 평평하게 놓인다. 새 화면이 아니라 A 위에서 일어나는 정리 동작(그룹 만들기 → 카드 끌어 담기)이다. 흐름도에는 A의 **부가 동작(점선·"(선택)" 라벨)**으로 표현한다.
- **QR 공유 패널** — B 안에서 [공유] 버튼으로 띄움(교실 프로젝터용 큰 화면). **공유할 권한을 먼저 고른다 — 편집자 / 참여자 / 뷰어.** 고른 권한에 해당하는 **QR·URL을 발급·표시**한다. 권한마다 별개 링크이고, 셋을 동시에 띄워 둘 수도 있다(설계는 와이어프레임에서). 이 권한 선택이 학생 입장 화면(D)의 분기를 결정한다.
- **드릴다운** — A(지휘석)의 동작. 한 캔버스 카드를 골라 그 캔버스로 들어가거나(=B), 카드를 크게 확대해 자기 화면에서 본다. (전체 공유 화면=broadcast는 없음 — PRD 비목표 5)

### 학생 화면 (4개 화면 + E의 권한 변형 2종)

| 코드 | 화면 | 무엇을 보여주나 | 무엇을 하나 | PRD 기능 |
|---|---|---|---|---|
| **D** | 입장 화면 | 캔버스 이름. **들어온 QR의 권한에 따라 입장 방식이 갈림**(아래 분기) | 권한별 분기에 따라 로그인 또는 닉네임+4글자 비번 입력 → 입장 | F5 |
| **E** | 캔버스 편집 | 편집자가 짜둔 페이지, 하단 도구바, 잔여 시간, presence(누가 들어왔나) | **권한 변형 2종**(아래) | F4·F10 |
| **F** | 안내 화면 | 상황 메시지 | 없음(읽고 대기) — 정원 초과 / 잠금(보기 전용) / 보관, 3종 변형 | F5·F8 |
| **G** | 내 활동 (로그인 시) | 참여했던 캔버스 목록 | **재진입 — 그때의 권한대로 E-참여자 또는 E-뷰어로** | F5 |

**E의 권한 변형 2종:**

- **E-참여자** — 객체 생성·이동·편집, 포스트잇 공감 투표. 페이지 추가·템플릿 수정은 불가(편집자가 짜둔 페이지에서 작업).
- **E-뷰어** — **보기 전용.** 같은 캔버스의 편집이 실시간(2초 이내) 반영되는 화면을 **보기만** 한다. 객체 생성·이동·편집 불가, 페이지 변경(추가·이동) 불가. 도구바는 비활성/숨김(와이어프레임에서 확정). 공감 투표 가능 여부는 미정(기본 불가) — PRD OQ-21. → **새 화면이 아니라 E의 권한 변형**이다.

**D 입장 화면의 권한 분기:** 어떤 권한 QR로 들어왔느냐로 갈린다.

| 들어온 QR | 로그인 | 비로그인 입장 | 입장 후 도착 |
|---|---|---|---|
| **편집자 QR** | **필수** | 불가 | 로그인 후 편집자로 → **B(편집자 뷰)** |
| **참여자 QR** | 선택 | **닉네임 + 4글자 비밀번호** | **E-참여자** |
| **뷰어 QR** | 선택 | **닉네임 + 4글자 비밀번호** | **E-뷰어 (보기 전용)** |

> - 편집자 QR로 들어오면 D는 사실상 **로그인 화면**으로 보인다(비로그인 입장 경로 없음).
> - 참여자·뷰어 QR은 입장 방식(로그인 선택 / 닉네임+비번)이 똑같고, **도착하는 E의 변형만 다르다**.
> - 교사가 캔버스에 직접 들어간 모습은 별도 화면이 아니라 **B(편집자 뷰)** 그 자체다 — 학생의 E에 타이머·잠금·페이지 편집 권한이 더해진 것.

> **재진입(G)도 같은 원리다.** 로그인 학생이 G(내 활동)에서 예전 캔버스를 다시 누르면, **그때 들어갔던 권한 그대로** 돌아간다 — 참여자로 참여했으면 E-참여자, 뷰어로 봤으면 E-뷰어. (v4에서는 G가 E-참여자로만 돌아갔는데, v5에서 뷰어 재진입을 추가한다.)

## 2. 화면 연결 (네비게이션)

```
교사 area (데스크톱)
  A 캔버스 목록(=지휘석)            ← 이 프로젝트의 landing/return 화면
   ├─[+ 캔버스 생성] ───────────────▶ T 템플릿 선택 ──[고름]──▶ B 캔버스 편집
   ├─(자동) 템플릿 선택 모달 ────────▶ (= T 내용) ──[고름]──▶ B 캔버스 편집   ← 링크 첫 진입 시
   ├─[복제] ────────────────────────▶ 캔버스 복제본 = 또 다른 B (QR 각자)
   ├┄[그룹 만들기 (선택)] ┄┄┄┄┄┄┄┄┄▶ 카드들을 그룹으로 묶음 (A 위 정리 동작, 새 화면 아님)
   ├─(드릴다운, 한 그룹 단위 기본) ──▶ B 캔버스 편집 / 카드 확대(자기 화면)
   └─[설정] ─────────────────────────▶ C 캔버스 설정

  B 캔버스 편집
   └─[공유] ─── QR 공유 패널 ─── 권한 선택(편집자 / 참여자 / 뷰어)
                                  ├─ 편집자 QR·URL ┐
                                  ├─ 참여자 QR·URL ┤  (권한별 별개 링크, 학생에게 전달)
                                  └─ 뷰어   QR·URL ┘

학생 area (태블릿·폰)
  편집자 QR ──▶ D 입장(로그인 필수) ──[로그인]──▶ B 캔버스 편집(교사 area로 합류)
  참여자 QR ──▶ D 입장(로그인 or 닉네임+4글자 비번) ──▶ E-참여자
  뷰어   QR ──▶ D 입장(로그인 or 닉네임+4글자 비번) ──▶ E-뷰어(보기 전용)
                         └─(정원초과·잠금·보관) ──▶ F 안내 화면   ※ 정원 20명은 참여자+뷰어 모두 포함
  로그인 학생: G 내 활동 ──▶ E-참여자 재진입  (참여자로 참여했던 캔버스)
                        └──▶ E-뷰어   재진입  (뷰어로 봤던 캔버스)
```

(┄ 점선 = 선택 동작 — 그룹은 안 묶어도 됨)

두 area의 연결고리는 **B의 QR 공유 패널에서 발급한 권한별 QR**이다. 그중 **편집자 QR은 D를 거쳐 다시 교사 area(B)로 합류**하고, 참여자·뷰어 QR은 학생 area(E)에 머문다. **G(내 활동)는 학생 area 안에서의 재진입 통로**로, 참여했던 캔버스를 그때 권한(참여자/뷰어)에 맞는 E 변형으로 되돌려 보낸다.

> 맨 위에 '프로젝트'가 한 층 더 있지만(여러 프로젝트 전환·목록), PoC에선 프로젝트 1개만 쓰므로 그 화면은 그리지 않는다. A(캔버스 목록)가 곧 그 단일 프로젝트의 안쪽이다. 그룹은 A 안에서 캔버스를 묶는 선택적 정리 단계로, 별도 화면을 만들지 않는다.

## 3. 핵심 흐름

**① 교사 첫 사용 (실측 2액션)**
```
퀴즈앤 링크 클릭 → A(캔버스 목록)+템플릿 선택 모달 자동 → [템플릿 선택] → B(캔버스) → [공유 버튼] → 권한 고르고 QR
```
PRD의 "3액션·30초" 기준 충족(2액션 = 템플릿 선택 1 + 공유 1). 권한 선택이 QR 패널 안에서 일어나므로 별도 액션으로 세지 않되, 와이어프레임에서 '기본 참여자 권한 미리 선택' 같은 절약을 검토(아래 §4). 그룹 만들기는 선택이라 이 첫 사용 흐름의 액션에 들어가지 않는다.

**② 교사 재사용 — 새로 만들기 / 복제 두 갈래**
```
(새로)  A → [+ 캔버스 생성] → T(빈 캔버스/3종 중) → B → 공유(권한)→QR
(복제)  A → [캔버스 복제] → 복제본(B) → 공유(권한)→QR
```
잘 만든 캔버스 하나를 1반·2반·3반용으로 통째 복제한다. 복제본은 각각 독립이고 QR도 따로. 묶어 두고 싶으면 A에서 **그룹으로 묶을 수 있다(선택)**.

**③ 학생 입장 — 권한별 3갈래**
```
(참여자) 참여자 QR/URL → D(로그인 or 닉네임+4글자 비번) → E-참여자(함께 만들기)
(뷰어)   뷰어   QR/URL → D(로그인 or 닉네임+4글자 비번) → E-뷰어(보기 전용)
(편집자) 편집자 QR/URL → D(로그인 필수) → B(편집자로 합류)
```
참여자·뷰어는 무설치·무가입, 로딩 3초 이내 목표. 편집자만 로그인을 통과해야 한다. **정원 20명은 참여자와 뷰어를 합쳐 센다** — 21번째로 들어오는 사람은 참여자든 뷰어든 F(정원 초과 안내)로 빠진다.

**④ 학생 재진입 (로그인 학생)**
```
G(내 활동) → 참여했던 캔버스 고름 → 그때 권한대로
                                  ├─ 참여자였으면 → E-참여자
                                  └─ 뷰어였으면   → E-뷰어(보기 전용)
```
로그인해 둔 학생은 G에서 예전 캔버스로 곧장 돌아간다. 들어가는 권한은 그때 참여한 권한을 따른다(새 권한을 고르는 게 아니라 기록을 잇는 것). (v5에서 뷰어 재진입을 추가했다.)

**⑤ 교사 지휘 (수업 중)**
```
A(=지휘석) 라이브 썸네일 → 드릴다운(→B, 한 그룹 단위 기본) / 카드 확대 / 타이머 제어
```
캔버스 6개를 한 화면에서 스크롤 없이 파악. 막힌 모둠을 돕고, 한 캔버스를 골라 자기 화면에서 크게 본다. 지휘 대상 범위는 '한 그룹'을 기본으로 두되 확정은 아래 §4(OQ-20). 캔버스를 그룹으로 묶어 두면 이 지휘 단위가 자연스럽게 한 그룹이 된다.

## 4. 아직 안 정한 것 (그림 그리며 판단)

이어지는 열린 점:

- **그룹 만들기·묶기 UX (선택 동작).** 그룹을 어떻게 만들고 카드를 담는지(드래그? 다중 선택 후 '그룹으로'?), 그룹 없는 캔버스는 어떻게 보이는지는 와이어프레임에서. 이번 IA는 'A 안에서 캔버스를 선택적으로 묶는다'까지만 확정. (PRD OQ-19 복제 자동 그룹과 연동.)
- **뷰어 공감 투표 가능 여부 (PRD OQ-21).** 현재 E-뷰어는 '보기만'이 기본이라 투표 불가로 그렸다. 확정되면 E-뷰어 변형에 투표 토글 여부만 반영하면 된다(흐름·화면 수 변화 없음).
- **QR 공유 패널의 권한 선택 UX.** 권한 3종을 탭으로 전환하나, 세 QR을 한 화면에 나란히 보여주나, 첫 사용 때 '참여자'를 기본 선택해 두어 2액션을 지키나 — 와이어프레임에서 확정.
- **D 입장 화면의 분기 표현.** 편집자 QR은 사실상 로그인 화면, 참여자·뷰어 QR은 같은 닉네임+비번 화면(도착 E만 다름). D를 권한별 별도 레이아웃으로 그릴지, 한 화면에서 분기 메시지만 바꿀지는 와이어프레임에서.
- **G(내 활동)의 권한 표시 (v5에서 추가됨).** G의 캔버스 목록에서 각 항목이 '참여자로 참여' / '뷰어로 봄'을 어떻게 구분해 보여줄지(아이콘? 라벨?), 그래서 재진입 시 어느 E 변형으로 갈지를 사용자가 미리 알게 할지는 와이어프레임에서. 흐름·화면 수는 그대로(E의 변형으로 갈 뿐 새 화면 아님).
- **4글자 비밀번호.** 비로그인 식별용 짧은 비번이 4글자로 고정됐다. 중복 닉네임 처리·분실 대응 등 세부는 PRD OQ-11.
- **정원 초과 안내(F)의 문구 (v5에서 변경됨).** 정원 20명에 **참여자와 뷰어가 모두 포함**되므로, F의 '정원 초과' 변형은 참여자·뷰어 둘 다에게 걸린다 — 두 경우 모두 자연스러운 안내 문구를 와이어프레임에서 확정(예: 들어온 권한에 무관하게 "정원이 가득 찼어요"). (v4의 '정원 초과는 참여자 전용' 표기를 뒤집은 것.)

이전부터 이어지는 열린 점:

- **지휘석 범위 (PRD OQ-20).** 프로젝트 안 전체 캔버스를 보나, 한 그룹만 보나? 캔버스 6개 한 화면 기준이면 보통 한 그룹(한 활동의 복제본들) 단위가 자연스럽다. → **'한 그룹 단위'를 기본값으로 깔되 flag.**
- **복제 시 자동 그룹 묶임 여부.** 1반·2반·3반 복제본이 자동으로 한 그룹("3반들")이 될지, 수동으로 묶을지 미정 → 일단 수동 가정. (PRD OQ-19와 연동.)
- **템플릿 선택지 개수.** PRD는 빈 캔버스 + 3종이나 OQ-03에서 3종 자체가 검증 중. 흐름도엔 '빈 캔버스 + 템플릿들'로 두고 개수는 와이어프레임에서.

> **PRD와의 정합 메모:** v3에서 변경 지시를 권위 있는 스펙으로 삼아 OQ-22(뷰어 지정 방식 + 정원 포함)를 반영했고, 그 당시엔 '별도 권한 QR + 뷰어 정원 **제외**'로 그렸다. **v5에서 뷰어를 정원에 포함으로 정정**하면서, 이 IA는 OQ-22의 '정원 포함' 쪽으로 정렬됐다(별도 권한 QR은 유지). 단 PRD v7 §9에는 OQ-22가 아직 blocking으로 남아 있다(@prd-writer가 v8로 병렬 정리 중). **PRD v8 확정 시 뷰어 정원 포함·G 뷰어 재진입이 PRD와 어긋나지 않는지 재확인 필요** — 현재로선 IA가 사용자 직접 수정(FigJam)을 앞서 반영한 상태다.
> (그 외 PRD 잔여 OQ는 prd-v7 §9 참조 — OQ-02 검증규모·OQ-03 템플릿·OQ-11 닉네임/비번·OQ-19 복제 규모·OQ-20 지휘 범위·OQ-21 뷰어 투표 등. 와이어프레임 진행에는 막힘 없음.)
