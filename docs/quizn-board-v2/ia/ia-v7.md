## 0. 이 문서가 무엇인가

이 제품에 **화면이 몇 개 있고, 각 화면이 무엇을 보여주고, 화면끼리 어떻게 이어지는지**를 적은 설계도다. 실제 그림(와이어프레임)을 그리기 전에 그릴 대상의 목록과 연결을 먼저 확정하는 단계다. 이번 버전은 그 와이어프레임을 HTML로 그리다 B(캔버스 편집)에서 또렷해진 구조와 A 카드 인터랙션을 거꾸로 IA에 반영한 것이다.

전체는 두 area로 나뉜다 — **교사가 쓰는 화면**(데스크톱)과 **학생이 쓰는 화면**(태블릿·폰). 둘이 만나는 유일한 접점은 캔버스다.

- 교사 화면 기준: 데스크톱 1440px
- 학생 화면 기준: 태블릿 1024px (+ 폰 390px 변형)

### 먼저 짚는 용어: 역할 4단계

화면을 보기 전에 한 줄씩 정의한다. PRD v9 §1을 그대로 따른다.

- **소유자** = 프로젝트를 만든 사람(보통 교사). 로그인 필수. 모든 것을 제어한다.
- **편집자** = 소유자나 다른 편집자가 임명·초대한 사람. 로그인 필수. 임명된 캔버스의 페이지 추가·템플릿 설정·편집이 가능하다.
- **참여자** = 캔버스에 들어와 **함께 만드는** 사람(보통 학생). 무계정 입장 가능. 객체 생성·이동·편집 + 포스트잇 공감 투표.
- **뷰어** = 캔버스에 들어와 **보기만** 하는 사람. 무계정 입장 가능. 객체 편집 불가, 페이지 변경 불가, 실시간 반영을 보기만. 공감 투표 가능 여부는 미정(기본 불가) — PRD OQ-21.

> 화면 수준에서 핵심은: 소유자·편집자는 **교사 area**(데스크톱)에 산다. 참여자·뷰어는 **학생 area**(태블릿·폰)에 산다. 참여자와 뷰어는 **같은 화면(E)**을 쓰되, 뷰어는 그 화면의 '보기 전용' 변형으로 들어온다.

### 계층 (PRD v9 §2 그대로)

위에서 아래로 담는 관계다. v6과 동일하다.

```
프로젝트 (최상위 통 — 소유자가 생성. PoC에선 사실상 1개. 다중 프로젝트 화면은 범위 밖, 개념상으로만 위에 존재)
 └ 그룹 (선택 — 프로젝트 안에서 캔버스끼리 묶는 단위. 예: "1차시", "모둠 발표". 필수가 아니라 안 묶어도 됨)
    └ 캔버스 (권한별 QR · 학생 진입 단위 · 진입자 20명(편집자·참여자·뷰어 모두 포함) · 최대 30페이지)
       └ 페이지 (편집자가 템플릿을 붙여 구성 · 최대 30개 · 좌우 추가·이동)
          └ 객체 (텍스트·그리기·포스트잇·도형)
```

> 화면 매핑상 '그룹' 층은 v6대로 **A의 한 단계 안쪽 화면(A-그룹 내부)**으로 드러난다. 계층 자체는 v6과 같다.

#### B(캔버스 편집 — 편집자 뷰) 구조 (v7 구체화)

B는 캔버스 하나를 편집자가 짜고 수업 중 운영하는 화면이다. 와이어프레임(`t-b-edit.html`) 기준으로 네 영역이다.

- **상단바** — 왼쪽부터 [← 목록 복귀], **캔버스명 + 상태 배지**(예: "1모둠 · 진행 중"), **타이머 제어**, **잠금** 버튼(타이머 옆), presence(접속자 아바타), **[설정]**(→C), **[공유]**.
  - **타이머 제어** = 잔여시간 표시 + **일시정지(⏸)** + **드롭다운(▾)**. 드롭다운을 열면 큰 잔여시간 표시·**정지**·**리셋**·**+5/+10/+30/+60분** 조정을 한곳에서 한다. (PRD F8 타이머의 편집자 제어 면.)
  - **잠금** = 타이머와 별개의 버튼. 캔버스를 잠가 학생 편집을 멈춘다(상태가 '잠금'으로). 타이머 만료로도 잠기지만, 이 버튼은 편집자가 수동으로 거는 통로다.
- **페이지 레일** — 상단바 아래 가로 줄. **페이지 칩**(현재 페이지 강조) + **[+ 페이지 추가]**(최대 30) + **N/30 표시**. 칩을 눌러 페이지를 오가고, 추가 버튼으로 페이지를 늘린다(F3·F4).
- **캔버스(작업 영역)** — 격자 배경 위에 객체(포스트잇·텍스트·도형·펜). **객체마다 작성자명이 상시 보인다**(편집자 뷰에서는 항상 표시 — 누가 무엇을 썼는지 운영상 봐야 하므로). 좌우 페이지 넘김 버튼(‹ ›). **템플릿은 페이지별로 설정**한다(한 캔버스 안에서 페이지마다 다른 템플릿 가능).
- **하단 플로팅 도구 팔레트** — 캔버스 위에 떠 있는 FigJam/Miro식 가로 막대. 그룹으로 묶여 있다: **선택·이동(손)** | **텍스트·포스트잇·펜·도형** | **+ 더보기** | **실행취소·다시실행**. → **도구 목록 자체는 미확정**(§4) — 와이어프레임의 항목은 예시다.

#### B의 공유 패널 (오버레이)

B 상단바의 [공유]를 누르면 위에 겹쳐 뜨는 모달이다(교실 프로젝터용 큰 QR). **초대 권한 탭을 먼저 고른다 — 참여자 / 편집자 / 뷰어.** 고른 권한에 맞춰 **QR · URL · [링크 복사]**를 발급·표시한다. 권한마다 별개 링크이고, 이 권한 선택이 학생 입장(D)의 분기를 결정한다(F6). (와이어프레임 기본 선택 = 참여자.)

**A와 A-그룹 내부의 관계 (v6과 동일):** 둘은 같은 종류의 화면(캔버스 카드 그리드)이고, 담는 내용만 다르다. A는 '그룹에 안 묶인 캔버스 + 그룹 폴더'를 섞어 보이고, A-그룹 내부는 '그 그룹에 묶인 캔버스만' 보인다. A의 그룹 폴더 카드를 클릭 → A-그룹 내부로 들어가고, 상단 **브레드크럼('← 캔버스 / 그룹명')으로 A 복귀**한다.

**별도 화면이 아니라 위에 겹쳐 뜨는 것들:**

- **템플릿 선택 모달** — A에서 [+ 캔버스 생성]을 누르거나, 퀴즈앤 링크로 처음 진입하면 자동으로 뜬다. 내용은 위 **T 화면과 같다**. → 목록에는 'T'로 한 번만 올리고, 자동/수동은 진입 경로 차이로만 본다. (A-그룹 내부의 [+ 이 그룹에 캔버스]도 같은 T를 띄우되, 만든 캔버스가 그 그룹에 들어간다.)
- **그룹 만들기 동작** — A 헤더의 [그룹 만들기] 또는 그리드 맨 앞 '새 그룹 만들기' 타일로 시작한다. 새 그룹을 만들고 캔버스를 담는 인터랙션은 아직 안 정함(§4). 만드는 동작은 A 위 오버레이/인라인이고, 만들어진 그룹은 폴더 카드 + A-그룹 내부라는 네비게이션 대상으로 드러난다.
- **B의 공유 패널** — 위 'B의 공유 패널' 항목 참조.
- **캔버스 카드 더보기(⋮) 메뉴 (v7 정정)** — A·A-그룹 내부의 각 캔버스 카드에 달린 메뉴. **복사하기 / 보관 / 삭제** 세 항목이 **모든 캔버스 카드에 상시** 뜬다. '복사하기' = 캔버스 통째 복제(PRD F2) — v6은 이를 '잠금 카드 전용'으로 적었으나, 와이어프레임에선 진행 중·잠금 가리지 않고 모든 카드에 들어간다. 카드 위에 잠깐 겹쳐 뜨는 작은 메뉴이지 화면이 아니다.

> **상태와 필터 (v7 정정).** 캔버스 상태 enum은 v5/v6대로 **진행 중 / 잠금(타이머 만료 또는 수동 잠금) / 보관** 셋이다(데이터). 목록 상단의 **상태 칩은 전체 / 진행 중 / 잠금** 세 가지 보기 전환이다(`t-a-list-v2.html`). v6은 "잠금은 배지일 뿐 필터 아님 / 필터 칩은 전체·진행 중·보관"이라 적었는데, 와이어프레임은 **잠금을 필터 칩에 둔다.** 따라서 v7에서는: **잠금 = 카드 위 상태 배지 + 목록 필터 칩 둘 다.** ('보관'이 별도 보기로 빠지는지 같은 칩 줄에 끼는지는 아직 미정 — §4.)

| **F** | 안내 화면 | 상황 메시지 | 없음(읽고 대기) — 정원 초과 / 잠금(보기 전용) / 보관, 3종 변형 | F5·F8 |
| **G** | 내 활동 (로그인 시) | 참여했던 캔버스 목록 | **재진입 — 그때의 권한대로 E-참여자 또는 E-뷰어로** | F5 |

**E의 권한 변형 2종:** E-참여자(객체 작업 + 공감 투표) / E-뷰어(보기 전용, 새 화면 아니라 E의 변형). **D의 권한 분기:** 편집자 QR=로그인 필수→B / 참여자 QR=로그인 or 닉네임+비번→E-참여자 / 뷰어 QR=로그인 or 닉네임+비번→E-뷰어. 정원 20명은 편집자·참여자·뷰어 모두 포함, 초과 시 F. (상세는 v5 §1과 동일 — 변경 없어 줄인다.)

## 2. 화면 연결 (네비게이션)

```
교사 area (데스크톱)
  A 캔버스 목록                          ← 이 프로젝트의 landing/return 화면 (개념상 '지휘석')
   │  ┌ 캔버스 카드(라이브 썸네일)  ─[열기]──────▶ B 캔버스 편집
   │  │                          └[크게 보기]──▶ 드릴다운(자기 화면 확대)  ※ broadcast 아님(PRD 비목표 5)
   │  │                          └[⋮]─────────▶ 복사하기 / 보관 / 삭제   (모든 카드 상시)
   │  └ 그룹 폴더 카드 ──[클릭]──────────────────▶ A-그룹 내부
   ├─[+ 캔버스 생성] ───────────────▶ T 템플릿 선택 ──[고름]──▶ B 캔버스 편집
   ├─(자동) 템플릿 선택 모달 ────────▶ (= T 내용) ──[고름]──▶ B 캔버스 편집   ← 링크 첫 진입 시
   ├─[그룹 만들기] / '새 그룹' 타일 ─▶ 새 그룹 생성(담기 동작, A 위 인라인) → 그룹 폴더 카드로 나타남
   └─ 상태 칩(전체 / 진행 중 / 잠금) ─▶ A 보기 전환(같은 화면)

  A-그룹 내부 (한 그룹의 캔버스 목록 = A의 드릴인 하위 뷰)
   ├─[← 캔버스] 브레드크럼 ──────────▶ A 복귀
   ├─ 캔버스 카드 ─[열기]────────────▶ B 캔버스 편집
   │              ├[크게 보기]──────▶ 드릴다운(이 그룹 단위)
   │              └[⋮]─────────────▶ 복사하기 / 보관 / 삭제
   └─[+ 이 그룹에 캔버스] ───────────▶ T 템플릿 선택 ──[고름]──▶ B (그 그룹에 소속)

  B 캔버스 편집 (편집자 뷰)
   ├─[← 목록] ────────────────────────▶ A 복귀
   ├─ 타이머 제어 (일시정지 / ▾ 정지·리셋·+5/+10/+30/+60분)  ← 화면 내 동작
   ├─ 잠금 버튼 ───────────────────────▶ 캔버스 상태 '잠금'으로 (화면 내)
   ├─ 페이지 레일 [+ 페이지 추가] (N/30) ─▶ 페이지 추가·이동 (화면 내)
   ├─ 하단 도구 팔레트 (선택·이동 | 텍스트·포스트잇·펜·도형 | + 더보기 | 취소·다시)  ← 도구 미확정
   ├─[설정] ──────────────────────────▶ C 캔버스 설정
   └─[공유] ─── 공유 패널 ─── 권한 탭(참여자 / 편집자 / 뷰어)
                                  ├─ 참여자 QR·URL·링크복사 ┐
                                  ├─ 편집자 QR·URL·링크복사 ┤  (권한별 별개 링크, 학생에게 전달)
                                  └─ 뷰어   QR·URL·링크복사 ┘

학생 area (태블릿·폰)  ※ v5/v6와 동일
  편집자 QR ──▶ D 입장(로그인 필수) ──[로그인]──▶ B 캔버스 편집(교사 area로 합류)
  참여자 QR ──▶ D 입장(로그인 or 닉네임+4글자 비번) ──▶ E-참여자
  뷰어   QR ──▶ D 입장(로그인 or 닉네임+4글자 비번) ──▶ E-뷰어(보기 전용)
                         └─(정원초과·잠금·보관) ──▶ F 안내 화면   ※ 정원 20명은 편집자+참여자+뷰어 모두 포함
  로그인 학생: G 내 활동 ──▶ E-참여자 재진입  (참여자로 참여했던 캔버스)
                        └──▶ E-뷰어   재진입  (뷰어로 봤던 캔버스)
```

두 area의 연결고리는 **B의 공유 패널에서 발급한 권한별 QR**이다. 그중 **편집자 QR은 D를 거쳐 다시 교사 area(B)로 합류**하고, 참여자·뷰어 QR은 학생 area(E)에 머문다. **G(내 활동)는 학생 area 안에서의 재진입 통로**다.

> **v7에서 달라진 곳은 B의 안쪽과 A 카드 메뉴뿐이다.** 화면 사이 연결(누가 어디로 가는지)은 v6과 같다 — A↔A-그룹 내부 드릴인, A/A-그룹→B(열기)·T(생성), B→C(설정)·공유 패널의 권한별 QR, 학생 area 3갈래. 바뀐 것은 ① B 안에 타이머·잠금·페이지 레일·도구 팔레트·공유 패널이라는 화면 내 동작이 명시됐고, ② 카드 ⋮ 메뉴가 '복사하기·보관·삭제'로 또렷해졌고, ③ A 상태 칩이 '전체·진행 중·잠금'이 된 것이다.

> 맨 위에 '프로젝트'가 한 층 더 있지만(여러 프로젝트 전환), PoC에선 프로젝트 1개만 쓰므로 그 화면은 그리지 않는다. A(캔버스 목록)가 곧 그 단일 프로젝트의 안쪽이다.

## 3. 핵심 흐름

**① 교사 첫 사용 (실측 2액션)**
```
퀴즈앤 링크 클릭 → A + 템플릿 선택 모달 자동 → [템플릿 선택] → B(캔버스) → [공유 버튼] → 권한 고르고 QR
```
PRD의 "3액션·30초" 기준 충족(2액션 = 템플릿 선택 1 + 공유 1). 권한 선택은 공유 패널 안에서 일어나고 기본값이 '참여자'로 미리 선택돼 있어 별도 액션으로 세지 않는다(§4). **그룹 만들기는 선택이라 첫 사용 흐름에 들어가지 않는다.**

**② 교사 재사용 — 새로 만들기 / 복사 두 갈래**
```
(새로)  A → [+ 캔버스 생성] → T(빈 캔버스/3종 중) → B → 공유(권한)→QR
(복사)  A → 캔버스 카드 [⋮ → 복사하기] → 복사본(B) → 공유(권한)→QR
```
잘 만든 캔버스 하나를 1반·2반·3반용으로 통째 복사한다(F2). **복사하기는 잠금·진행 중 가리지 않고 모든 카드의 ⋮ 메뉴에 있다.** 복사본은 각각 독립이고 QR도 따로. 묶어 두고 싶으면 A에서 그룹으로 묶을 수 있다(선택).

**③ 교사 — 그룹으로 정리하고 그 안에서 작업 (v6과 동일)**
```
A → [그룹 만들기] / '새 그룹' 타일 → 캔버스 담기 → 그룹 폴더 카드 생성
A → 그룹 폴더 [클릭] → A-그룹 내부 → 캔버스 [열기]→B / [크게 보기]→드릴다운 / [+ 이 그룹에 캔버스]→T
A-그룹 내부 → [← 캔버스] 브레드크럼 → A 복귀
```
"1차시", "모둠 발표"처럼 캔버스를 묶어 두고, 폴더를 눌러 그 그룹의 캔버스만 모아 본다. 이 그룹 단위가 곧 지휘(드릴다운)의 자연스러운 범위가 된다(아래 ⑥, OQ-20).

**④ 교사 — 캔버스 운영 (B 안에서, v7 구체화)**
```
B → 타이머 ▾ (정지 / 리셋 / +5·+10·+30·+60분)  ← 활동 시간 조절
B → 페이지 레일 [+ 페이지 추가] (N/30)            ← 페이지 늘리기 (최대 30)
B → 페이지 칩 클릭 ‹ › 넘김 / 페이지별 템플릿 설정
B → 도구 팔레트로 객체 만들기·편집 (작성자명 상시 표시)
B → [잠금]  ← 학생 편집 멈춤(상태 '잠금'으로)
B → [공유] → 권한 탭(참여자/편집자/뷰어) → 그 권한 QR·URL·링크복사
B → [설정] → C
```
편집자가 수업 전 페이지를 짜고, 수업 중 타이머·잠금으로 활동을 운영하며, 공유 패널로 권한별 QR을 띄운다. 도구 팔레트의 도구 목록은 아직 예시(§4).

**⑤ 학생 입장 — 권한별 3갈래 (v5/v6와 동일)**
```
(참여자) 참여자 QR/URL → D(로그인 or 닉네임+4글자 비번) → E-참여자(함께 만들기)
(뷰어)   뷰어   QR/URL → D(로그인 or 닉네임+4글자 비번) → E-뷰어(보기 전용)
(편집자) 편집자 QR/URL → D(로그인 필수) → B(편집자로 합류)
```
참여자·뷰어는 무설치·무가입, 로딩 3초 이내 목표. **정원 20명은 편집자·참여자·뷰어를 합쳐 센다** — 21번째는 누구든 F(정원 초과 안내)로 빠진다.

**⑥ 교사 지휘 (수업 중)**
```
A(또는 A-그룹 내부) 라이브 썸네일 → '크게 보기'(드릴다운, →자기 화면 확대) / '열기'(→B) / 타이머 제어
```
캔버스 6개를 한 화면에서 스크롤 없이 파악(PRD F7 품질 기준). 막힌 모둠을 돕고, 한 캔버스를 골라 자기 화면에서 크게 본다. '지휘석'은 A의 개념적 역할로만 두고 UI 라벨로 쓰지 않는다 — 드릴다운은 카드의 '크게 보기'로 표현. 지휘 대상 범위 확정은 §4(OQ-20). 그룹으로 묶어 두면 A-그룹 내부가 그 지휘 단위가 된다.

## 4. 아직 안 정한 것 (그림 그리며 판단)

이번 버전에서 새로 또렷해진/남은 열린 점:

- **B 하단 도구 팔레트의 도구 목록.** 와이어프레임은 선택·이동 / 텍스트·포스트잇·펜·도형 / + 더보기 / 실행취소·다시로 그렸으나 **이는 예시이고 확정 아님**. 어떤 도구를 1차에 넣을지(F4 객체 종류와 맞춰), '+ 더보기' 안에 무엇을 둘지는 와이어프레임/스펙에서.
- **공유 패널의 권한 선택 UX.** 탭 전환식(한 번에 한 권한 QR, 참여자 기본 선택)으로 그렸다 — 이대로 2액션을 지킬지, 세 QR 동시 표시안을 둘지 확정 필요.
- **타이머 제어의 표면 분리.** 상단바에 일시정지(⏸)는 노출, 정지·리셋·시간 조정은 드롭다운(▾) 안에 넣었다. 일시정지/정지의 의미 차이(잠깐 멈춤 vs 종료)와 잠금 버튼과의 관계 정리 필요.
- **A 상태 칩 '잠금'과 '보관'의 자리.** 칩을 전체 / 진행 중 / 잠금으로 그렸다. **'보관'이 칩 줄에 끼는지, 별도 보기로 빠지는지** 미정. 보관 캔버스가 같은 그리드에 섞이는지도 함께 판단.
- **그룹 만들기·담기 인터랙션.** 그룹 폴더가 네비게이션 대상인 건 확정. 새 그룹을 만들고 캔버스를 담는 동작(드래그? 다중 선택 후 '그룹으로'? 그룹 만들고 안에서 추가?)은 와이어프레임에서. 이번 IA는 'A 헤더 [그룹 만들기] + 그리드 맨 앞 새 그룹 타일에서 시작 → 폴더 카드로 나타남'까지만 확정.
- **그룹 폴더 카드의 라이브성.** 폴더 미니 썸네일 2×2를 라이브로 갱신할지 정적 대표 썸네일로 둘지 미정(성능·인지 부하 함께 판단).
- **(이번 범위 밖, 다음 패스)** A **빈 상태**(캔버스 0개일 때)와 **C 캔버스 설정** 화면은 아직 작업 중이다. C는 v5/v6 정의(편집자 임명·익명/닉네임·비번 옵션)를 그대로 유지한다.
- **D 입장 화면 분기 표현 / G 권한 표시 / 뷰어 공감 투표(OQ-21) / 4글자 비밀번호(OQ-11) / F 정원 초과 문구** — 모두 v5 §4 그대로 유지(학생 area는 이번에 안 건드림). 상세는 ia-v5 §4 참조.

이전부터 이어지는 열린 점:

- **지휘석(드릴다운) 범위 (PRD OQ-20).** 프로젝트 안 전체 캔버스를 보나, 한 그룹만 보나? v6에서 A-그룹 내부 화면이 생기며 이 화면이 곧 '한 그룹 단위 지휘'의 자연스러운 그릇이 됐다 → '한 그룹 단위'를 기본값으로 깔되 여전히 flag(OQ-20 미해소).
- **복제 시 자동 그룹 묶임 여부 (PRD OQ-19).** 1반·2반·3반 복사본이 자동으로 한 그룹이 될지 수동으로 묶을지 미정 → 일단 수동 가정.
- **템플릿 선택지 개수 (PRD OQ-03).** PRD v9 = 빈 캔버스 + 3종이나 OQ-03에서 3종 자체가 검증 중. 흐름엔 '빈 캔버스 + 템플릿들'로 두고 개수는 와이어프레임에서(현재 HTML은 빈 캔버스 + 2종으로 그림).

> **PRD와의 정합 메모:** 이번 v7의 결정들은 **PRD v9와 어긋나지 않는다.**
> - B의 타이머·잠금·페이지 추가·작성자명·도구는 **F4(캔버스 편집)·F3(페이지·템플릿)·F8(타이머)** 범위 안이다.
> - 공유 패널의 권한별 QR은 **F6(공유·권한)** 범위 안이다.
> - 카드 '복사하기'는 **F2(캔버스 복제)** 범위 안이다. v6이 이를 '잠금 카드 전용'으로 좁게 적었던 것을 '모든 카드 상시'로 넓힌 것이며, 이는 화면 수준 결정이지 새 제품 결정이 아니다(F2는 어느 캔버스든 복제 가능).
> - A 상태 칩에 '잠금'을 둔 것도 상태 enum(진행 중/잠금/보관) 안에서의 보기 선택이라 새 스펙을 만들지 않는다.
> **상위 문서로 돌려보낼 갭은 발견되지 않음.** (PRD 잔여 OQ는 prd-v9 §9 참조 — OQ-02·OQ-03·OQ-11·OQ-19·OQ-20·OQ-21 등. 와이어프레임 진행에는 막힘 없음.)

> **FigJam 흐름도 갱신 필요(미반영).** 이 v7은 백그라운드 작업으로 진행돼 FigJam 인증을 거치지 않았다. 기존 흐름도(`ux/ia-flowchart-v5.png`, fileKey `OT6YlTAIzTNYMrf0m9yRoo`)는 v5 구조라 v6·v7과 어긋난다. 다음 대화에서 갱신할 때 반영할 변경점:
> - (v6 미반영분) 교사 area에 **A-그룹 내부** 노드 추가, 그룹 폴더→A-그룹 내부 네비게이션 엣지, 브레드크럼 복귀 양방향 엣지.
> - (v7) **B 노드의 라벨/하위 엣지**를 타이머·잠금·페이지 레일·도구 팔레트·[설정]→C·[공유]→권한별 QR로 분기 표기(밀집하면 B 내부는 주석으로 빼고 노드는 간결히).
> - (v7) 캔버스 카드 ⋮ 분기를 **복사하기 / 보관 / 삭제**로 표기(v6의 '보관·삭제·(잠금만)복제'에서 정정).
> - (v7) A 상태 칩을 **전체 / 진행 중 / 잠금**으로(필요 시 주석).
> - 학생 area는 v5 흐름도 그대로 유지.
