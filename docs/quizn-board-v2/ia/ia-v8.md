## 0. 이 문서가 무엇인가

이 제품에 **화면이 몇 개 있고, 각 화면이 무엇을 보여주고, 화면끼리 어떻게 이어지는지**를 적은 설계도다. 실제 그림(와이어프레임)을 그리기 전에 그릴 대상의 목록과 연결을 먼저 확정하는 단계다. 이번 버전은 그 와이어프레임을 HTML로 그리다 교사 area 전체가 또렷해진 것을 거꾸로 IA에 반영한 것이다.

전체는 두 area로 나뉜다 — **교사가 쓰는 화면**(데스크톱)과 **학생이 쓰는 화면**(태블릿·폰). 둘이 만나는 유일한 접점은 캔버스다.

- 교사 화면 기준: 데스크톱 1440px
- 학생 화면 기준: 태블릿 1024px (+ 폰 390px 변형)

### 먼저 짚는 용어: 역할 4단계

화면을 보기 전에 한 줄씩 정의한다. PRD v9 §1을 그대로 따른다.

- **소유자** = 프로젝트를 만든 사람(보통 교사). 로그인 필수. 모든 것을 제어한다.
- **편집자** = 소유자나 다른 편집자가 임명·초대한 사람. 로그인 필수. 임명된 캔버스의 페이지 추가·템플릿 설정·편집이 가능하다.
- **참여자** = 캔버스에 들어와 **함께 만드는** 사람(보통 학생). 무계정 입장 가능. 객체 생성·이동·편집 + 포스트잇 공감 투표.
- **뷰어** = 캔버스에 들어와 **보기만** 하는 사람. 무계정 입장 가능. 객체 편집 불가, 페이지 변경 불가, 실시간 반영을 보기만. 공감 투표 가능 여부는 미정(기본 불가) — PRD OQ-21.

> 화면 수준에서 핵심은: 소유자·편집자는 **교사 area**(데스크톱)에 산다. 참여자·뷰어는 **학생 area**(태블릿·폰)에 산다. 참여자와 뷰어는 **같은 화면(E)**을 쓰되, 뷰어는 그 화면의 '보기 전용' 변형으로 들어온다.

### 계층 (PRD v9 §2 그대로)

위에서 아래로 담는 관계다. v6·v7과 동일하다.

```
프로젝트 (최상위 통 — 소유자가 생성. PoC에선 사실상 1개. 다중 프로젝트 화면은 범위 밖, 개념상으로만 위에 존재)
 └ 그룹 (선택 — 프로젝트 안에서 캔버스끼리 묶는 단위. 예: "1차시", "모둠 발표". 필수가 아니라 안 묶어도 됨)
    └ 캔버스 (권한별 QR · 학생 진입 단위 · 진입자 20명(편집자·참여자·뷰어 모두 포함) · 최대 30페이지)
       └ 페이지 (편집자가 템플릿을 붙여 구성 · 최대 30개 · 좌우 추가·이동)
          └ 객체 (텍스트·그리기·포스트잇·도형)
```

> 화면 매핑상 '그룹' 층은 v6대로 **A의 한 단계 안쪽 화면(A-그룹 내부)**으로 드러난다. 계층 자체는 v6·v7과 같다.

#### 새 캔버스 만들기 모달 (v8 정정 — 독립 화면 아님)

v7까지 'T 템플릿 선택'을 독립 화면처럼 한 줄 올렸으나, 와이어프레임에서 이것은 **여러 화면에서 같은 코드로 뜨는 공용 모달**(`wf-screen.js`)이다. 그래서 화면 목록에서 빼고 오버레이로 옮긴다.

- 내용 = **빈 캔버스 + 템플릿 3종**(빈 캔버스·브레인스토밍·마인드맵, 개수는 OQ-03) 미리보기 그리드 + [취소]·[이 템플릿으로 시작].
- 뜨는 경로(트리거 = `[data-new-canvas]`) — 모두 같은 모달:
  - A 헤더의 **[+ 캔버스 생성]** / 그리드 맨 앞 **'새 캔버스 만들기' 타일**
  - **A 빈 상태**의 중앙 CTA
  - **A-그룹 내부**의 **[+ 이 그룹에 캔버스]**(만든 캔버스가 그 그룹에 들어감)
  - **B의 페이지 레일 [+ 페이지 추가]** — 즉 **페이지 추가 = 그 페이지에 깔 템플릿을 고르는 일**(F3, 캔버스당 최대 30페이지)
  - 링크 첫 진입 시 자동(F9)
- 한 가지 의미 차이: A 쪽에서 고르면 **새 캔버스**가 만들어지고, B의 [+ 페이지 추가]에서 고르면 **그 캔버스에 새 페이지**가 붙는다. 화면(모달)은 같고 결과 단위만 다르다.

#### B(캔버스 편집 — 편집자 뷰) 구조

B는 캔버스 하나를 편집자가 짜고 수업 중 운영하는 화면이다. 와이어프레임(`t-b-edit.html`) 기준으로 네 영역이다.

- **상단바** — 왼쪽부터 [← 목록 복귀], **캔버스명 + 상태 배지**(예: "1모둠 · 진행 중"), **타이머 제어**, **잠금** 버튼, presence(접속자 아바타), **[설정]**(→C), **[공유]**.
  - **타이머 제어** = 잔여시간 + **일시정지(⏸)** + **드롭다운(▾)**. 드롭다운을 열면 큰 잔여시간 표시·**정지**·**리셋**·**+5/+10/+30/+60분** 조정을 한곳에서 한다(F8).
  - **잠금** = 타이머와 별개 버튼. 캔버스를 잠가 학생 편집을 멈춘다(상태 '잠금'으로). 타이머 만료로도 잠기지만, 이 버튼은 수동 통로다.
- **페이지 레일** — 가로 줄. **페이지 칩**(현재 페이지 강조) + **[+ 페이지 추가]** + **N/30 표시**(예: "2 / 5 페이지 · 최대 30"). 칩으로 페이지를 오가고, **추가 버튼은 새 캔버스 모달을 띄워 페이지별 템플릿을 고른다**(F3·F4).
- **캔버스(작업 영역)** — 격자 배경 위 객체(포스트잇·텍스트·도형·펜). **포스트잇은 작성자명을 좌하단에 상시 표시 + 감정표현 버튼을 우하단**에 둔다(아래 박스). 좌우 페이지 넘김 버튼(‹ ›). **템플릿은 페이지별** 설정.
- **하단 플로팅 도구 팔레트** — 캔버스 위 가로 막대. 그룹: **선택·이동(손)** | **텍스트·포스트잇·펜·도형** | **+ 더보기** | **실행취소·다시실행**. → **도구 목록 자체는 미확정**(§4) — 와이어프레임 항목은 예시다.

#### B 포스트잇: 작성자명 + 감정표현 (v8 명시)

B의 포스트잇 객체는 본문 아래 발(footer)에 두 가지를 둔다 — **좌하단에 작성자명**(예: "환경지킴이", 편집자 뷰에서 상시 표시. PRD §1 가시성 규칙 — 소유자·편집자는 항상 작성자 확인, 작성자 표기 방식 자체는 OQ-18), **우하단에 감정표현 버튼**(☺). 이 감정표현은 **F10 공감/반응 계열**이며, 학생 area의 **E-참여자 공감 투표와 같은 계열**이다. 다만 둘을 하나의 표현(같은 아이콘·같은 카운트 규칙)으로 통일할지는 **미정**으로 둔다(§4) — F10 세부와 OQ-16에 묶임.

#### B의 공유 패널 (오버레이)

B 상단바의 [공유]를 누르면 위에 겹쳐 뜨는 모달이다(교실 프로젝터용 큰 QR). **초대 권한 탭을 먼저 고른다 — 참여자 / 편집자 / 뷰어.** 고른 권한에 맞춰 **QR · URL · [링크 복사]**를 발급·표시한다. 권한마다 별개 링크이고, 이 권한 선택이 학생 입장(D)의 분기를 결정한다(F6). 와이어프레임 기본 선택 = 참여자.

#### C(캔버스 설정) 구조 (v8 추가)

B 상단바 [설정]에서 들어가는 화면이다. 상단 **브레드크럼('← 1모둠 편집 / 설정')으로 B 복귀**한다. 와이어프레임(`t-c-settings.html`) 기준 세 묶음이다(F6).

- **편집자** — 소유자 행 + 임명된 편집자 행(각 행에 '임명 해제'). **[+ 편집자 임명]** + "로그인 사용자만 임명 / '편집자' QR·링크로도 초대" 안내(QR 발급 자체는 B 공유 패널의 편집자 탭에서 한다).
- **옵션** — **익명 옵션 토글**(켜면 참여자끼리 작성자 비공개, 소유자·편집자는 항상 확인) / **닉네임 + 비밀번호 4글자**는 "끌 수 없는 기본 설정"으로 잠긴 채 표시(PRD §1·F5 — 토글 아님).
- **위험 구역** — **캔버스 삭제**(객체·페이지 모두 사라짐, 되돌릴 수 없음 안내).

**A·A-빈 상태·A-그룹 내부의 관계:** 셋 다 같은 종류의 화면(캔버스 그리드)이고 담는 내용만 다르다. A는 '그룹에 안 묶인 캔버스 + 그룹 폴더'를 섞어 보이고, A-빈 상태는 '캔버스 0개'일 때의 A이며, A-그룹 내부는 '그 그룹에 묶인 캔버스만' 보인다. A의 그룹 폴더 카드를 클릭 → A-그룹 내부로 들어가고, 상단 **브레드크럼('← 캔버스 / 그룹명')으로 A 복귀**한다.

**별도 화면이 아니라 위에 겹쳐 뜨는 것들:**

- **새 캔버스 만들기 모달** — 위 'v8 정정' 박스 참조. A·A-빈 상태·A-그룹 내부·B(페이지 추가)·링크 첫 진입에서 같은 코드로 뜬다. 목록에는 올리지 않는다.
- **그룹 만들기 동작** — A 헤더의 [그룹 만들기] 또는 그리드 맨 앞 '새 그룹 만들기' 타일로 시작. 새 그룹을 만들고 캔버스를 담는 인터랙션은 아직 안 정함(§4). 만드는 동작은 A 위 오버레이/인라인이고, 만들어진 그룹은 폴더 카드 + A-그룹 내부라는 네비게이션 대상으로 드러난다.
- **B의 공유 패널** — 위 'B의 공유 패널' 항목 참조.
- **캔버스 카드 더보기(⋮) 메뉴** — A·A-그룹 내부의 각 캔버스 카드에 달린 메뉴. **복사하기 / 보관 / 삭제** 세 항목이 **모든 캔버스 카드에 상시** 뜬다. '복사하기' = 캔버스 통째 복제(F2), 진행 중·잠금 가리지 않고 들어간다. 카드 위에 잠깐 겹쳐 뜨는 작은 메뉴이지 화면이 아니다.

> **상태와 필터.** 캔버스 상태 enum은 **진행 중 / 잠금(타이머 만료 또는 수동 잠금) / 보관** 셋이다(데이터). 목록 상단의 **상태 칩은 전체 / 진행 중 / 잠금** 세 가지 보기 전환이다(`t-a-list-v2.html`). 따라서 **잠금 = 카드 위 상태 배지 + 목록 필터 칩 둘 다.** ('보관'이 별도 보기로 빠지는지 같은 칩 줄에 끼는지는 아직 미정 — §4.)

### 학생 화면 (4개 화면 + E의 권한 변형 2종)

> 학생 area는 v5~v7에서 바뀐 것이 없다. 아래는 그대로다(요약). 이번 v8의 변경은 교사 area 안에서만 일어나며 학생 동선에 영향이 없다. **학생 area는 아직 HTML 미작업** — 정의를 건드리지 않는다.

| 코드 | 화면 | 무엇을 보여주나 | 무엇을 하나 | PRD 기능 |
|---|---|---|---|---|
| **D** | 입장 화면 | 캔버스 이름. **들어온 QR의 권한에 따라 입장 방식이 갈림** | 권한별 분기에 따라 로그인 또는 닉네임+4글자 비번 입력 → 입장 | F5 |
| **E** | 캔버스 편집 | 편집자가 짜둔 페이지, 하단 도구바, 잔여 시간, presence | **권한 변형 2종**(아래) | F4·F10 |
| **F** | 안내 화면 | 상황 메시지 | 없음(읽고 대기) — 정원 초과 / 잠금(보기 전용) / 보관, 3종 변형 | F5·F8 |
| **G** | 내 활동 (로그인 시) | 참여했던 캔버스 목록 | **재진입 — 그때의 권한대로 E-참여자 또는 E-뷰어로** | F5 |

**E의 권한 변형 2종:** E-참여자(객체 작업 + 공감 투표) / E-뷰어(보기 전용, 새 화면 아니라 E의 변형). **D의 권한 분기:** 편집자 QR=로그인 필수→B / 참여자 QR=로그인 or 닉네임+비번→E-참여자 / 뷰어 QR=로그인 or 닉네임+비번→E-뷰어. 정원 20명은 편집자·참여자·뷰어 모두 포함, 초과 시 F. (상세는 v5 §1과 동일.)

## 2. 화면 연결 (네비게이션)

```
교사 area (데스크톱)
  A 캔버스 목록                          ← 이 프로젝트의 landing/return 화면 (개념상 '지휘석')
   │  ┌ 캔버스 카드(라이브 썸네일)  ─[카드 클릭]──▶ B 캔버스 편집   ※ '열기'/'크게 보기' 버튼 없앰 — 카드 클릭이 곧 열기
   │  │                          └[⋮]─────────▶ 복사하기 / 보관 / 삭제   (모든 카드 상시)
   │  └ 그룹 폴더 카드 ──[클릭]──────────────────▶ A-그룹 내부
   ├─[+ 캔버스 생성] / '새 캔버스' 타일 ─▶ 새 캔버스 모달 ──[고름]──▶ B 캔버스 편집
   ├─(자동) 새 캔버스 모달 ──────────▶ (같은 모달) ──[고름]──▶ B 캔버스 편집   ← 링크 첫 진입 시
   ├─[그룹 만들기] / '새 그룹' 타일 ─▶ 새 그룹 생성(담기 동작, A 위 인라인) → 그룹 폴더 카드로 나타남
   └─ 상태 칩(전체 / 진행 중 / 잠금) ─▶ A 보기 전환(같은 화면)

  A-빈 상태 (캔버스 0개일 때의 A)
   └─ 중앙 [+ 캔버스 생성] ───────────▶ 새 캔버스 모달 ──[고름]──▶ B 캔버스 편집

  A-그룹 내부 (한 그룹의 캔버스 목록 = A의 드릴인 하위 뷰)
   ├─[← 캔버스] 브레드크럼 ──────────▶ A 복귀
   ├─ 캔버스 카드 ─[카드 클릭]───────▶ B 캔버스 편집
   │              └[⋮]─────────────▶ 복사하기 / 보관 / 삭제
   └─[+ 이 그룹에 캔버스] ───────────▶ 새 캔버스 모달 ──[고름]──▶ B (그 그룹에 소속)

  B 캔버스 편집 (편집자 뷰)
   ├─[← 목록] ────────────────────────▶ A 복귀
   ├─ 타이머 제어 (일시정지 / ▾ 정지·리셋·+5/+10/+30/+60분)  ← 화면 내 동작
   ├─ 잠금 버튼 ───────────────────────▶ 캔버스 상태 '잠금'으로 (화면 내)
   ├─ 페이지 레일 [+ 페이지 추가] (N/30) ─▶ 새 캔버스 모달(페이지별 템플릿 선택) → 그 캔버스에 새 페이지
   ├─ 포스트잇 (작성자명 좌하단 · 감정표현 ☺ 우하단)  ← 화면 내 (F10 계열)
   ├─ 하단 도구 팔레트 (선택·이동 | 텍스트·포스트잇·펜·도형 | + 더보기 | 취소·다시)  ← 도구 미확정
   ├─[설정] ──────────────────────────▶ C 캔버스 설정
   └─[공유] ─── 공유 패널 ─── 권한 탭(참여자 / 편집자 / 뷰어)
                                  ├─ 참여자 QR·URL·링크복사 ┐
                                  ├─ 편집자 QR·URL·링크복사 ┤  (권한별 별개 링크, 학생에게 전달)
                                  └─ 뷰어   QR·URL·링크복사 ┘

  C 캔버스 설정
   ├─[← 1모둠 편집] 브레드크럼 ────────▶ B 복귀
   ├─ 편집자 목록 (임명 / 임명 해제, 'QR 초대'는 B 공유 패널 편집자 탭)
   ├─ 익명 옵션 토글 / 닉네임+비번4(고정·끌 수 없음)
   └─ 캔버스 삭제

학생 area (태블릿·폰)  ※ v5~v7와 동일 (HTML 미작업)
  편집자 QR ──▶ D 입장(로그인 필수) ──[로그인]──▶ B 캔버스 편집(교사 area로 합류)
  참여자 QR ──▶ D 입장(로그인 or 닉네임+4글자 비번) ──▶ E-참여자
  뷰어   QR ──▶ D 입장(로그인 or 닉네임+4글자 비번) ──▶ E-뷰어(보기 전용)
                         └─(정원초과·잠금·보관) ──▶ F 안내 화면   ※ 정원 20명은 편집자+참여자+뷰어 모두 포함
  로그인 학생: G 내 활동 ──▶ E-참여자 재진입  (참여자로 참여했던 캔버스)
                        └──▶ E-뷰어   재진입  (뷰어로 봤던 캔버스)
```

두 area의 연결고리는 **B의 공유 패널에서 발급한 권한별 QR**이다. 그중 **편집자 QR은 D를 거쳐 다시 교사 area(B)로 합류**하고, 참여자·뷰어 QR은 학생 area(E)에 머문다. **G(내 활동)는 학생 area 안에서의 재진입 통로**다.

> **v8에서 달라진 곳은 교사 area의 진입·생성 동선과 카드 클릭, 그리고 C 설정 추가뿐이다.** ① '템플릿 선택 T'가 독립 화면에서 빠지고 **새 캔버스 모달**로 합쳐졌다(A·A-빈 상태·A-그룹 내부·B 페이지 추가에서 같은 모달). ② A 카드의 '열기'·'크게 보기' 버튼이 사라지고 **카드 클릭=열기**로 단순화됐다. ③ **C(캔버스 설정)**가 B [설정]에서 진입하는 실제 화면으로 추가됐다. ④ **A-빈 상태**가 첫 진입 화면으로 추가됐다. 학생 area 연결은 v5~v7과 같다.

> 맨 위에 '프로젝트'가 한 층 더 있지만(여러 프로젝트 전환), PoC에선 프로젝트 1개만 쓰므로 그 화면은 그리지 않는다. A(캔버스 목록)가 곧 그 단일 프로젝트의 안쪽이다.

## 3. 핵심 흐름

**① 교사 첫 사용 (실측 2액션)**
```
퀴즈앤 링크 클릭 → A + 새 캔버스 모달 자동 → [템플릿 선택] → B(캔버스) → [공유 버튼] → 권한 고르고 QR
```
PRD의 "3액션·30초" 기준 충족(2액션 = 템플릿 선택 1 + 공유 1). 권한 선택은 공유 패널 안에서 일어나고 기본값이 '참여자'로 미리 선택돼 있어 별도 액션으로 세지 않는다(§4). 캔버스가 0개면 A 대신 **A-빈 상태**가 뜨지만, 모달이 자동으로 떠 같은 2액션이 된다. **그룹 만들기는 선택이라 첫 사용 흐름에 들어가지 않는다.**

**② 교사 재사용 — 새로 만들기 / 복사 두 갈래**
```
(새로)  A → [+ 캔버스 생성] / '새 캔버스' 타일 → 새 캔버스 모달(빈/3종) → B → 공유(권한)→QR
(복사)  A → 캔버스 카드 [⋮ → 복사하기] → 복사본(B) → 공유(권한)→QR
```
잘 만든 캔버스 하나를 1반·2반·3반용으로 통째 복사한다(F2). **복사하기는 잠금·진행 중 가리지 않고 모든 카드의 ⋮ 메뉴에 있다.** 복사본은 각각 독립이고 QR도 따로. 묶어 두고 싶으면 A에서 그룹으로 묶을 수 있다(선택).

**③ 교사 — 그룹으로 정리하고 그 안에서 작업**
```
A → [그룹 만들기] / '새 그룹' 타일 → 캔버스 담기 → 그룹 폴더 카드 생성
A → 그룹 폴더 [클릭] → A-그룹 내부 → 캔버스 [카드 클릭]→B / [+ 이 그룹에 캔버스]→새 캔버스 모달
A-그룹 내부 → [← 캔버스] 브레드크럼 → A 복귀
```
"1차시", "모둠 발표"처럼 캔버스를 묶어 두고, 폴더를 눌러 그 그룹의 캔버스만 모아 본다. 이 그룹 단위가 곧 지휘의 자연스러운 범위가 된다(아래 ⑥, OQ-20).

**④ 교사 — 캔버스 운영 (B 안에서)**
```
B → 타이머 ▾ (정지 / 리셋 / +5·+10·+30·+60분)        ← 활동 시간 조절
B → 페이지 레일 [+ 페이지 추가] → 새 캔버스 모달(페이지별 템플릿) → 새 페이지 (최대 30)
B → 페이지 칩 클릭 ‹ › 넘김
B → 도구 팔레트로 객체 만들기·편집 (포스트잇: 작성자명 좌하단 · 감정표현 ☺ 우하단)
B → [잠금]  ← 학생 편집 멈춤(상태 '잠금'으로)
B → [공유] → 권한 탭(참여자/편집자/뷰어) → 그 권한 QR·URL·링크복사
B → [설정] → C → (편집자 임명·임명 해제 / 익명 토글 / 캔버스 삭제) → [← 편집] 브레드크럼으로 B 복귀
```
편집자가 수업 전 페이지를 짜고, 수업 중 타이머·잠금으로 활동을 운영하며, 공유 패널로 권한별 QR을 띄운다. 페이지 추가가 곧 템플릿 선택이라는 점이 v8에서 또렷해졌다. 도구 팔레트의 도구 목록은 아직 예시(§4).

**⑤ 학생 입장 — 권한별 3갈래 (v5~v7와 동일)**
```
(참여자) 참여자 QR/URL → D(로그인 or 닉네임+4글자 비번) → E-참여자(함께 만들기)
(뷰어)   뷰어   QR/URL → D(로그인 or 닉네임+4글자 비번) → E-뷰어(보기 전용)
(편집자) 편집자 QR/URL → D(로그인 필수) → B(편집자로 합류)
```
참여자·뷰어는 무설치·무가입, 로딩 3초 이내 목표. **정원 20명은 편집자·참여자·뷰어를 합쳐 센다** — 21번째는 누구든 F(정원 초과 안내)로 빠진다.

**⑥ 교사 지휘 (수업 중) — ⚠️ 진입점 갭**
```
A(또는 A-그룹 내부) 라이브 썸네일 → 막힌 모둠 파악 / 캔버스 [카드 클릭]→B / 타이머 제어
```
캔버스 6개를 한 화면에서 스크롤 없이 파악(PRD F7 품질 기준)하고 막힌 모둠을 돕는 것까지는 A의 라이브 썸네일 그리드로 된다. **그런데 F7의 핵심인 '특정 캔버스를 자기 화면에 크게 드릴다운해 들여다보기'의 별도 진입점이 v8에서 사라졌다** — v7의 카드 '크게 보기' 버튼을 없애고 카드 클릭을 'B 열기'로 단순화하면서, 드릴다운(원본 편집이 아닌 '확대 열람')으로 가는 통로가 없어졌다. 이는 화면 수준에서 메울 수 없는 문제다(§4·§5 참조). 그룹으로 묶어 두면 A-그룹 내부가 그 지휘 단위 후보가 된다(OQ-20).

## 4. 아직 안 정한 것 (그림 그리며 판단)

이번 v8에서 새로 또렷해진/남은 열린 점:

- **⚠️ F7 드릴다운(자기 화면 확대 열람)의 진입점.** A 카드 인터랙션을 '카드 클릭=B 열기'로 단순화하면서 '크게 보기'(드릴다운) 진입점이 사라졌다. F7은 '캔버스를 자기 화면에 **크게 열어 들여다보기**(원본 열람·진입) + 타이머·잠금 제어'를 P0로 요구하는데, 지금 카드 클릭은 곧장 **편집(B)**으로 간다. 드릴다운(확대 열람)과 편집(B)이 **같은 동작인지 다른 동작인지**가 결정돼야 진입점을 다시 그릴 수 있다 → **상위 문서(PRD)로 돌려보낼 갭**(§5).
- **B 포스트잇 감정표현 ↔ E 참여자 공감 투표의 통일 여부.** B의 포스트잇 우하단 감정표현 버튼(☺)과 E-참여자의 F10 공감 투표가 같은 계열인 건 맞지만, **같은 표현(아이콘·카운트·취소 규칙)으로 통일할지** 미정. F10 세부·OQ-16과 묶임.
- **새 캔버스 모달의 '페이지 추가' 결과 표현.** A에서 띄우면 새 캔버스, B [+ 페이지 추가]에서 띄우면 그 캔버스의 새 페이지 — 모달은 같다. 두 경우의 카피(예: "이 템플릿으로 시작" vs "이 페이지 추가")나 헤더 문구를 분기할지 와이어프레임에서 확정.
- **B 하단 도구 팔레트의 도구 목록.** 와이어프레임은 선택·이동 / 텍스트·포스트잇·펜·도형 / + 더보기 / 실행취소·다시로 그렸으나 **예시이고 확정 아님**. 1차에 넣을 도구(F4 객체 종류와 맞춰)와 '+ 더보기' 내용은 와이어프레임/스펙에서.
- **공유 패널의 권한 선택 UX.** 탭 전환식(한 번에 한 권한 QR, 참여자 기본 선택)으로 그렸다 — 이대로 2액션을 지킬지, 세 QR 동시 표시안을 둘지 확정 필요.
- **타이머 제어의 표면 분리.** 상단바에 일시정지(⏸)는 노출, 정지·리셋·시간 조정은 드롭다운(▾) 안. 일시정지/정지의 의미 차이(잠깐 멈춤 vs 종료)와 잠금 버튼과의 관계 정리 필요.
- **A 상태 칩 '잠금'과 '보관'의 자리.** 칩을 전체 / 진행 중 / 잠금으로 그렸다. **'보관'이 칩 줄에 끼는지, 별도 보기로 빠지는지** 미정. 보관 캔버스가 같은 그리드에 섞이는지도 함께 판단.
- **그룹 만들기·담기 인터랙션.** 그룹 폴더가 네비게이션 대상인 건 확정. 새 그룹을 만들고 캔버스를 담는 동작(드래그? 다중 선택 후 '그룹으로'? 그룹 만들고 안에서 추가?)은 와이어프레임에서. 이번 IA는 'A 헤더 [그룹 만들기] + 그리드 맨 앞 새 그룹 타일에서 시작 → 폴더 카드로 나타남'까지만 확정.
- **그룹 폴더 카드의 라이브성.** 폴더 미니 썸네일 2×2를 라이브로 갱신할지 정적 대표 썸네일로 둘지 미정.
- **C 편집자 'QR 초대' 위치.** 편집자 QR 발급은 B 공유 패널의 편집자 탭에 있고, C 설정은 '편집자 임명·임명 해제 + QR 초대 안내'만 둔다. C에서도 직접 발급할지(중복 통로)는 미정 — 우선 B 공유 패널 한 곳으로 둠.

이전부터 이어지는 열린 점:

- **지휘석(드릴다운) 범위 (PRD OQ-20).** 프로젝트 안 전체 캔버스인지 한 그룹만인지. A-그룹 내부 화면이 '한 그룹 단위 지휘'의 자연스러운 그릇이 됨 → '한 그룹 단위' 기본 가정, 여전히 flag(미해소). **위 F7 진입점 갭과 함께 풀어야 한다.**
- **복제 시 자동 그룹 묶임 여부 (PRD OQ-19).** 1반·2반·3반 복사본이 자동 한 그룹이 될지 수동일지 미정 → 수동 가정.
- **템플릿 선택지 개수 (PRD OQ-03).** PRD v9 = 빈 캔버스 + 3종이나 OQ-03에서 검증 중. 모달엔 '빈 캔버스 + 템플릿들'로 두고 개수는 와이어프레임에서(현재 HTML은 빈 캔버스 + 브레인스토밍 + 마인드맵 3종으로 그림).
- **D 입장 분기 표현 / G 권한 표시 / 뷰어 공감 투표(OQ-21) / 4글자 비밀번호(OQ-11) / F 정원 초과 문구** — 모두 v5 §4 그대로 유지(학생 area는 이번에 안 건드림 — HTML 미작업).

## 5. PRD와의 정합 (상위 문서로 돌려보낼 갭 1건)

> **이번 v8의 결정 대부분은 PRD v9 범위 안이다.**
> - **새 캔버스 모달(B 페이지 추가 포함)**: F1(생성 흐름, 자동/수동 동일)·F3(페이지별 템플릿 설정)·F9(링크 첫 진입 자동 모달) 안. 'T를 독립 화면에서 모달로' 바꾼 것은 화면 수준 결정이지 새 제품 결정이 아니다(PRD F1이 이미 '템플릿 선택 화면/모달'을 동일 흐름으로 규정).
> - **A 카드 인터랙션 단순화·메타 축소**: 카드 클릭=열기, 메타 '참여 N/20'만 — F1·F7의 정보(참여 인원) 범위 안. 표기를 줄인 것이지 데이터를 없앤 것이 아니다.
> - **C 캔버스 설정·A 빈 상태**: F6(편집자 임명·익명 옵션·닉네임+비번4 고정)·F1(빈 상태도 생성 진입) 안.
> - **B 포스트잇 작성자명·감정표현**: §1 작성자 가시성(OQ-18)·F10(공감) 안.
>
> **⚠️ 상위 문서(PRD)로 돌려보낼 갭 1건 — F7 드릴다운 진입점.**
> PRD F7은 '특정 캔버스를 **자기 화면에 크게 열어(드릴다운) 들여다보기** + 타이머·잠금 제어'를 P0로 명시한다(원본 열람·진입). v7까지는 카드의 '크게 보기'(드릴다운)와 '열기'(편집 B)를 **별개 진입점**으로 두어 이를 충족했다. v8에서 카드 인터랙션을 '카드 클릭=B 열기'로 단순화하면서 **드릴다운(확대 열람) 전용 진입점이 사라졌다.** 이는 화면 배치로 메울 문제가 아니라 **제품 정의 질문**이다 — 교사의 '드릴다운(확대 열람·지휘)'과 '편집(B 진입)'이 **같은 동작인가 다른 동작인가?**
> - 같다면: 카드 클릭=B로 충분하고 F7 진입은 곧 편집 진입 → IA에 추가 화면 불필요(F7 서술을 그에 맞게 정리).
> - 다르다면: 드릴다운(편집과 구분되는 '읽기 중심 확대 열람') 진입점을 IA에 다시 그려야 함 → 카드 더블클릭/별도 아이콘/카드 클릭=드릴다운+드릴다운 안에서 편집 진입 등.
> 이 결정은 OQ-20(지휘 대상 범위)과도 엮인다. **@prd-writer로 라우팅해 F7과 OQ-20을 함께 정리하는 것을 권한다.** 그 전까지 IA는 위 ⑥·§4에 갭으로 표시해 두고, 카드 클릭=B 열기를 잠정값으로 둔다.
>
> (PRD 잔여 OQ는 prd-v9 §9 참조 — OQ-02·OQ-03·OQ-11·OQ-16·OQ-19·OQ-20·OQ-21 등. 위 F7 갭을 빼면 와이어프레임 진행에 막힘 없음.)

## 부록. 흐름도 상태

> **FigJam 흐름도 갱신 필요(미반영).** 이 v8은 백그라운드 작업으로 진행돼 FigJam 인증을 거치지 않았다. 기존 흐름도(`ux/ia/ia-flowchart-v5.png`, fileKey `OT6YlTAIzTNYMrf0m9yRoo`)는 v5 구조라 v6~v8과 어긋난다. 다음 대화에서 갱신할 때 반영할 변경점:
> - (v6 미반영분) 교사 area에 **A-그룹 내부** 노드 + 그룹 폴더→A-그룹 내부 엣지 + 브레드크럼 복귀 양방향 엣지.
> - (v7 미반영분) **B 노드 하위 엣지**를 타이머·잠금·페이지 레일·도구 팔레트·[설정]→C·[공유]→권한별 QR로 분기 표기.
> - (v8) **'T 템플릿 선택' 노드를 없애고 '새 캔버스 모달' 오버레이 노드(육각형)로 통합** — A·A-빈 상태·A-그룹 내부·B(페이지 추가)에서 이 모달로 향하는 엣지. 자동 진입은 점선.
> - (v8) A 카드 엣지를 **'카드 클릭→B'** 단일로(‘크게 보기’ 드릴다운 엣지 제거). **단, F7 드릴다운 진입점은 갭이므로 노드 옆 주석으로 '드릴다운 진입점 미정(PRD F7 라우팅)'을 남길 것.**
