# quizn-board-v2 — 화이트보드(canvas) 형식의 실시간 공동 참여 보드
> Project: `quizn-board-v2`
> One-line summary: AIDT 붕괴로 유휴 상태가 된 1:1 교실 기기 위에, 교사가 가볍게 학생을 참여시키는 canvas형 실시간 공동참여 보드를 퀴즈앤 채널로 진입시킨다
> Date: 2026-06-10 / Version: v3

---

## 1. 배경 / 현황 → 가설

- 한국 중·고에 1:1 기기는 이미 깔림(컴퓨터 1대당 학생 중 0.9 / 고 1.0) ([서울경제](https://www.sedaily.com/article/20028437)). 보급 명분이던 AIDT는 한 학기 만에 붕괴 — 평균 활용률 8.1%, 1회도 미접속 60%, 채택 32%→19% ([교육을비추다](https://www.kyobit.com/news/articleView.html?idxno=3879), [THE AI](https://www.newstheai.com/news/articleView.html?idxno=10327)); 원인 = 교사 업무 가중·무거운 도구 거부 ([focusnjn](https://focusnjn.com/article/1065591818096765))
- 여기까지는 '현상'이지 '문제'가 아니다 — "학생 생각을 가볍게 모아 함께 보는 도구가 없다"는 관찰된 현상일 뿐, 교사가 그것을 원하면서 못 해 겪는 **Pain Point**는 아직 미입증
- 우리가 세우는 가설(미검증·PoC 검증 대상): 무거운 도구에 데인 교사에게 가벼운 공동참여 도구의 미충족 니즈가 있고, canvas형 공동구성이 그 니즈를 충족 → 이 가설이 참일 때만 위 현상이 비로소 '문제'가 됨

## 2. 왜 지금?

- **디커플링 요소** — 1:1 기기 보급은 완료됐는데(경기 2025 상반기 약 120만 명 등, [시사저널](https://www.sisajournal.com/news/articleView.html?idxno=359113)), 그 자리를 채우려던 AIDT가 무게(교사 부담)로 무너짐 → 유휴 기기 + 무거운 도구에 데인 교사라는 빈 공간이 지금 열려 있음
- **퀴즈앤 철학과 정합** — 퀴즈앤이 검증한 두 철학 ① "교사가 가볍게 수업을 설계" ② "교사 주도로 학생을 참여시킴" 에 이 아이디어가 모두 정합; 무거움(AIDT)이 밀려난 자리를 그 가벼움·교사 주도성으로 메우는 타이밍

## 3. 제안

- 기기는 채워졌으나 그 위에서 가볍게 쓸 공동 참여형 도구가 부재하고, 기존 canvas(Miro·FigJam)는 비즈니스·영어·자유캔버스라 교실에 부적합 → 교사가 구조화된 틀(만다라트·마인드맵·클러스터링)을 깔면, 학생이 무계정으로 들어와 같은 캔버스 위에서 동시에 함께 항목을 놓고·묶고·연결하며 공동 구성하는 보드를 제안
- 이 프로덕트의 킥 — ① 30초 이내 원클릭 수업 템플릿 생성 ② 직관적 UI(별도 학습 없이 즉시 사용) ③ 타이머 설정(수업 리듬 제어)
- 단, 이건 '검증된 해결책'이 아니라 '가설적 제안'이다 — '공백이 있다'가 '교사가 이 canvas를 쓴다'를 보장하지 않음. 그 사이엔 "교사가 가벼운 공동참여를 원하고 canvas형이 그 답이다"라는 도약이 있으며, 이것이 PoC가 가장 먼저 검증할 핵심 가설
- 차별점 — canvas 인터랙션(가설) × 퀴즈앤 distribution × 한국 교실 현지화의 결합; 셋 중 하나만으론 성립 안 함, "가벼움"만은 Padlet·퀴즈앤 자체와 겹침

## 4. 벤치마킹

> wall형(Padlet·띵커벨·퀴즈앤 자체 보드)은 비교셋에서 제외 — 담벼락 카테고리라 직접 비교 대상이 아님. 비교셋 = canvas 카테고리.

- **[Miro](https://miro.com)** — 기업/디자인 맥락 최강; 무한 캔버스·기능과잉·영어우선 → 교실 30초 진입 불가
- **[FigJam](https://www.figma.com/figjam/)** — 협업 UX 탁월; Figma 계정 전제·서구 교사 중심 → 한국 K-12 온보딩 장벽
- **[알로(Allo)](https://allo.io/ko)** — 국산 유일 캔버스 시도, 자체 부검 4대 사인 "규모의 카오스·연결끊김·PM 확장불가·복잡도" ([알로 블로그](https://allo.io/blog/whiteboard-apps-are-dead-but-visual-collaboration-thrives-in-the-ai-era/)); 비즈니스 AI로 피벗하며 교실 포기 → 우리는 구조화 틀로 카오스 차단 + 교실은 프로젝트 실행 연결 불필요(알로 사인 ①④만 해당, ②③은 교실 맥락에서 무관)
- **Google Jamboard(종료 2024.12.31)** — 수요 증거 아닌 벤더 결정; 난민 발생 → 한국어·교실 맞춤 대체재 공백 확인
- **공통 gap** — 글로벌 canvas는 비즈니스·영어, 국산 canvas는 B2B로 이탈 → "한국 교실용 구조화 canvas + 퀴즈앤 distribution" 조합이 비어 있음

## 5. 시장 진입 전략

- **초기 진입 유저** — 퀴즈앤을 이미 쓰는 중·고 일반 교과 교사(이미 로그인·신뢰가 선 채널)
- **핵심 기능(MVP 범위)** — 구조화 틀 3종(만다라트·마인드맵·클러스터링) + 학생 무계정 QR/링크 입장 + 교사 대시보드(모둠 썸네일 갤러리) + 10명 실시간 공동편집 + 타이머
- **진입 흐름** — 퀴즈앤 Labs에서 신규 프로덕트 클릭 → 링크 이동·템플릿 팝업 → 템플릿 선택 후 보드 생성 → 학생 QR 입장 → 실시간 공동 구성 → 교사 읽기(분포·이상치·하이라이트)

## 6. 기대 효과

- **사용자 가치** — 교사: 준비 없이 30초에 켜는 공동 구성 활동 도구; 학생: 한 화면에서 함께 만드는 참여 경험
- **회사 가치** — 퀴즈앤 기존 교사 기반의 체류·재방문 강화; wall(기존 보드) vs canvas(v2) 제품 라인 분리로 TAM 확장
- **전략 연결** — AIDT 붕괴 후 유휴 기기 시장 선점; 장기 = 교실 협업 habit의 대학·기업 확장(단, 북극성 가설 — PoC 이후 검토)

## 7. 검증 계획

- **다음 행동** — 수요 검증용 PoC 제작 착수
- **PoC 성격** — 기능 빌드가 아니라 수요 검증: "교사가 도구를 받았을 때 공동 구성 활동을 반복 수업에 넣는가"를 가장 싸게 확인
- **착수 전 확정 — 초기 템플릿 3종 확정(현장 분석 필요)** — 만다라트·마인드맵·클러스터링 후보 중 PoC에 넣을 것을 교사 현장 분석으로 확정(브레인스토밍→분류는 근거 약함 → 비주얼씽킹 등 대안 검토)
- **1차 검증 질문** — 퀴즈앤 진입 교사 중 첫 세션 생성 10%(잠정) 달성하는가
- **2차(핵심) 검증 질문** — 첫 생성 교사의 2회+ 반복 생성률("반복 사용"이 수요의 진짜 신호); 기각 기준 = 퍼널 미미 + 재사용 한 자릿수 → distribution·수요 가설 재검토
- **필요 리소스** — PoC 1~3개 틀 × 10명 동시편집 엔진 + 교사 5~10명 실사용 인터뷰(러프)

## 8. 리스크 / 미해결 질문

- **핵심 수요 미입증(최대 리스크)** — "교실이 실시간 공동참여 canvas를 실제로 원하는가"는 desk research로 입증·반증 불가; 배경·제안에서 명시한 논리 도약이 곧 이 리스크 → 수요 검증 PoC가 유일한 해결책
- **자기잠식 경계** — 퀴즈앤 기존 '보드'(wall)와 v2(canvas)의 경계; wall을 다시 베끼면 사내 자기잠식이므로 canvas 차별화 유지 필요
