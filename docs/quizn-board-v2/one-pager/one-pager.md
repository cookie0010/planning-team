## 1. 배경 / 문제 (Problem)

- 한국 중·고 교사, 1:1 기기 보급 교실에서 **학생 전원의 생각을 가볍게 끌어내 한 화면에 모아 함께 보는** 도구가 비어 있음
- AIDT — 기기 보급의 명분 — 한 학기 만에 붕괴: **평균 활용률 8.1%, 1회도 미접속 학생 60%, 채택률 1학기 32% → 2학기 19% 급락** ([교육을비추다](https://www.kyobit.com/news/articleView.html?idxno=3879), [THE AI](https://www.newstheai.com/news/articleView.html?idxno=10327))
- 붕괴 원인 = **교사 업무 가중**: 기기 접속불량 해결·유해사이트 감시로 에너지 소모, 무거운 도구 거부 ([focusnjn](https://focusnjn.com/article/1065591818096765))
- canvas 수요 단일 수치는 `미확보` — 이것이 PoC로 검증해야 할 가설임을 정직하게 인정

## 2. 왜 지금? (Why Now)

- **디커플링 fact**: 컴퓨터 1대당 학생 수 **중 0.9 / 고 1.0** — 기기는 이미 깔렸다 ([서울경제](https://www.sedaily.com/article/20028437)); 경기 2025 상반기 초3~고3 약 120만 명 보급 마무리 ([시사저널](https://www.sisajournal.com/news/articleView.html?idxno=359113))
- **빈 공간 생성**: 유휴/저활용 기기 + 무거운 도구에 데인 교사 → AIDT가 채우지 못한 자리가 비어 있음
- **퀴즈앤 철학 차용**: "교사가 *가볍게* 수업을 설계" → "교사가 *가볍게* 학생을 참여시킴" — 가벼움이 AIDT를 밀어낸 그 자리를 파고드는 포지셔닝
- **(보조)** 2026 학운위 에듀테크 심의 의무화 + 만14세 규제 → 무계정 설계 우위; 단 **패키징(퀴즈앤 내부 모듈 vs 별도 제품)에 따라 심의 우산 성립 여부 가름** `[가설, 확정 전]`

## 3. 제안 / 해결책 (Proposed Solution)

- **원인**: 기기는 채워졌으나 그 위에서 가볍게 쓸 *공동 참여형* 도구가 부재; 기존 canvas(Miro·FigJam)는 비즈니스·영어우선·자유캔버스라 교실 부적합
- **핵심 해결책**: 교사가 구조화된 틀(만다라트·마인드맵·클러스터링)을 깔면, 학생이 무계정으로 들어와 **같은 캔버스 위에서 동시에 함께** 항목을 놓고·묶고·연결하며 공동 구성; 퀴즈앤 진입으로 준비 마찰 최소화
- **차별점**: canvas 인터랙션(가설, PoC 검증 대상) × 퀴즈앤 distribution × 한국 교실 현지화의 결합 — 셋 중 하나만으론 모트 불성립; "가벼움"만은 Padlet·퀴즈앤 자체와 겹침을 인지

## 4. 벤치마킹 (Benchmarking)

> wall형(Padlet·띵커벨·퀴즈앤 자체 보드)은 **의도적 제외** — 사내 자기잠식 방지. 비교셋 = canvas 카테고리.

- **[Miro](https://miro.com)**: 기업/디자인 맥락 최강; 무한 캔버스·기능과잉·영어우선 → 교실 30초 진입 불가
- **[FigJam](https://www.figma.com/figjam/)**: 협업 UX 탁월; Figma 계정 전제·서구 교사 중심 → 한국 K-12 온보딩 장벽
- **[알로(Allo)](https://allo.io/ko)**: 국산 유일 캔버스 시도, 자체 부검 4대 사인 → "규모의 카오스·연결끊김·PM 확장불가·복잡도" ([알로 블로그](https://allo.io/blog/whiteboard-apps-are-dead-but-visual-collaboration-thrives-in-the-ai-era/)); 비즈니스 AI로 피벗, 교실 포기 → **우리는 구조화 틀로 카오스 차단 + 교실은 프로젝트 실행 연결 불필요** (알로 사인 ①④만 해당, ②③은 교실 맥락에서 무관)
- **Google Jamboard (종료 2024.12.31)**: 수요 증거 아닌 벤더 결정; 난민 발생 → 한국어·교실 맞춤 대체재 공백 확인
- **공통 gap**: 글로벌 canvas는 비즈니스·영어; 국산 canvas는 B2B로 이탈 → **"한국 교실용 구조화 canvas + 퀴즈앤 distribution"** 조합 비어 있음

## 5. 시장 진입 전략 (How it works)

- **초기 진입 유저**: 퀴즈앤을 이미 사용 중인 중·고 일반 교과 교사 (이미 심의·인증된 채널)
- **핵심 기능 (MVP 범위)**: 구조화 틀 3종(만다라트·마인드맵·클러스터링) + 학생 무계정 QR/링크 입장 + 교사 대시보드(모둠 썸네일 갤러리) + 10명 실시간 공동편집
- **진입 흐름**: 퀴즈앤 진입 → 보드 생성(틀 선택) → 학생 QR 입장 → 공동 구성 → 교사 읽기(분포·이상치·하이라이트)

## 6. 기대 효과 (Why it matters)

- **사용자 가치**: 교사 — 준비 없이 30초 켜는 공동 구성 활동 도구; 학생 — 한 화면에서 함께 만드는 참여 경험
- **회사 가치**: 퀴즈앤 기존 교사 기반의 체류·재방문 강화; wall(기존 보드) vs canvas(v2) 제품 라인 분리로 TAM 확장
- **전략 연결**: AIDT 붕괴 후 유휴 기기 시장 선점; 장기 = 교실 협업 habit의 대학·기업 확장(단, 북극성 가설 — PoC 이후 검토)

## 7. 검증 계획 (Next Step)

- **다음 행동**: 수요 탐침 PoC 제작 착수 *(잠정 — 사용자 확인 필요)*
- **PoC 성격**: 기능 빌드가 아니라 수요 탐침 — "교사가 도구 받았을 때 공동 *구성* 활동을 *반복* 수업에 넣는가"를 가장 싸게 측정
- **1차 검증 질문**: 퀴즈앤 진입 교사 중 첫 세션 생성 **10%** *(잠정 — 사용자 확인 필요)* 달성하는가
- **2차(핵심) 검증 질문**: 첫 생성 교사의 **2회+ 반복 생성률** — "반복 사용"이 수요 탐침의 진짜 revealed signal; 기각 기준 = 퍼널 미미 + 재사용 교사 한 자릿수 → distribution·수요 가설 재검토
- **필요 리소스**: PoC 1개 틀(만다라트 또는 마인드맵) × 10명 동시편집 엔진 + 교사 5~10명 실사용 인터뷰 (러프 추산)

## 8. 리스크 / 미해결 질문 (Risks & Open Questions)

- **A1 (최대 리스크): 핵심 수요 미입증** — "교실이 실시간 공동참여 canvas를 실제로 원하는가"는 desk research로 입증·반증 불가 확정; 해석 B("wall이 이미 이겼다")의 반증도 0-3 폐기됐으나, 해석 A의 입증도 없음 → **PoC 수요 탐침이 유일한 해결책**
- **A2**: 10명 실시간 공동편집 카오스·무임승차 — 동시 자유발산은 개인보다 열위(1958~); 설계 차단(구조화 틀)이 이를 실제로 막는지 미검증; 동시성 N을 3~4명으로 축소 검토 여지
- **A3**: 퀴즈앤 내부 모듈 vs 별도 제품 패키징 미결 — 별도 제품이면 심의 우산 가설 무너짐; 상위 결정 보류 중
