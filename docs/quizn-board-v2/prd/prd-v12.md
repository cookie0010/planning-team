# PRD: quizn-board-v2

> Project: `quizn-board-v2`
> Date: 2026-06-17 / Version: v12

> **v11→v12 바뀐 점:** 사용자 확정 OQ 결정 묶음을 반영했다.
>
> 1. **OQ-15 해소(보관 캔버스 보존)** — 보관(archive)한 캔버스는 **사용자가 명시적으로 '삭제'하지 않는 한 사라지지 않는다**(보존 기간 제한·자동 만료·자동 삭제 없음, 언제든 재열람). 8절 보존·삭제 정책에서 보관 캔버스 자체를 영구 보존으로 확정. 개인정보(닉네임/비밀번호) 보존 세부만 잔여로 분리(OQ-23 신설).
> 2. **OQ-16 해소(공감 투표 세부)** — 공감 투표는 **포스트잇에만** 적용(다른 객체엔 없음). 한 포스트잇에 **1인 1표**(인당 1개 초과 불가, 자기 표는 토글로 내림). 이모지 등 추가 반응 레이어는 두지 않는다. F10 상세 확정.
> 3. **OQ-17 해소(편집자 권한 세부)** — 편집자 **QR 초대 링크는 발급 후 10분에 만료**, **임명 해제 기능 있음**. F6 상세에 반영. 소유자 이전 등 미명시 세부는 잔여로 분리(OQ-24 신설).
> 4. **OQ-18 해소(작성자 표기 방식) + 객체 잠금 신설** — 작성자는 **객체 클릭 시** 드러난다: 객체 클릭 → **객체 수정 옵션 바가 객체 상단에 노출**되고 **그 바 제일 우측에 작성자 프로필** 표시. 같은 옵션 바에 **객체 잠금**을 신설(잠긴 객체는 다른 사람이 수정·이동·삭제 불가). F13에 두 동작을 반영(작성자 표기 방식·객체 잠금). 객체 잠금은 OQ-13(동시 조작 충돌)을 부분 완화하는 장치임을 명시(단 OQ-13 자체는 계속 열어둠).
> 5. **OQ-21 해소(뷰어 공감)** — **뷰어는 공감 투표 불가**로 확정. 1절 뷰어 정의·F4·F10·완료 조건의 [잠정 — OQ-21] 표기를 '뷰어 공감 불가(확정)'로 교체.
> 6. **OQ-19 변경 없음** — 캔버스당 페이지 최대 30개(고정) 유지, 프로젝트당 캔버스 수는 계속 OQ-19로 열어둠.
>
> 그 외 본문은 v11 그대로다. (그룹=캔버스와 동급 선택 묶음, F7 드릴다운=편집 B 진입, F12 객체 컨트롤 권한, OQ-13 동시 조작 충돌=열림 — 모두 유지.)

## 0. 요약

**한국 교실용 실시간 협업 canvas 보드** — 학생들이 같은 캔버스 위에서 동시에 텍스트·그리기·포스트잇·도형으로 함께 만든다. 편집자(전형적으로 교사)는 하나의 프로젝트 안에 여러 캔버스를 만들고, 각 캔버스는 여러 페이지로 구성한다. 잘 만든 캔버스는 통째로 복제해 반별로 재사용하고, 캔버스끼리는 그룹으로 묶어 정리한다. 자기 화면에서 특정 캔버스를 크게 드릴다운해 보는 것이 곧 지휘다. 진입은 퀴즈앤 채널을 탄다 — 교사는 퀴즈앤에서 링크로 넘어와 30초 안에 수업 환경에 접속하고, 학생은 무계정 QR/URL로 들어온다.

> Why(배경·근거·차별화)는 one-pager 참조: `one-pager-v5.1.md`
> 캔버스 UI 레퍼런스(구속력 없음): `reference/` — FigJam·Miro·알로·미리캔버스 스크린샷

**포지셔닝 한 문장:** 퀴즈앤 기존 '보드'(wall — 각자 제출해 모아 붙이기)가 못 하는 것, 즉 **같은 캔버스 위에서 여러 명이 동시에 함께 조작하는 공동 구성**(canvas)을 제공한다.

**이 빌드의 성격:** [x] PoC/실험 — 목적은 기능 완성이 아니라 **"교사가 실시간 공동참여 canvas를 수업에 반복해서 쓰는가"(수요)의 검증**이며, 성패는 6절 지표로 판정한다.

> **제품 명칭:** "board-v2"는 가명(코드네임)이다. 정식 명칭은 미정 — 로고 영역에는 우선 'board-v2' 텍스트로 표기한다.

## 1. 대상 사용자 및 권한

교실 맥락의 기본 그림: 교사 = 소유자, 학생 = 참여자. 권한 구조 자체는 역할 일반형(소유자/편집자/참여자/뷰어)으로 설계한다.


| 역할  | 정의                                   | 계정 요건                                                                   | 핵심 권한                                                                                            |
| --- | ------------------------------------ | ----------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| 소유자 | 프로젝트 생성자 (전형적으로 교사)                  | **로그인 필수** (퀴즈앤 계정)                                                     | 프로젝트 전체 제어, 캔버스 생성·복제, 편집자 임명·해제, 닉네임/비밀번호·익명 옵션 설정, 권한별 QR/URL 발급, **모든 객체 컨트롤·잠금**             |
| 편집자 | 소유자 또는 다른 편집자가 임명, 또는 편집자 QR/URL로 초대 | **로그인 필수** — QR/URL로 초대되어도 입장하려면 반드시 로그인                                | 임명된 **캔버스**의 페이지 추가·템플릿 설정·캔버스 편집, 편집자 임명·해제, 닉네임/비밀번호·익명 옵션 설정, 권한별 QR/URL 발급, **모든 객체 컨트롤·잠금** |
| 참여자 | 캔버스에 입장한 사람 (전형적으로 학생)               | 불필요 — 무계정 입장 가능(로그인해도 됨). **비로그인 입장 시 닉네임 + 비밀번호 4글자 필수**, 로그인 시 재진입 혜택 | 편집자가 설정해 둔 페이지에서 캔버스 작업(객체 생성·이동·편집), 포스트잇 공감 투표(1인 1표). **단 객체 컨트롤(수정·이동·삭제)·잠금은 본인이 작성한 객체만**  |
| 뷰어  | 캔버스에 입장해 **보기만** 하는 사람               | 불필요 — 무계정 입장 가능(로그인해도 됨). **비로그인 입장 시 닉네임 + 비밀번호 4글자 필수**, 로그인 시 재진입 혜택 | 캔버스를 **보기 전용**. 객체 생성·이동·편집·컨트롤·잠금 불가, 페이지 변경 불가, **공감 투표 불가(확정 — OQ-21 해소)**                    |


**참여자·뷰어 식별 규칙:**

- 로그인 사용자(참여자·뷰어 모두): 본인이 참여했던 캔버스에 언제든 재진입 가능. 뷰어로 참여했던 캔버스에는 보기 전용으로 재진입한다.
- 비로그인 사용자: **닉네임 + 비밀번호 4글자(고정)**를 입력해 입장한다. 동일한 닉네임/비밀번호를 다시 입력했을 때만 동일 사용자로 취급. 그렇지 않으면 진입 당시 브라우저 세션별로 분리되어 다른 사용자로 인식.
- 비밀번호는 **4글자 고정**. 닉네임+비밀번호4는 비로그인 입장의 **필수 식별 정보**다(옵션 토글이 아님 — v7의 "옵션 사용 여부 조정"에서 변경).
- **중복 닉네임(OQ-11 해소):** 닉네임은 **같은 캔버스 안에서 유일**하다. 이미 같은 닉네임을 쓰는 사람이 있는 캔버스에 같은 닉네임으로 입장하려 하면 **"이미 같은 닉네임을 쓰는 사람이 있어요" 식 안내를 띄워 차단**하고 다른 닉네임을 입력하게 한다(닉네임이 같으면서 비밀번호가 다른 동시 사용자를 막는다).
- **비밀번호 분실(OQ-11 해소):** 비회원(참여자·뷰어)의 **비밀번호 찾기·재설정 장치는 두지 않는다**. 비로그인 사용자가 닉네임·비밀번호를 잊으면 그 식별로의 복구 수단은 없다(같은 닉네임/비밀번호4를 다시 입력해야만 동일 사용자로 복원되므로, 잊으면 새 식별로 입장하게 된다).

**역할별 권한 경계:**

- 편집자: 페이지 추가 생성, 각 페이지에 템플릿 설정, 캔버스 편집이 가능하다. 닉네임/비밀번호·익명 옵션을 설정한다. **편집자는 로그인 필요 역할**이므로, 편집자 QR/URL로 초대받아 들어와도 로그인해야 편집자로 입장한다(F5).
- 참여자: 템플릿 수정·페이지 추가는 **불가**. 편집자가 설정해 놓은 페이지에서 캔버스 작업(객체 생성·이동·편집)과 포스트잇 공감 투표(1인 1표)가 가능하다. 비로그인 입장 시 닉네임+비밀번호4로 식별.
- 뷰어: **보기 전용**. 객체 생성·이동·편집 불가, 페이지 변경 불가, **공감 투표 불가**. 참여자와의 차이는 — 참여자는 객체를 만들고 옮기고 편집하며 공감 투표도 하지만, 뷰어는 캔버스에 들어와 **보기만** 한다. **뷰어는 포스트잇 공감 투표도 할 수 없다(확정 — OQ-21 해소).** 비로그인 입장 시 닉네임+비밀번호4로 식별.

**객체 컨트롤 권한 규칙 (OQ-13 해소 — 객체 컨트롤 권한):**

- **소유자·편집자**: 누가 만든 객체든 **모든 객체를 컨트롤**(수정·이동·삭제)할 수 있다. 또한 **모든 객체를 잠글 수 있다**(F13 객체 잠금).
- **참여자**: **본인이 작성한 객체만** 컨트롤(수정·이동·삭제·잠금)할 수 있다. 다른 사람이 만든 객체는 보기·공감 투표만 되고 옮기거나 지울 수 없다.
- **뷰어**: 객체 컨트롤·잠금 불가(보기 전용 — 위 권한 경계와 동일).
- 소유자·편집자는 각 객체의 **작성자·수정자가 누구인지 객체 단위로 식별**할 수 있어야 한다(누가 만들고 누가 고쳤는지). 표기·동작 방식은 아래 작성자 가시성 규칙과 F13(객체 클릭 시 작성자 표시·객체 잠금)로 구현한다.

**작성자 가시성 규칙 (OQ-18 해소):**

- 객체(포스트잇 등)의 작성자는 **객체를 클릭했을 때** 드러난다. 객체를 클릭하면 **객체 수정 옵션 바가 객체 위(상단)에 노출**되고, 그 **옵션 바의 제일 우측에 작성자(사용자) 프로필**이 표시된다. (상시 표시·호버가 아니라 '클릭 시 수정 옵션 바 + 우측 작성자 프로필'로 확정.)
- 소유자·편집자는 익명 옵션과 무관하게 이 방식으로 작성자를 **항상** 확인할 수 있다.
- **익명 옵션**을 켜면 참여자 간에는 작성자 비공개 — 그래도 소유자·편집자에게는(옵션 바 우측 프로필로) 보인다. 옵션은 소유자·편집자가 조정.

**편집자 임명·초대 규칙 (OQ-17 해소 — 링크 만료·임명 해제):** 임명·초대 단위 = **캔버스**. 편집자가 되는 경로는 둘이다 — ① 소유자·편집자가 직접 임명, ② **편집자 권한으로 발급한 QR/URL로 초대**(공유 패널에서 '편집자'를 선택해 발급한 링크). 어느 경로든 편집자로 입장하려면 **반드시 로그인**한다. 임명·초대되면 그 캔버스의 페이지 추가·템플릿 설정·편집이 가능하다.

- **편집자 QR 초대 링크는 발급 후 10분에 만료**된다(만료 후 그 링크로는 편집자 입장 불가, 재발급 필요).
- **편집자 임명 해제 기능이 있다** — 소유자·편집자는 임명한 편집자를 다시 해제할 수 있다(HTML 와이어프레임에 반영됨).

**공유·입장 권한 규칙 (OQ-22 확정):**

- 캔버스 공유 시(QR/URL 발급, F2·F9) **초대할 권한을 먼저 선택**한다 — 편집자 / 참여자 / 뷰어. 선택한 권한에 해당하는 **별개의 QR·URL**이 발급·표시된다(권한별로 링크가 다르다).
- **편집자 링크**로 들어오면 로그인 게이트를 거쳐 편집자로 입장한다(로그인 필수). 편집자 초대 링크는 발급 후 **10분에 만료**된다(OQ-17 해소).
- **참여자 링크 / 뷰어 링크**로 들어오면 로그인 없이 입장 가능하며, 비로그인 시 닉네임+비밀번호4로 식별한다.
- **뷰어도 캔버스 정원(20명)에 포함**한다 — 정원은 편집자·참여자·뷰어 모두를 센다. 정원 초과 시 뷰어도 입장 불가 안내를 받는다(F5).
- **참여자 입장 승인(F13, OQ-14 해소):** 참여자가 캔버스에 접근할 때 **기본은 수동 승인**이며, 소유자·편집자가 **자동 승인 옵션**으로 바꿀 수 있다. 승인 대기가 여럿이면 **'모두 승인' 버튼**으로 한 번에 받는다.

**학생 대상:** **초·중·고** 학생, 태블릿·크롬북·스마트폰.

**범위 밖 사용자:** 대학·기업, 퀴즈앤 미사용 신규 교사(PoC 기간은 기존 퀴즈앤 채널로 한정).

## 2. 핵심 개념 모델 및 용어

```
프로젝트 (최상위 컨테이너 — 소유자가 생성. 권한·관리의 큰 틀)
 └── 캔버스 (핵심 단위 — 프로젝트 바로 아래. 권한별 QR/URL 발급, 학생 진입 단위, 진입자 최대 20명(뷰어 포함), 최대 30페이지)
      └── 페이지 1~30장 (캔버스의 작업 면 — 편집자가 템플릿을 붙여 구성, 좌우로 추가·이동, 유한)
           └── 객체 (텍스트·그리기·포스트잇·도형 등 기본 도구로 만드는 것 — 클릭 시 수정 옵션 바·작성자·잠금)

  그룹 (선택 — 프로젝트 안에서 캔버스들을 묶는 폴더/라벨. 캔버스와 같은 레벨이며 상위 계층이 아니다. 안 묶어도 됨)
```

- **프로젝트**: 최상위 컨테이너. 소유자가 생성하며, 그 안에서 캔버스들을 만든다. 권한·관리가 걸리는 큰 틀이다. **PoC에선 사실상 단일 프로젝트로 운영**한다 — 다중 프로젝트 생성·전환·관리 UI는 범위 밖이며, 여기서는 개념적 분리만 둔다.
- **캔버스(핵심 단위)**: 프로젝트 바로 아래의 핵심 단위다. 여러 페이지로 구성된 하나의 작업 공간이자 학생이 들어오는 진입 단위. 캔버스는 프로젝트 안에 바로 존재하며 **그룹에 묶이는 것은 선택**이다(그룹 없이도 존재 가능). 공유 시 **초대 권한(편집자/참여자/뷰어)을 선택해 권한별 QR·URL을 발급**한다. **진입자 최대 20명(편집자·참여자·뷰어 모두 정원에 포함)**이다. 편집자 임명·닉네임/비밀번호·익명 옵션이 이 단위에 묶인다. 상태 = 진행 중 / 잠금(타이머 만료) / 보관. **보관 상태의 캔버스는 사용자가 명시적으로 삭제하지 않는 한 영구 보존된다(자동 만료·자동 삭제 없음, 언제든 재열람 — OQ-15 해소).**
- **그룹(선택 — 캔버스를 묶는 단위, 상위 계층 아님)**: 프로젝트 안에서 **캔버스들을 묶는 선택적 폴더/라벨**이다(예: "1차시", "모둠 발표", "2026 1학기"). 캔버스 목록을 정리하는 수단일 뿐이며, **캔버스의 상위 계층이 아니라 캔버스와 같은 레벨**이다 — 캔버스는 그룹에 들어가 있을 수도, 어느 그룹에도 묶이지 않고 프로젝트 바로 아래 놓일 수도 있다. 그룹 카드 안에 속한 캔버스 썸네일들이 보이고 그룹명 라벨이 붙는다.
- **페이지**: 캔버스의 작업 면. 편집자가 템플릿을 붙여 구성한다(예: 템플릿을 이것저것 붙여 10페이지짜리 캔버스를 완성). 좌우로 페이지를 추가하고 이동할 수 있다(미리캔버스·알로 참고). 무한 줌 월드가 아니라 **유한한 페이지**다. **캔버스당 페이지는 최대 30개(고정)** — 그 이상은 추가할 수 없다.
- **템플릿**: 페이지에 깔리는 객체 묶음(캔버스 기본 도구들을 조합해 만든 틀이며 별도 엔진이 아니다). 편집자가 페이지별로 설정한다. **기본값 = 빈 캔버스**(OQ-03 해소). 선택지 = **빈 캔버스 · 브레인스토밍 · 마인드맵**.
- **객체**: 페이지 위에 만들고 옮기는 것 — 텍스트, 그리기(펜), 포스트잇, 도형 등 일반 캔버스 도구가 만드는 요소(FigJam·Miro·알로·미리캔버스 참고). 모든 객체에는 작성자(및 수정자) 정보가 남는다(1절 가시성·객체 컨트롤 권한 규칙). 객체를 클릭하면 **객체 상단에 수정 옵션 바**가 뜨고(제일 우측에 작성자 프로필), 그 바에서 **객체 잠금**도 할 수 있다(F13). **공감 투표는 포스트잇 객체에만 적용된다(다른 객체엔 없음 — OQ-16 해소).**
- **복제**: 잘 만든 **캔버스** 하나를 통째로 복제해 재사용하는 것(예: 1반·2반·3반용으로 복제). 복제본은 같은 프로젝트 안의 독립된 캔버스이고 각자 다른 QR을 가진다. 복제본끼리 내용은 동기화되지 않는다.

**용어 금지/통일:** 이 문서에서 "보드" 단독 사용 금지 — 사내에서 '보드'는 퀴즈앤 기존 wall 제품을 가리킨다. **프로젝트 / 그룹 / 캔버스 / 페이지 / 객체**로 통일. 단 위계상 **캔버스가 핵심 단위**이고 **그룹은 캔버스를 묶는 선택 수단**임을 혼동하지 말 것(그룹은 캔버스의 상위 통이 아니다).

### 기능 상세

**F1. 캔버스 생성·관리** *(소유자·편집자)*

- **필수 정보**: 프로젝트 안 캔버스 목록(이름·생성일·페이지 수, 그룹 소속 여부), 캔버스별 설정(편집자 목록, 닉네임/비밀번호·익명 옵션), 캔버스별 **권한별 진입 QR·URL**(편집자/참여자/뷰어)
- **필수 행동**:
  - **캔버스 생성**: 두 경로 모두 같은 생성 흐름을 탄다 — ① 퀴즈앤 링크로 처음 진입할 때 템플릿 선택 모달이 자동으로 뜨는 경로(F9), ② 캔버스 목록 화면에서 명시적인 **'캔버스 생성'** 액션을 눌러 시작하는 경로. 어느 쪽이든 **템플릿 선택 화면에서 빈 캔버스(기본값) 또는 템플릿 3종 중 선택**(F3)해 캔버스를 만든다.
  - 캔버스 이름 지정·삭제, 페이지 구성(F4, 최대 30페이지), 편집자 임명·초대·해제(F6), 옵션 설정, 공유 패널에서 **권한(편집자/참여자/뷰어) 선택 후 QR/URL 발급·재표시**(F6)
- **품질 기준**: 진입 후 첫 캔버스 QR 발급까지 3액션·30초 이내(실측 동선은 F9 참조)
- **하지 않는 것**: 캔버스 간 내용 동기화(복제본은 서로 독립 — F2) · 다중 프로젝트 생성·전환·관리 UI(2절 — PoC 범위 밖)

**F2. 캔버스 복제** *(소유자·편집자)*

- **필수 정보**: 복제 대상 캔버스, 복제본 목록(각자 권한별 QR·URL)
- **필수 행동**: 잘 만든 캔버스 하나를 통째로 복제(예: 1반·2반·3반용) · 복제본은 같은 프로젝트 안에서 각각 독립된 캔버스로 생성되고 각자 다른 QR을 가짐 · 복제본도 공유 시 권한(편집자/참여자/뷰어)을 선택해 권한별 QR/URL을 발급(F6)
- **플로우/상태**: 복제 즉시 진행 중 + (권한 선택 시) QR 표시 · 복제 횟수·프로젝트당 캔버스 수 상한은 미정(OQ-19) — 정해지기 전에는 임의 제한을 두지 않음
- **하지 않는 것**: 복제본 간 내용 동기화(복제본은 서로 독립) · 프로젝트 통째 복제(별도 컨테이너 단위 복제 수요는 OQ-19에서 함께 판단 — 임의로 만들지 않음)

**F3. 템플릿 3종** *(편집자가 페이지별 설정)*

- **필수 정보**: 템플릿 선택지 3종과 미리보기 — **빈 캔버스(기본값) / 브레인스토밍 / 마인드맵**. 새 캔버스·새 페이지 모달의 **기본 선택값 = 빈 캔버스**(OQ-03 해소)
- **필수 행동**: 캔버스 생성 시(F1) 그리고 편집자가 페이지를 추가할 때 템플릿을 설정하면 해당 객체 묶음이 그 페이지에 깔린다. 템플릿 = 캔버스 기본 도구들을 조합해 만든 틀(별도 엔진 아님)
- **하지 않는 것**: 사용자 커스텀 템플릿 제작·저장(후속). ⚠️ 브레인스토밍 템플릿은 '전원 동시 발산'이 아니라 공동 분류·구조화 단계에 초점(동시 자유발산의 열위 근거 — `working-notes.md`)

**F4. 캔버스 공동 편집** *(참여자·소유자·편집자 / 뷰어는 보기 전용)*

- **필수 정보**: 같은 캔버스 사용자들의 편집이 실시간 반영된 페이지, 페이지 위치(N/M, M ≤ 30), 잔여 시간(F8), 객체 작성자 정보(1절 가시성 규칙에 따라 권한별 노출 — 객체 클릭 시 옵션 바 우측 프로필)
- **필수 행동**:
  - 참여자: 편집자가 설정해 둔 페이지에서 기본 도구로 객체 생성·이동·편집 — **텍스트 입력, 그리기(펜), 포스트잇, 도형** 등 일반 캔버스 기본 기능(FigJam·Miro·알로·미리캔버스 수준 참고). **단 수정·이동·삭제·잠금은 본인이 작성한 객체만 가능**(F12). 페이지 추가·템플릿 수정은 불가.
  - 편집자: 위에 더해 페이지 좌우 추가·이동(최대 30페이지까지), 각 페이지에 템플릿 설정. **누가 만든 객체든 모두 컨트롤·잠금 가능**(F12·F13).
  - 뷰어: **보기 전용** — 같은 캔버스의 편집이 실시간 반영되는 화면을 보기만 한다. 객체 생성·이동·편집 불가, 페이지 변경(추가·이동) 불가, **공감 투표 불가(확정 — OQ-21 해소)**.
- **플로우/상태**: 진행 중(편집 가능) / 잠금(보기만 가능, 객체 유지). 뷰어는 진행 중·잠금 어느 상태에서도 보기만 한다.
- **품질 기준**: 한 사용자의 편집이 같은 캔버스 전원(뷰어 포함) 화면에 **2초 이내** 반영 · 동시 조작에서 객체 유실·중복 0건
- **하지 않는 것**: 다른 캔버스 열람·편집(참여자·뷰어 기준 — 소유자·편집자는 F7로 접근)

**F5. 입장·사용자 식별** *(편집자·참여자·뷰어)*

- **필수 정보**: 입장 화면 — 캔버스 이름. 권한별로 분기:
  - 편집자 링크: 로그인 화면(또는 로그인 유도). 만료(10분 초과)된 링크면 만료 안내.
  - 참여자·뷰어 링크: 닉네임 입력란 + 비밀번호 4글자 입력란(비로그인 입장 시 필수)
- **필수 행동**: QR 스캔 또는 URL 접속 → **링크에 담긴 권한에 따라 분기**:
  - **편집자 QR/URL**: 로그인 → 편집자로 입장(로그인 필수 — 비로그인은 편집자로 입장 불가). 링크는 발급 후 **10분에 만료**, 만료 시 입장 불가 안내(재발급 필요 — OQ-17 해소).
  - **참여자·뷰어 QR/URL**: 로그인 없이 입장 가능. 비로그인 시 **닉네임 + 비밀번호 4글자 입력** → 해당 권한으로 입장. 로그인 사용자(참여자·뷰어 모두)는 닉네임/비밀번호 없이 입장하고 참여했던 캔버스에 언제든 재진입한다(뷰어로 참여했던 캔버스는 보기 전용으로 재진입).
  - **중복 닉네임 차단(OQ-11 해소)**: 같은 캔버스 안에 같은 닉네임이 이미 있으면 **"이미 같은 닉네임을 쓰는 사람이 있어요" 식 안내를 띄우고 입장을 막아** 다른 닉네임을 입력하게 한다(닉네임은 캔버스 내 유일).
  - **참여자 입장 승인(F13)**: 참여자가 입장하려 하면 기본은 **수동 승인 대기** — 소유자·편집자가 승인해야 입장. 자동 승인 옵션을 켜면 바로 입장, '모두 승인' 버튼으로 대기자를 한 번에 받는다.
- **플로우/상태**: 비로그인 + 동일 닉네임/비밀번호4 = 동일 사용자로 복원 / 그 외 = 브라우저 세션별 별도 사용자 · **정원(20명, 뷰어 포함) 초과 시 입장 불가 안내**(편집자·참여자·뷰어 모두 정원에 셈) · 잠금·보관 캔버스 접속 시 안내 화면
- **비밀번호 분실(OQ-11 해소)**: 비회원(참여자·뷰어) **비밀번호 찾기·재설정 장치는 없다**. 분실 시 동일 식별로의 복구 수단은 제공하지 않는다.
- **품질 기준**: 스캔→진입 로딩 3초 이내(4G+) · 참여자·뷰어는 무설치·무가입(로그인은 선택) · 편집자는 로그인 필수
- **하지 않는 것**: 참여자·뷰어에게 가입·로그인 강제, 닉네임·비밀번호4 외 개인정보 수집

**F6. 권한 관리** *(소유자·편집자)*

- **필수 정보**: 캔버스별 편집자 목록, 닉네임/비밀번호·익명 옵션 상태, **공유 패널의 권한 선택 상태와 권한별(편집자/참여자/뷰어) QR·URL**, 참여자 입장 승인 모드(수동/자동) 상태
- **필수 행동**:
  - **공유·권한별 QR/URL 발급**: 캔버스 공유 시 **초대할 권한을 먼저 선택(편집자/참여자/뷰어)**하면 그 권한에 해당하는 QR·URL이 발급·표시된다(권한별 별개 링크). **편집자 초대 링크는 발급 후 10분에 만료**된다(OQ-17 해소).
  - **편집자 임명·초대·해제**: 소유자·편집자가 로그인 사용자를 그 캔버스의 편집자로 직접 임명하거나, **'편집자' 권한으로 발급한 QR/URL로 초대**한다. 편집자로 입장하려면 반드시 로그인(F5). 임명·초대된 편집자는 해당 캔버스의 페이지 추가·템플릿 설정·편집 가능, 다른 편집자 임명 가능. **임명한 편집자를 다시 해제할 수 있다**(임명 해제 기능 — OQ-17 해소, HTML 와이어프레임에 반영).
  - **옵션 제어**: 익명 옵션 on/off(켜면 참여자 간 작성자 비공개, 소유자·편집자는 항상 확인). 비로그인 입장의 닉네임+비밀번호4는 **항상 필수**(토글 아님). **참여자 입장 승인 모드**(기본 수동 / 자동) 전환(F13).
- **뷰어 관리**: 뷰어(보기 전용) 입장은 **'뷰어' 권한으로 발급한 QR/URL**로 들어오는 경로로 확정(OQ-22 해소). 뷰어도 정원(20명)에 포함된다.
- **하지 않는 것**: 페이지 단위 개별 권한, 읽기 전용 공유 링크의 별도 체계화(뷰어 링크가 사실상 보기 전용 입장 경로를 담당 — 추가 공유 링크 유형 수요는 후속) · 소유자 권한 이전(잔여 — OQ-24)

**F7. 캔버스 드릴다운 (지휘)** *(소유자·편집자)*

- **개념**: 별도의 전체 공유 화면이나 broadcast는 없다. 교사는 캔버스 목록에서 보고 싶은 캔버스를 **자기 화면에 크게 열어(드릴다운)** 들여다보는 것으로 지휘한다. 캔버스 목록 화면이 곧 지휘의 출발점이다.
- **드릴다운 = 편집 화면(B) 진입 (확정)**: 교사가 막힌 모둠을 '들여다보는 것'은 그 캔버스의 **편집 화면(B)에 들어가는 것과 같은 동작**이다. 캔버스 목록에서 카드를 누르면 그 캔버스가 자기 화면에 크게 열리고, 그것이 곧 편집 진입이다. **별도의 읽기 전용 확대 열람 모드는 두지 않는다** — 드릴다운과 편집은 하나의 동작이다.
  - '끼어들지 않고 보기만' 하고 싶을 때(학생 작업을 멈추고 들여다보기)는 **잠금(F8)**으로 흡수한다. 캔버스를 잠근 채로 열어 보면 학생 편집이 멈춰 있는 상태에서 들여다볼 수 있다. 보기 전용 드릴다운을 위한 별도 장치는 불필요하다.
  - **지휘 대상 범위는 교사가 보고 있는 목록 화면의 범위로 자연 결정된다** — 프로젝트 전체 목록에서 들어가면 프로젝트 전체가, 한 그룹 안 목록에서 들어가면 그 그룹이 지휘 대상이다. 지휘 대상을 따로 고르는 장치는 PoC 범위 밖이다(OQ-20 해소).
- **필수 정보**: 캔버스 목록(썸네일, 편집 후 2초 이내 갱신), 캔버스별 참여 인원·객체 수, 드릴다운(편집 화면 B)으로 연 캔버스의 **객체별 작성자**(객체 클릭 시 옵션 바 우측 프로필, 익명 옵션과 무관하게 소유자·편집자에게 항상 표시)
- **필수 행동**: 목록에서 특정 캔버스를 크게 열어 보기(= 편집 화면 B 진입), 타이머·잠금 제어, **객체 클릭 시 수정 옵션 바로 작성자 확인**(F13)
- **품질 기준**: 캔버스 6개 기준 목록 한 화면에서 스크롤 없이 전체 파악 가능
- **하지 않는 것**: 선택 캔버스를 별도 전체 공유 화면으로 송출(broadcast — 범위 밖) · 채점·참여자별 기여도 표시(읽기 = 분포 파악·막힌 곳 돕기·좋은 사례 하이라이트) · 편집과 구분되는 별도의 읽기 전용 확대 열람 모드(드릴다운 = 편집 진입으로 확정)

**F8. 타이머** *(소유자·편집자)*

- **필수 행동**: 캔버스에 제한 시간 설정, 시작·일시정지·연장·종료 · 만료 시 해당 캔버스 편집 자동 잠금(객체 유지), 재개방 가능
- **품질 기준**: 잔여 시간이 교사 화면·참여자 화면에 동기 표시, 오차 1초 이내

**F9. 퀴즈앤 진입 동선·퍼널 계측** *(distribution)*

- **진입 방식**: 퀴즈앤 서비스에서 **링크 이동**으로 board-v2에 진입한다. board-v2는 **데이터 관리·도메인이 퀴즈앤과 완전 별개**이므로, v2 내부의 진입점 위치 설계는 우리 범위 밖이며 퀴즈앤은 링크만 연결한다.
- **첫 사용 동선**: 링크 진입 → board-v2 화면으로 이동하면서 동시에 '템플릿 생성' 모달 자동 오픈 → 빈 캔버스 또는 템플릿 3종 중 선택 시 즉시 캔버스 진입 → 캔버스 안에서 **공유 패널을 열어 초대 권한(편집자/참여자/뷰어)을 선택하고 그 권한의 QR을 발급**(실측 2액션 + 권한 선택 1액션). 이후 캔버스 목록으로 돌아오면 명시적 '캔버스 생성' 액션으로 같은 흐름을 다시 탈 수 있다(F1).
- **필수 행동**: 진입 → 캔버스 생성/복제 → (권한 선택 후 QR 발급) → 참여자 입장 → 재사용, 각 단계 이벤트 계측(소유자 계정+캔버스 ID 키, 비로그인 참여자는 디바이스 근사)
- **품질 기준**: 6절 지표 산출에 필요한 전 이벤트가 빠짐없이 수집될 것

**F10. 포스트잇 공감 투표** *(참여자·소유자·편집자 / 뷰어 불가)*

- **적용 범위**: 공감 투표는 **포스트잇 객체에만** 적용된다(OQ-16 해소). 텍스트·도형·그리기 등 다른 객체에는 공감 투표가 없다.
- **필수 정보**: 포스트잇별 공감 수
- **필수 행동**: 다른 사람의 포스트잇에 공감을 남김·취소. **한 포스트잇에 1인 1표** — 한 사람이 같은 포스트잇에 1개를 초과해 줄 수 없고, 자기 표는 **토글로 내려(취소)** 0으로 되돌릴 수 있다(OQ-16 해소).
- **뷰어(OQ-21 해소)**: 뷰어는 **공감 투표 불가**(보기 전용). 확정.
- **품질 기준**: 공감 반영 2초 이내(같은 캔버스 전원)
- **하지 않는 것**: 댓글(영구 제외 — 7절 비목표 1) · 이모지 등 추가 반응 레이어(두지 않음 — 포스트잇 공감 1표로 한정, OQ-16 해소) · 포스트잇 외 객체에 대한 공감

**F11. 그룹 관리** *(소유자·편집자)*

- **개념**: 그룹은 **캔버스의 상위 계층이 아니라**, 프로젝트 안에서 **캔버스들을 묶는 선택적 폴더/라벨**이다(캔버스와 같은 레벨 — 2절). 캔버스는 그룹에 넣지 않아도 프로젝트 바로 아래 존재한다.
- **필수 정보**: 캔버스 목록 화면의 그룹 카드(그룹명 라벨, 소속 캔버스 썸네일들)
- **필수 행동**: 프로젝트 안에서 그룹 생성·이름 지정(예: "1차시", "모둠 발표"), 캔버스를 그룹에 넣고 빼기, 그룹 단위 정리. 캔버스를 어느 그룹에도 넣지 않은 채 둘 수도 있다.
- **하지 않는 것**: 그룹 단위 일괄 권한·일괄 설정(후속) · 프로젝트당 캔버스 수·그룹당 캔버스 수 상한은 미정(OQ-19)

**F12. 객체 컨트롤 권한** *(전 역할 — 객체 단위)*

- **개념**: 객체(포스트잇·텍스트·도형·그리기)를 **누가 수정·이동·삭제·잠금할 수 있는가**의 규칙(1절 객체 컨트롤 권한 규칙, OQ-13 해소). 객체 생성 권한(참여자가 새 객체를 만드는 것)과 별개로, 이미 놓인 객체를 **컨트롤(수정·이동·삭제)·잠금**하는 권한을 정한다.
- **필수 행동**:
  - **소유자·편집자**: 작성자가 누구든 모든 객체를 수정·이동·삭제·잠금할 수 있다.
  - **참여자**: **본인이 작성한 객체만** 수정·이동·삭제·잠금할 수 있다. 다른 사람 객체는 보기·공감 투표만 가능, 옮기거나 지울 수 없다.
  - **뷰어**: 객체 컨트롤·잠금 불가(보기 전용).
- **필수 정보**: 각 객체의 작성자(및 수정자)가 누구인지 — **소유자·편집자는 객체 단위로 식별 가능해야 한다**. 식별은 객체 클릭 시 수정 옵션 바 우측 작성자 프로필로 드러난다(F13, OQ-18 해소).
- **하지 않는 것**: 참여자가 다른 참여자 객체를 컨트롤(수정·이동·삭제) · 객체별 개별 권한 부여(역할 기준으로만 결정)

**F13. 편집 통제 (카오스 가드레일)** *(소유자·편집자, 일부 전 역할)*

- **개념**: 실시간 동시 편집의 카오스를 교사가 통제하기 위한 묶음(OQ-14 해소, A3 가드레일의 구체화). 아래 동작들로 구성한다.
- **필수 행동**:
  - **참여자 커서 끄기** *(P0)*: 참여자의 마우스 커서(화살표)는 기본적으로 캔버스에 실시간 표시된다. 소유자·편집자는 이를 **끌 수 있다**(커서가 많아 화면이 어지러울 때 카오스를 줄인다). 커서 표시 on/off는 소유자·편집자가 제어한다. — **P0 근거**: 동시 편집 카오스 억제가 A3 핵심 가정의 검증 조건이고, 다수 참여자 환경에서 가장 먼저 부딪히는 산만함이라 PoC에서 바로 필요.
  - **객체 클릭 시 수정 옵션 바 — 작성자 표시·객체 잠금** *(P0)*: 객체를 클릭하면 **객체 위(상단)에 수정 옵션 바**가 뜬다. 이 옵션 바의 **제일 우측에 작성자(사용자) 프로필**이 표시되어 어느 사용자가 만든 것인지 확인할 수 있다(OQ-18 해소). 소유자·편집자에게는 익명 옵션과 무관하게 항상 보인다(F12의 작성자 식별을 동작으로 구현). — **P0 근거**: 객체 컨트롤 권한(F12)과 교사의 '막힌 곳 돕기·하이라이트' 동선이 작성자 식별에 직접 의존하므로 함께 들어가야 한다.
    - **객체 잠금** *(P0)*: 같은 수정 옵션 바에서 객체를 **잠글 수 있다**. **잠긴 객체는 다른 사람이 수정·이동·삭제할 수 없다.** 잠글 수 있는 주체 = 그 객체를 컨트롤할 수 있는 사람(소유자·편집자 = 모든 객체 / 참여자 = 본인 작성 객체, F12 정합). — **객체 잠금은 OQ-13(같은 객체 동시 조작 충돌)을 부분 완화하는 장치**다(잠가 두면 그 객체에 대한 동시 조작 충돌 자체가 줄어든다). 단 동시 조작 충돌 처리 일반(두 사람이 잠기지 않은 같은 객체를 동시에 건드릴 때 무엇을 보여줄지)은 별개로 OQ-13에서 계속 다룬다. — **P0 근거**: 작성자 표시와 같은 옵션 바에서 제공되는 통제 동작이고, 교사가 완성·중요 객체를 학생 실수로부터 보호하는 데 즉시 쓰인다.
  - **참여자 입장 승인** *(P1)*: 참여자가 캔버스에 접근할 때 **기본 = 수동 승인**(교사가 승인해야 입장), **자동 승인 옵션** 제공, 다수 대기 시 **'모두 승인' 버튼** 제공(F5·F6). — **P1 근거**: 카오스 억제에는 도움이 되나, 기본을 '수동 승인'으로 두면 진입 마찰이 늘어 입장 무마찰(핵심 가치)과 긴장 관계다. 동선이 자리잡은 뒤 켜도 되므로 커서 끄기·작성자 표시·객체 잠금보다 한 단계 뒤로 둔다.
- **품질 기준**: 커서 표시 on/off 전환이 같은 캔버스 전원 화면에 2초 이내 반영 · 객체 잠금/해제가 같은 캔버스 전원 화면에 2초 이내 반영 · 입장 승인 후 참여자 입장까지 추가 마찰 없음(승인 즉시 입장)
- **하지 않는 것**: 페이지 고정 등 추가 가드레일(수요 확인 후) · 참여자별 권한 개별 조정(역할 기준만) · **잠긴 객체에 대한 편집자 전권 우선순위(잠금이 편집자 권한을 이기는지 = 편집자가 남이 잠근 객체를 풀 수 있는지)는 잔여 — OQ-25에서 결정**

## 6. 성공 지표 및 기각 기준

> 사전 벤치마크가 없는 첫 측정이므로 목표치는 [잠정]이며, 검증 모수·기간(OQ-02) 확정 후 고정한다.

**선행 지표 (일~주 단위):**

- 첫 캔버스 생성률 = 퀴즈앤 링크로 v2 진입한 교사 중 첫 캔버스 생성 비율 · 검증 대상 A1 · 목표 **5% [잠정 — OQ-02 확정 후 고정]** · 측정 = 오픈 후 4주, F9 퍼널 로그
- 캔버스당 참여율 = 입장 참여자 중 객체 1개+ 생성 비율 · 검증 대상 A3 · 목표 TBD — OQ-02 확정 후 고정 · 측정 = F4 편집 로그

**후행 지표 (주~월 단위):**

- 반복 사용률 = 첫 생성 교사 중 **2회 이상 활동** 비율("재사용" = 새 캔버스 생성 또는 기존 캔버스 복제 [잠정]) · 검증 대상 A2 — **이 PoC의 핵심 신호** · 목표 TBD — OQ-02 확정 후 고정

**기각(킬) 기준:** 오픈 4주 후 첫 캔버스 생성률 2% 미만 **AND** 반복 사용 교사 한 자릿수 → distribution 가설(A1)·수요 가설(A2) 재검토, 빌드 확장 중단.

## 8. 제약 및 NFR

- **성능**: 캔버스 편집·공감 반영 2초 이내(같은 캔버스 전원, 뷰어 포함) · 캔버스 목록 썸네일 갱신 2초 이내 · 타이머 동기 오차 1초 이내 · 참여자 입장 로딩 3초 이내(4G+) · 참여자 커서 표시 on/off 전환 2초 이내 반영 · 객체 잠금/해제 전원 반영 2초 이내(F13)
- **규모**: **캔버스당 진입자 최대 20명(고정) — 편집자·참여자·뷰어 모두 정원에 포함**(셋 모두 카운트) · **캔버스당 페이지 최대 30개(고정)** · 복제 횟수·프로젝트당 캔버스 수·그룹당 캔버스 수 상한 = 미정(OQ-19, 임의 고정값 두지 않음) · 전사 동시 캔버스 수 TBD — OQ-02와 함께 산정
- **데이터·도메인 분리**: board-v2는 퀴즈앤과 **데이터 관리·도메인이 완전 별개**다. 퀴즈앤과의 연결은 링크 이동뿐(F9). 단 로그인 계정 체계는 보류 상태로, **퀴즈앤 계정계 기반 SSO 연동을 검토 중**(미확정 — OQ-12)
- **플랫폼**: 태블릿·크롬북·스마트폰의 브라우저에서 동작, 특정 OS·브라우저 종속 금지, 앱 설치 없음
- **언어**: 한국어 전용(PoC 한정)
- **보존·삭제 정책 (OQ-15 해소)**: **보관(archive)한 캔버스는 사용자가 명시적으로 '삭제'하지 않는 한 영구 보존된다** — 보존 기간 제한·자동 만료·자동 삭제 없음, 언제든 재열람 가능(2절 캔버스 상태, 7절 비목표 18). 캔버스 삭제는 사용자의 명시적 동작으로만 일어난다.
- **개인정보**: 비로그인 참여자·뷰어 수집 항목 = 닉네임 + 비밀번호 4글자뿐, 이름·학번·연락처 미수집 · 단 객체 작성자 기록이 닉네임과 연결되므로 익명 옵션 시 노출 범위 준수(1절 가시성 규칙) · **비회원 비밀번호 찾기 장치 없음**(분실 시 복구 미제공 — OQ-11 해소, 7절 비목표 15) · **초등 포함으로 만 14세 미만 해당 확정** → 아동 개인정보 규정 적합 여부 설계 단계 필수 확인 · **보관 캔버스는 영구 보존(위)되므로, 그 안에 남는 닉네임/비밀번호4 등 개인정보의 보존·삭제 세부 정책은 TBD — OQ-23과 함께 결정**(보관 캔버스 자체의 영구 보존은 확정, 개인정보 보존 세부만 잔여)
- **비용**: PoC 운영비 최소화 원칙 · 상한 TBD — OQ-02와 함께 산정
- **자기잠식 경계**: 설계 산출물마다 퀴즈앤 기존 '보드'(wall)와의 중복 여부 점검 — canvas 차별점(동시 공동 조작) 유지
- **제품 명칭**: "board-v2"는 가명(코드네임), 정식 명칭 미정 — 로고 영역은 우선 'board-v2' 텍스트로 표기(0절)
