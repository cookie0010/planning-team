## 0. 요약

**한국 교실용 실시간 협업 canvas 보드** — 학생들이 모둠별 캔버스 위에서 동시에 함께 항목을 놓고·묶고·연결하며 공동 구성한다. 교사는 구조화된 틀(만다라트·마인드맵 등)을 깔고 지휘석에서 모든 모둠을 실시간으로 본다. 진입은 퀴즈앤 채널을 탄다 — 교사는 Labs에서 30초 안에 켜고, 학생은 무계정 QR로 들어온다.

> Why(배경·근거·차별화)는 one-pager 참조: `one-pager-v5.1.md`

**포지셔닝 한 문장:** 퀴즈앤 기존 '보드'(wall — 각자 제출해 모아 붙이기)가 못 하는 것, 즉 **같은 캔버스 위에서 여러 명이 동시에 함께 조작하는 공동 구성**(canvas)을 제공한다.

**이 빌드의 성격:** [x] PoC/실험 — 목적은 기능 완성이 아니라 **"교사가 실시간 공동참여 canvas를 수업에 반복해서 쓰는가"(수요)의 검증**이며, 성패는 6절 지표로 판정한다. 이 수요는 desk research로 입증도 반증도 안 된 상태다(`research-notes.md`).

## 2. 핵심 개념 모델 및 용어

```
세션 (교사가 1회 수업 활동 단위로 생성 — 틀 1종 + 타이머 + 모둠 수 설정)
 └── 모둠 1~N개 (세션 생성 시 교사가 수 지정 / 배정 방식은 OQ-12)
      └── 캔버스 1개 (모둠당 정확히 1개 — 최대 10명 동시 편집)
           └── 항목 (학생이 배치·이동·묶기·연결하는 카드/노드)
```

- **세션**: 교사가 만드는 활동 단위. 상태 = 진행 중(편집 가능) / 잠금(타이머 만료·편집 불가) / 종료(기록으로 보존). QR/링크는 세션 종료 전까지 유효.
- **모둠**: 세션 안의 참여 단위. 한 모둠 = 한 캔버스, 동시 편집 최대 10명.
- **캔버스**: 모둠이 공동 편집하는 화면. 교사가 선택한 틀이 깔린 채 시작하며, 빈 무한 캔버스는 존재하지 않는다.
- **틀**: 캔버스에 깔리는 구조(만다라트·마인드맵·클러스터링 [잠정 — OQ-03]).
- **지휘석**: 교사 전용 화면. 세션 1개의 모든 모둠 캔버스를 썸네일로 보고, 드릴다운·스포트라이트한다.

**용어 금지/통일:** 이 문서에서 "보드" 단독 사용 금지 — 사내에서 '보드'는 퀴즈앤 기존 wall 제품을 가리킨다. 학생이 편집하는 화면은 **캔버스**, 활동 단위는 **세션**으로 통일.

## 3. 기능 정의

### 한눈에 보기

|---|---|---|---|
| F1 | 세션 생성 | 교사가 틀·모둠 수·타이머를 정해 3액션 이내로 세션을 열고 QR 발급 | P0 |
| F2 | 무계정 입장·모둠 배정 | 학생이 QR/링크 + 닉네임만으로 자기 모둠 캔버스에 입장 | P0 |
| F3 | 구조화 틀 3종 | 만다라트·마인드맵·클러스터링 [잠정 — OQ-03] 중 1종을 캔버스에 깖 | P0 |
| F4 | 모둠 캔버스 실시간 공동 편집 | 모둠 최대 10명이 같은 캔버스에서 동시에 항목을 놓고·묶고·연결 | P0 |
| F5 | 타이머 | 교사가 제한 시간을 설정·제어, 만료 시 전 캔버스 편집 잠금 | P0 |
| F6 | 교사 지휘석 | 전 모둠 썸네일 실시간 갤러리 + 드릴다운 + 스포트라이트 | P0 |
| F7 | 세션 기록·재진입 | 교사가 지난 세션을 다시 열고 결과를 봄 — 재사용 지표의 측정 표면 | P0 |
| F8 | 퀴즈앤 진입 동선·퍼널 계측 | 퀴즈앤 → v2 진입점과 진입→생성 전환 측정 (위치는 OQ-01) | P0 |
| F9 | 실시간 반응 | 캔버스 위 이모지 등 가벼운 반응 레이어 [잠정 — OQ-16] | P1 |

> "3액션" 정의: Labs 진입 이후 클릭/탭 1회 = 1액션. 화면 간 이동도 클릭으로 센다.

### 기능 상세

**F1. 세션 생성** *(교사)*

- **필수 정보**: 틀 3종 선택지(미리보기 포함), 모둠 수 설정값, 타이머 설정값(선택), 생성 직후 QR·공유 링크
- **필수 행동**: 틀 선택 → 모둠 수 지정 → 생성. 생성 후 타이머 추가·변경, 세션 종료
- **플로우/상태**: 생성 즉시 '진행 중' + QR 표시 화면. 종료 시 F7 기록으로 전환
- **품질 기준**: Labs 진입 후 3액션·30초 이내 생성 완료
- **하지 않는 것**: 세션 예약·반복 일정, 학생 명단 사전 등록

**F2. 무계정 입장·모둠 배정** *(학생)*

- **필수 정보**: 입장 화면에 세션명·닉네임 입력란·(배정 방식에 따라) 모둠 선택지
- **필수 행동**: QR 스캔 또는 링크 접속 → 닉네임 입력 → 모둠 배정 → 캔버스 진입
- **플로우/상태**: 잠금·종료된 세션 접속 시 안내 화면(편집 불가 사유 표시)
- **품질 기준**: 스캔→캔버스 진입까지 절차는 닉네임 입력 1회뿐, 로딩 3초 이내(4G 이상). 앱 설치·가입·로그인 없음
- **하지 않는 것**: 학생 계정·출석 연동·개인정보 수집(닉네임은 세션 한정 임시값)

**F3. 구조화 틀 3종** *(교사 선택 → 캔버스 기반)*

- **필수 정보**: 각 틀의 구조가 캔버스에 시각적으로 깔린 상태(만다라트 3×3 그리드 / 마인드맵 중심 노드 / 클러스터링 발산·분류 영역)
- **필수 행동**: 교사가 틀의 중심 주제(텍스트)를 생성 시 입력
- **품질 기준**: 틀 없는 캔버스로 시작하는 경로가 존재하지 않을 것
- **하지 않는 것**: 빈 무한 캔버스, 틀 커스텀 제작. ⚠️ 클러스터링은 '전원 동시 발산'이 아니라 공동 *분류·구조화* 단계에 초점(동시 자유발산의 열위 근거: Diehl & Stroebe 1987, 홍기칠 2013 — `working-notes.md`)
- **[잠정]** 3종 구성 자체가 현장 분석 전 후보다 — OQ-03 확정 후 교체 가능(비주얼씽킹 등 대안 검토 중)

**F4. 모둠 캔버스 실시간 공동 편집** *(학생, 모둠당 최대 10명)*

- **필수 정보**: 같은 모둠원의 편집이 실시간 반영된 캔버스, 잔여 시간(F5), 내 닉네임
- **필수 행동**: 항목 추가(텍스트 입력)·이동·묶기·연결, 별도 안내 없이 입장 즉시 조작 시작
- **플로우/상태**: 진행 중(편집 가능) / 잠금(보기만 가능, 배치된 항목 유지)
- **품질 기준**: 한 학생의 편집이 같은 모둠 전원 화면에 **2초 이내** 반영. 10명 동시 조작에서 항목 유실·중복 생성 0건
- **하지 않는 것**: 자유 그리기(드로잉), 다른 모둠 캔버스 열람·편집, 교사 제어 기능 노출

**F5. 타이머** *(교사 제어)*

- **필수 정보**: 잔여 시간이 교사 지휘석·전 학생 캔버스에 동기 표시(오차 1초 이내)
- **필수 행동**: 교사가 시작·일시정지·연장·종료. 만료 시 전 캔버스 자동 잠금, 교사는 재개방 가능
- **하지 않는 것**: 모둠별 개별 타이머

**F6. 교사 지휘석** *(교사 전용)*

- **필수 정보**: 세션 내 전 모둠 캔버스 썸네일(편집 후 2초 이내 갱신), 모둠별 참여 인원·항목 수
- **필수 행동**: 모둠 드릴다운(원본 크기 열람), 스포트라이트(선택 모둠을 전체 공유 화면에 표시), 타이머·세션 제어
- **품질 기준**: 모둠 6개 기준 한 화면에서 스크롤 없이 전 모둠 파악 가능
- **하지 않는 것**: 채점·학생 개인별 기여도 표시(교사의 읽기 = 분포 파악·막힌 모둠 돕기·좋은 사례 하이라이트)

**F7. 세션 기록·재진입** *(교사)*

- **필수 정보**: 지난 세션 목록(주제·일시·모둠 수·항목 수), 세션별 모둠 캔버스 썸네일
- **필수 행동**: 결과 보기(읽기 전용), 다시 열기(잠금 해제 재개)
- **품질 기준**: 모든 재진입 이벤트가 로그로 남을 것 — A2(반복 사용) 측정의 표면
- **하지 않는 것**: 내보내기·인쇄 등 상세 깊이는 OQ-15에서 결정

**F8. 퀴즈앤 진입 동선·퍼널 계측** *(distribution)*

- **필수 정보**: 퀴즈앤 안의 v2 진입점(위치 = OQ-01, **미결정 시 진입 화면 설계 불가**)
- **필수 행동**: 진입 클릭 → 템플릿 선택 화면 직행. 진입·생성·학생 입장·재진입 각 단계의 이벤트 계측(교사 계정+세션 ID 키, 학생은 디바이스 근사)
- **품질 기준**: 6절 지표를 산출하는 데 필요한 전 이벤트가 빠짐없이 수집될 것

**F9. 실시간 반응** *(학생·교사, P1)*

- one-pager의 "인터랙티브 요소(실시간 반응·이모지)"에 해당. 포함 여부·형태가 미정(OQ-16)이라 P0에서 제외, 설계만 대비
- **하지 않는 것**: wall형 공감투표·댓글(기존 보드 영역과 중복 — 7절 비목표 1)

## 6. 성공 지표 및 기각 기준

> 사전 벤치마크가 없는 첫 측정이므로 목표치는 [잠정]이며, 검증 모수·기간(OQ-02) 확정 후 고정한다.

**선행 지표 (일~주 단위):**
- 첫 세션 생성률 = Labs에서 v2 진입한 교사 중 첫 세션 생성 비율 · 검증 대상 A1 · 목표 **10% [잠정 — OQ-02 확정 후 고정]** · 측정 = 오픈 후 4주, F8 퍼널 로그
- 세션당 학생 참여율 = 입장 학생 중 항목 1개+ 배치 비율 · 검증 대상 A3 · 목표 TBD — OQ-02 확정 후 고정 · 측정 = F4 편집 로그

**후행 지표 (주~월 단위):**
- 반복 생성률 = 첫 세션 생성 교사 중 **2회 이상** 세션 생성 비율("재사용" = 별도 세션을 새로 생성) · 검증 대상 A2 — **이 PoC의 핵심 신호** · 목표 TBD — OQ-02 확정 후 고정 · 측정 = F7·F8 로그

**기각(킬) 기준:** 오픈 4주 후 첫 세션 생성률 3% 미만 **AND** 반복 생성 교사 한 자릿수 → distribution 가설(A1)·수요 가설(A2) 재검토, 빌드 확장 중단.
