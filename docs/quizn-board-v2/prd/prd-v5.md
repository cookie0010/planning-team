## 0. 요약

**한국 교실용 실시간 협업 canvas 보드** — 학생들이 같은 캔버스 위에서 동시에 텍스트·그리기·포스트잇·도형으로 함께 만든다. 편집자(전형적으로 교사)는 여러 페이지를 붙여 하나의 프로젝트(=캔버스)를 만들고, 잘 만든 프로젝트는 통째로 복제해 반별로 재사용한다. 캔버스들을 한 화면에 모아 보는 것이 곧 지휘석이다. 진입은 퀴즈앤 채널을 탄다 — 교사는 퀴즈앤에서 링크로 넘어와 30초 안에 수업 환경에 접속하고, 학생은 무계정 QR/URL로 들어온다.

> Why(배경·근거·차별화)는 one-pager 참조: `one-pager-v5.1.md`
> 캔버스 UI 레퍼런스(구속력 없음): `reference/` — FigJam·Miro·알로·미리캔버스 스크린샷

**포지셔닝 한 문장:** 퀴즈앤 기존 '보드'(wall — 각자 제출해 모아 붙이기)가 못 하는 것, 즉 **같은 캔버스 위에서 여러 명이 동시에 함께 조작하는 공동 구성**(canvas)을 제공한다.

**이 빌드의 성격:** [x] PoC/실험 — 목적은 기능 완성이 아니라 **"교사가 실시간 공동참여 canvas를 수업에 반복해서 쓰는가"(수요)의 검증**이며, 성패는 6절 지표로 판정한다.

> **제품 명칭:** "board-v2"는 가명(코드네임)이다. 정식 명칭은 미정 — 로고 영역에는 우선 'board-v2' 텍스트로 표기한다.

## 1. 대상 사용자 및 권한

교실 맥락의 기본 그림: 교사 = 소유자, 학생 = 참여자. 권한 구조 자체는 역할 일반형(소유자/편집자/참여자)으로 설계한다.

| 역할 | 정의 | 계정 요건 | 핵심 권한 |
|---|---|---|---|
| 소유자 | 프로젝트 생성자 (전형적으로 교사) | **로그인 필수** (퀴즈앤 계정) | 프로젝트 전체 제어, 편집자 임명, 닉네임/비밀번호·익명 옵션 설정 |
| 편집자 | 소유자 또는 다른 편집자가 임명 | **로그인 필수** | 해당 **캔버스(=프로젝트)**의 페이지 추가·템플릿 설정·캔버스 편집, 편집자 임명, 닉네임/비밀번호·익명 옵션 설정 |
| 참여자 | 캔버스에 입장한 사람 (전형적으로 학생) | 불필요 — 무계정 입장 가능, 로그인 시 재진입 혜택 | 편집자가 설정해 둔 페이지에서 캔버스 작업(객체 생성·이동·편집), 포스트잇 공감 투표 |

**참여자 식별 규칙:**

- 로그인 참여자: 본인이 참여했던 캔버스에 언제든 재진입 가능.
- 비로그인 참여자: 동일한 **닉네임/비밀번호**를 입력했을 때만 동일 사용자로 취급. 그렇지 않으면 진입 당시 브라우저 세션별로 분리되어 다른 사용자로 인식.
- 닉네임/비밀번호 옵션의 사용 여부는 소유자·편집자가 조정.

**편집자·참여자 권한 경계:**

- 편집자: 페이지 추가 생성, 각 페이지에 템플릿 설정, 캔버스 편집이 가능하다. 닉네임/비밀번호·익명 옵션을 설정한다.
- 참여자: 템플릿 수정·페이지 추가는 **불가**. 편집자가 설정해 놓은 페이지에서 캔버스 작업(객체 생성·이동·편집)과 포스트잇 공감 투표만 가능하다.

**작성자 가시성 규칙:**

- 객체(포스트잇 등)의 작성자를 소유자·편집자는 **항상** 확인할 수 있다.
- **익명 옵션**을 켜면 참여자 간에는 작성자 비공개 — 그래도 소유자·편집자에게는 보인다. 옵션은 소유자·편집자가 조정.
- 작성자 표기 방식(상시 표시·호버·교사 뷰 오버레이 등)은 추가 논의 필요 — OQ-18.

**편집자 임명 규칙:** 임명 단위 = 캔버스(=프로젝트). 소유자→편집자, 편집자→편집자 임명 가능. 임명되면 그 캔버스의 페이지 추가·템플릿 설정·편집이 가능.

**학생 대상:** **초·중·고** 학생, 태블릿·크롬북·스마트폰.

**범위 밖 사용자:** 대학·기업, 퀴즈앤 미사용 신규 교사(PoC 기간은 기존 퀴즈앤 채널로 한정).

## 2. 핵심 개념 모델 및 용어

```
그룹 (프로젝트들을 묶는 폴더 — 프로젝트 목록 화면에서 정리·관리)
 └── 프로젝트 (= 캔버스. 소유자가 생성. 여러 페이지로 구성된 하나의 완성품
      — QR 1개 발급, 학생 진입 단위, 진입자 최대 20명, 권한·옵션의 단위)
      └── 페이지 1~N장 (캔버스의 작업 면 — 편집자가 템플릿을 붙여 구성, 좌우로 추가·이동, 유한)
           └── 객체 (텍스트·그리기·포스트잇·도형 등 기본 도구로 만드는 것)
```

- **그룹**: 프로젝트(=캔버스)들을 묶는 폴더. 프로젝트 목록 화면에서 정리·관리하는 용도다. 그룹 카드 안에 그 그룹에 속한 프로젝트 썸네일들이 보이고 그룹명 라벨이 붙는다(예: "2026 1학기").
- **프로젝트(= 캔버스)**: 여러 페이지로 구성된 하나의 완성품. 프로젝트와 캔버스는 같은 한 덩어리다(별도 2층 구조가 아니다). QR 1개를 발급하며, 학생이 들어오는 진입 단위이고 **진입자 최대 20명**이다. 편집자 임명·옵션 설정이 이 단위에 묶인다. 상태 = 진행 중 / 잠금(타이머 만료) / 보관.
- **페이지**: 캔버스의 작업 면. 편집자가 템플릿을 붙여 구성한다(예: 템플릿을 이것저것 붙여 10페이지짜리 프로젝트를 완성). 좌우로 페이지를 추가하고 이동할 수 있다(미리캔버스·알로 참고). 무한 줌 월드가 아니라 **유한한 페이지**다.
- **템플릿**: 페이지에 깔리는 객체 묶음(캔버스 기본 도구들을 조합해 만든 틀이며 별도 엔진이 아니다). 편집자가 페이지별로 설정한다. 우선 3종 = **빈 캔버스 · 브레인스토밍 · 마인드맵** [잠정 — OQ-03].
- **객체**: 페이지 위에 만들고 옮기는 것 — 텍스트, 그리기(펜), 포스트잇, 도형 등 일반 캔버스 도구가 만드는 요소(FigJam·Miro·알로·미리캔버스 참고). 모든 객체에는 작성자 정보가 남는다(1절 가시성 규칙).
- **복제**: 잘 만든 프로젝트 하나를 통째로 복제해 재사용하는 것(예: 1반·2반·3반용으로 복제). 복제본은 각각 독립된 프로젝트이고 각자 다른 QR을 가진다.

**용어 금지/통일:** 이 문서에서 "보드" 단독 사용 금지 — 사내에서 '보드'는 퀴즈앤 기존 wall 제품을 가리킨다. **그룹 / 프로젝트(=캔버스) / 페이지 / 객체**로 통일.

### 한눈에 보기

|---|---|---|---|
| F1 | 프로젝트 생성·관리 | 소유자가 여러 페이지로 구성된 프로젝트(=캔버스)를 만들고 관리, QR 발급 | P0 |
| F2 | 프로젝트 복제 | 잘 만든 프로젝트를 통째로 복제해 재사용(반별 등), 복제본은 독립·QR 각자 | P0 |
| F3 | 템플릿 3종 | 빈 캔버스·브레인스토밍·마인드맵, 편집자가 페이지별로 설정 [잠정 — OQ-03] | P0 |
| F4 | 캔버스 공동 편집 | 기본 도구(텍스트·그리기·포스트잇·도형) + 페이지 구성 + 실시간 동시 편집 | P0 |
| F5 | 입장·사용자 식별 | QR/URL 입장, 무계정(닉네임/비밀번호) 또는 로그인, 재진입 | P0 |
| F6 | 권한 관리 | 소유자/편집자/참여자 3단계, 캔버스 단위 편집자 임명, 익명 옵션 | P0 |
| F7 | 지휘석 (캔버스 모아 보기) | 캔버스들을 한 화면에 모아 보기 = 썸네일 갤러리 + 드릴다운 + 스포트라이트 + 타이머/잠금 + 작성자 항상 표시 | P0 |
| F8 | 타이머 | 소유자·편집자가 제한 시간 설정·제어, 만료 시 캔버스 잠금 | P0 |
| F9 | 퀴즈앤 진입 동선·퍼널 계측 | 퀴즈앤 → v2 링크 진입과 전환 측정(v2는 퀴즈앤과 데이터·도메인 별개) | P0 |
| F10 | 포스트잇 공감 투표 | 참여자가 다른 사람의 포스트잇에 공감을 남김 | P0 |
| F11 | 그룹 관리 | 프로젝트 목록을 그룹(폴더)으로 묶어 정리 | P0 |

### 기능 상세

**F1. 프로젝트 생성·관리** *(소유자)*

- **필수 정보**: 내 프로젝트 목록(이름·생성일·페이지 수, 그룹 소속), 프로젝트별 설정(편집자 목록, 닉네임/비밀번호·익명 옵션), 프로젝트별 진입 QR·URL
- **필수 행동**: 프로젝트 생성·이름 지정·삭제, 페이지 구성(F4), 편집자 임명(F6), 옵션 설정, QR/URL 발급·재표시
- **품질 기준**: 진입 후 첫 프로젝트 QR 발급까지 3액션·30초 이내(실측 동선은 F9 참조)
- **하지 않는 것**: 프로젝트 간 내용 동기화(복제본은 서로 독립 — F2)

**F2. 프로젝트 복제** *(소유자·편집자)*

- **필수 정보**: 복제 대상 프로젝트, 복제본 목록(각자 QR·URL)
- **필수 행동**: 잘 만든 프로젝트 하나를 통째로 복제(예: 1반·2반·3반용) · 복제본은 각각 독립된 프로젝트로 생성되고 각자 다른 QR을 가짐
- **플로우/상태**: 복제 즉시 진행 중 + QR 표시 · 복제 횟수·그룹당 프로젝트 수 상한은 미정(OQ-19) — 정해지기 전에는 임의 제한을 두지 않음
- **하지 않는 것**: 복제본 간 내용 동기화(복제본은 서로 독립)

**F3. 템플릿 3종** *(편집자가 페이지별 설정)*

- **필수 정보**: 템플릿 선택지 3종과 미리보기 — **빈 캔버스 / 브레인스토밍 / 마인드맵** [잠정 — OQ-03]
- **필수 행동**: 편집자가 페이지에 템플릿을 설정하면 해당 객체 묶음이 그 페이지에 깔린다. 템플릿 = 캔버스 기본 도구들을 조합해 만든 틀(별도 엔진 아님)
- **하지 않는 것**: 사용자 커스텀 템플릿 제작·저장(후속). ⚠️ 브레인스토밍 템플릿은 '전원 동시 발산'이 아니라 공동 분류·구조화 단계에 초점(동시 자유발산의 열위 근거 — `working-notes.md`)

**F4. 캔버스 공동 편집** *(참여자·소유자·편집자)*

- **필수 정보**: 같은 캔버스 사용자들의 편집이 실시간 반영된 페이지, 페이지 위치(N/M), 잔여 시간(F8), 객체 작성자 정보(1절 가시성 규칙에 따라 권한별 노출)
- **필수 행동**:
  - 참여자: 편집자가 설정해 둔 페이지에서 기본 도구로 객체 생성·이동·편집 — **텍스트 입력, 그리기(펜), 포스트잇, 도형** 등 일반 캔버스 기본 기능(FigJam·Miro·알로·미리캔버스 수준 참고). 페이지 추가·템플릿 수정은 불가.
  - 편집자: 위에 더해 페이지 좌우 추가·이동, 각 페이지에 템플릿 설정.
- **플로우/상태**: 진행 중(편집 가능) / 잠금(보기만 가능, 객체 유지)
- **품질 기준**: 한 사용자의 편집이 같은 캔버스 전원 화면에 **2초 이내** 반영 · 동시 조작에서 객체 유실·중복 0건
- **하지 않는 것**: 다른 프로젝트(캔버스) 열람·편집(참여자 기준 — 소유자·편집자는 F7로 접근)

**F5. 입장·사용자 식별** *(참여자)*

- **필수 정보**: 입장 화면 — 캔버스(프로젝트) 이름, 닉네임 입력란(옵션 설정 시 비밀번호 입력란)
- **필수 행동**: QR 스캔 또는 URL 접속 → (옵션에 따라) 닉네임/비밀번호 입력 → 캔버스 진입 · 로그인 사용자는 참여했던 캔버스에 언제든 재진입
- **플로우/상태**: 비로그인 + 동일 닉네임/비밀번호 = 동일 사용자로 복원 / 그 외 = 브라우저 세션별 별도 사용자 · **정원(20명) 초과 시 입장 불가 안내** · 잠금·보관 캔버스 접속 시 안내 화면
- **품질 기준**: 스캔→진입 로딩 3초 이내(4G+) · 무설치·무가입(로그인은 선택)
- **하지 않는 것**: 참여자에게 가입·로그인 강제, 닉네임 외 개인정보 수집

**F6. 권한 관리** *(소유자·편집자)*

- **필수 정보**: 캔버스(프로젝트)별 편집자 목록, 닉네임/비밀번호·익명 옵션 상태
- **필수 행동**: 소유자·편집자가 로그인 사용자를 그 캔버스의 편집자로 임명 · 임명된 편집자는 해당 캔버스의 페이지 추가·템플릿 설정·편집 가능 · 닉네임/비밀번호 옵션 on/off · **익명 옵션 on/off**(켜면 참여자 간 작성자 비공개, 소유자·편집자는 항상 확인)
- **하지 않는 것**: 페이지 단위 개별 권한, 읽기 전용 공유 링크(후속 — OQ-15와 연동)

**F7. 지휘석 (캔버스 모아 보기)** *(소유자·편집자)*

- **개념**: 캔버스(프로젝트)들을 한 화면에 모아 보는 것이 곧 지휘석이다 — 캔버스 목록 화면과 지휘석은 사실상 같은 화면이다.
- **필수 정보**: 캔버스 썸네일 갤러리(편집 후 2초 이내 갱신), 캔버스별 참여 인원·객체 수, **객체별 작성자**(익명 옵션과 무관하게 항상)
- **필수 행동**: 캔버스 드릴다운(원본 열람·진입), 스포트라이트(선택 캔버스를 전체 공유 화면에 표시), 타이머·잠금 제어
- **품질 기준**: 캔버스 6개 기준 한 화면에서 스크롤 없이 전체 파악 가능
- **하지 않는 것**: 채점·참여자별 기여도 표시(읽기 = 분포 파악·막힌 곳 돕기·좋은 사례 하이라이트)

**F8. 타이머** *(소유자·편집자)*

- **필수 행동**: 캔버스에 제한 시간 설정, 시작·일시정지·연장·종료 · 만료 시 해당 캔버스 편집 자동 잠금(객체 유지), 재개방 가능
- **품질 기준**: 잔여 시간이 지휘석·참여자 화면에 동기 표시, 오차 1초 이내

**F9. 퀴즈앤 진입 동선·퍼널 계측** *(distribution)*

- **진입 방식**: 퀴즈앤 서비스에서 **링크 이동**으로 board-v2에 진입한다. board-v2는 **데이터 관리·도메인이 퀴즈앤과 완전 별개**이므로, v2 내부의 진입점 위치 설계는 우리 범위 밖이며 퀴즈앤은 링크만 연결한다.
- **첫 사용 동선**: 링크 진입 → board-v2 프로젝트 목록 화면으로 이동하면서 동시에 '템플릿 생성' 모달 자동 오픈 → 템플릿 선택 시 즉시 캔버스(프로젝트) 진입 → 캔버스 안에서 QR 오픈 가능(실측 2액션).
- **필수 행동**: 진입 → 프로젝트 생성/복제 → 참여자 입장 → 재사용, 각 단계 이벤트 계측(소유자 계정+프로젝트 ID 키, 비로그인 참여자는 디바이스 근사)
- **품질 기준**: 6절 지표 산출에 필요한 전 이벤트가 빠짐없이 수집될 것

**F10. 포스트잇 공감 투표** *(참여자·소유자·편집자)*

- **필수 정보**: 포스트잇별 공감 수
- **필수 행동**: 다른 사람의 포스트잇에 공감을 남김·취소
- **품질 기준**: 공감 반영 2초 이내(같은 캔버스 전원)
- **하지 않는 것**: 댓글(영구 제외 — 7절 비목표 1) · 이모지 등 추가 반응 레이어는 미정(OQ-16, 설계만 대비) · 1인당 투표 횟수 등 세부 규칙은 OQ-16에서 결정

**F11. 그룹 관리** *(소유자·편집자)*

- **필수 정보**: 프로젝트 목록 화면의 그룹 카드(그룹명 라벨, 소속 프로젝트 썸네일들)
- **필수 행동**: 그룹 생성·이름 지정(예: "2026 1학기"), 프로젝트를 그룹에 넣고 빼기, 그룹 단위 정리
- **하지 않는 것**: 그룹 단위 일괄 권한·일괄 설정(후속) · 그룹당 프로젝트 수 상한은 미정(OQ-19)

## 6. 성공 지표 및 기각 기준

> 사전 벤치마크가 없는 첫 측정이므로 목표치는 [잠정]이며, 검증 모수·기간(OQ-02) 확정 후 고정한다.

**선행 지표 (일~주 단위):**

- 첫 프로젝트 생성률 = 퀴즈앤 링크로 v2 진입한 교사 중 첫 프로젝트 생성 비율 · 검증 대상 A1 · 목표 **5% [잠정 — OQ-02 확정 후 고정]** · 측정 = 오픈 후 4주, F9 퍼널 로그
- 캔버스당 참여율 = 입장 참여자 중 객체 1개+ 생성 비율 · 검증 대상 A3 · 목표 TBD — OQ-02 확정 후 고정 · 측정 = F4 편집 로그

**후행 지표 (주~월 단위):**

- 반복 사용률 = 첫 생성 교사 중 **2회 이상 활동** 비율("재사용" = 새 프로젝트 생성 또는 기존 프로젝트 복제 [잠정]) · 검증 대상 A2 — **이 PoC의 핵심 신호** · 목표 TBD — OQ-02 확정 후 고정

**기각(킬) 기준:** 오픈 4주 후 첫 프로젝트 생성률 2% 미만 **AND** 반복 사용 교사 한 자릿수 → distribution 가설(A1)·수요 가설(A2) 재검토, 빌드 확장 중단.

## 8. 제약 및 NFR

- **성능**: 캔버스 편집·공감 반영 2초 이내(같은 캔버스 전원) · 지휘석 썸네일 갱신 2초 이내 · 타이머 동기 오차 1초 이내 · 참여자 입장 로딩 3초 이내(4G+)
- **규모**: **캔버스(=프로젝트)당 진입자 최대 20명(고정)** · 복제 횟수·그룹당 프로젝트 수 상한 = 미정(OQ-19, 임의 고정값 두지 않음) · 전사 동시 캔버스 수 TBD — OQ-02와 함께 산정
- **데이터·도메인 분리**: board-v2는 퀴즈앤과 **데이터 관리·도메인이 완전 별개**다. 퀴즈앤과의 연결은 링크 이동뿐(F9)
- **플랫폼**: 태블릿·크롬북·스마트폰의 브라우저에서 동작, 특정 OS·브라우저 종속 금지, 앱 설치 없음
- **언어**: 한국어 전용(PoC 한정)
- **개인정보**: 비로그인 참여자 수집 항목 = 닉네임(+옵션 비밀번호)뿐, 이름·학번·연락처 미수집 · 단 객체 작성자 기록이 닉네임과 연결되므로 익명 옵션 시 노출 범위 준수(1절 가시성 규칙) · **초등 포함으로 만 14세 미만 해당 확정** → 아동 개인정보 규정 적합 여부 설계 단계 필수 확인 · 닉네임/비밀번호의 보존·삭제 정책 TBD — OQ-15와 함께 결정
- **비용**: PoC 운영비 최소화 원칙 · 상한 TBD — OQ-02와 함께 산정
- **자기잠식 경계**: 설계 산출물마다 퀴즈앤 기존 '보드'(wall)와의 중복 여부 점검 — canvas 차별점(동시 공동 조작) 유지
- **제품 명칭**: "board-v2"는 가명(코드네임), 정식 명칭 미정 — 로고 영역은 우선 'board-v2' 텍스트로 표기(0절)
