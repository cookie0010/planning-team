## 한눈에 보기

| 축 | FigJam | Miro | 알로 | 미리캔버스 |
|---|---|---|---|---|
| 구조 계층 | 팀 > 파일 | 스페이스 > 보드 | **프로젝트 > 캔버스 > 페이지** | 드라이브 > 디자인 > 페이지 |
| 템플릿 선택 시점 | 캔버스 안 모달 | 캔버스 안 모달 | **캔버스 생성 시 모달** | 생성 시(용도/사이즈) + 에디터 안 |
| 무계정 입장 | ○ Open session (이름만 입력, 24h 만료) | △ Visitor — 편집은 유료/Education 전용 | △ 지원 추정, 세부 미확인 | ✕ 멤버 초대만(이메일 수락) |
| 타이머·운영 장치 | 타이머 상단 상시 | 타이머·투표·비공개모드 묶음 메뉴 | 거의 없음 | 없음 |

| 작업 면 | 무한 캔버스 | 무한 캔버스 | **유한 페이지 (1/N)** | **유한 페이지 (1/N)** |
| 도구바 위치 | **하단 중앙 가로** | 좌측 세로 | 좌측 세로 | 좌측 패널(요소 중심) |
| 페이지 내비 | — | — | 좌측 세로 썸네일 패널 | 하단 가로 스트립 |

요약하면: **계층 구조와 유한 페이지는 알로가, 무계정 입장은 FigJam이, 수업 운영 장치는 Miro가 우리의 직접 참고 모델**이다. 지휘석에 해당하는 기능은 4개 도구 모두 없다 — 레퍼런스 없이 설계해야 하고, 그 자체가 차별점이라는 뜻이기도 하다.

## 1. 세션 시작 동선

**FigJam** — 파일 브라우저 상단에 템플릿 추천 줄("Get started with a template" + 용도 탭), 그 아래 파일 그리드. 새 파일은 우상단 Create. 템플릿을 먼저 들이미는 구조라 "빈 캔버스로 빨리 시작"과는 거리가 있다.

**Miro** — 대시보드 상단에 "템플릿 용도" 추천 줄: **첫 칸이 빈 보드(+), 그 옆이 템플릿 카드들**. 빠른 시작과 템플릿 권유를 한 줄에 공존시킨 패턴. 보드 목록은 테이블형(이름·온라인·최종 열람일·소유자).

**알로** — 좌측 사이드바(홈/프로젝트/캘린더), 본문에 "최근" 줄 + 프로젝트 카드 그리드. **"+ 새 프로젝트"와 "+ 새 캔버스" 버튼이 분리**되어 있고, 프로젝트 카드에 진행 상태 태그·진행률이 붙는다. 프로젝트>캔버스 2계층 운영이 우리 PRD 개념 모델과 같다.

**미리캔버스** — "새 디자인 만들기" CTA → "무엇을 만들어야 하나요?" 모달(용도·사이즈 선택)이 끼어든다. 디자인 도구라 생성 전 결정이 많은 편 — 교실 빠른 시작에는 부적합한 패턴.

**와이어프레임에 가져갈 것:**
- T1 프로젝트 목록 = 알로형 카드 그리드(+ 새 프로젝트)가 기본. 캔버스 수·상태를 카드에 표기
- "빈 캔버스 우선, 템플릿은 옆에" 배치는 Miro 추천 줄 패턴 참고
- 미리캔버스식 생성 전 다단계 결정은 회피 — 3액션 제약과 충돌

## 2. 참여자 입장 (공식 문서 확인)

**FigJam Open session** — 우리 F5와 가장 가까운 실존 모델. 공유 모달에서 Start → 링크 복사 → 받은 사람은 **계정 없이 이름(별칭)만 입력하고 편집 참여**. 비밀번호 옵션 제공. 세션은 24시간 후 자동 종료(우리의 타이머→잠금과 비교 가능). 단 댓글·음성은 로그인 필요.

**Miro** — Visitor(공개 링크·무계정)와 Guest(이메일 초대·로그인 필수)를 구분. **Visitor의 편집 권한은 유료/Education 플랜 전용**, 무료는 보기만. 링크 권한은 보기/댓글/편집 3단. 비밀번호 옵션.

**알로** — 교육 페이지에서 게스트 참여를 내세우지만 공식 도움말에서 무계정 편집의 세부 동작(닉네임 입력 여부 등)은 확인 못 함. 단정하지 않고 비워 둔다.

**미리캔버스** — 무계정 게스트 편집 없음. 공동 편집 = 워크스페이스 멤버(이메일 초대 수락) 기반, 링크 공유는 보기용 웹 게시.

**와이어프레임에 가져갈 것:**
- S1 입장 화면 = FigJam 패턴: 캔버스 이름 + 닉네임 입력 + (옵션) 비밀번호 + 입장 버튼, "로그인" 링크는 보조 동선으로 약하게
- QR을 입장 화면 앞단에 두는 건 4개 도구 어디에도 없는 우리 요구사항(교실 프로젝터) — T5를 별도 화면으로 유지하는 근거

## 3. 교사 통제 장치

**Miro가 유일하게 묶음 메뉴를 제공** — 우상단 아이콘에서 타이머·투표·**비공개 모드**·메모 + 이모지 리액션 6종(캡처 miro-4). 비공개 모드(작성 내용을 서로 안 보이게)는 우리 익명 옵션(F6)과 발상이 같은 운영 장치다. FigJam은 타이머가 상단에 상시 노출(03:00)되고 음악·투표가 따라온다. 알로·미리캔버스는 수업 운영 장치가 사실상 없다.

**와이어프레임에 가져갈 것:**
- 타이머 잔여 시간은 FigJam처럼 상단 상시 노출(S2·T6 공통)
- 교사용 운영 도구는 Miro처럼 한 묶음으로 — 단 우리는 지휘석(T6)이 본진이므로, 캔버스 안 통제는 최소(타이머·잠금)로
- 공감 투표(F10)는 Miro 리액션처럼 객체에 직접 남기는 패턴으로 — 별도 투표 세션 만들기(Miro 투표)는 과함

## 4. 템플릿 제공 방식

두 패턴이 갈린다:

- **생성 시 선택** — 알로: "+ 새 캔버스" → 템플릿 모달. **첫 줄이 "빈 캔버스(가로)/빈 캔버스(세로)/PDF로 만들기"**, 아래로 용도별 템플릿(데일리 스탠드업·주간 회의·디자인 리뷰 등) 카드 그리드 + 좌측 분류 + 검색. 우리 PRD F2("캔버스 추가 시 적용 여부 결정")와 정확히 같은 모델.
- **캔버스 안 삽입** — FigJam·Miro: 빈 캔버스로 떨어진 뒤 모달에서 템플릿을 불러옴. FigJam 모달은 검색 + 좌측 분류(활동별/역할별 — "Student & educator" 항목 존재) + 추천 카드. 템플릿은 별도 엔진이 아니라 **캔버스에 깔리는 객체 묶음**(브레인스토밍 템플릿 = 색 섹션 + 프롬프트 박스 4개, 캡처 figjam-3) — PRD F3의 "템플릿 = 기본 도구 조합" 정의가 업계 표준과 일치함을 확인.
- 알로 템플릿 적용 결과(캡처 allo-3): 4P 매트릭스가 페이지에 깔리고 사분면마다 색 구분 + 안내 텍스트 + 예시 포스트잇. 브레인스토밍 템플릿의 좋은 참고.

**와이어프레임에 가져갈 것:**
- T4 캔버스 생성 = 알로형 모달: 빈 캔버스가 첫 자리, 템플릿 2종이 미리보기 카드로 옆에. 3종뿐이라 분류·검색 불필요 — 카드 3장 + 이름 입력이면 끝
- 템플릿 미리보기 카드는 실제 틀 모양을 보여줄 것(알로·FigJam 공통 패턴)

## 5. 캔버스 편집 화면

| | 도구바 | 페이지 내비 | 특징 |
|---|---|---|---|
| FigJam | **하단 중앙 가로** (선택·손·펜·도형·텍스트·포스트잇·테이블·스탬프·코멘트) | — | 포스트잇에 작성자명 자동 표시, 좌측 Shapes 패널은 필요 시 |
| Miro | 좌측 세로 | — | 우상단에 리액션·채팅·화상·공유 집결 |
| 알로 | 좌측 세로 | **좌측 세로 썸네일 패널** + "새 페이지 추가" | 상단에 페이지 제목 + "페이지 1/2" 표시 |
| 미리캔버스 | 좌측 패널(요소 라이브러리형) | **하단 가로 스트립** + "+ 페이지 추가" | 유한 페이지(사이즈 고정)가 중앙에 |

**와이어프레임에 가져갈 것:**
- S2 도구바는 **하단 중앙(FigJam형)** — 학생 디바이스가 태블릿·폰이라 thumb zone과 일치하고, 도구 수가 4종(텍스트·펜·포스트잇·도형)뿐이라 가로 한 줄에 다 들어감. 좌측 세로(Miro·알로형)는 데스크톱 전제라 보류
- 페이지 내비는 **미리캔버스형 하단 스트립이 아닌 좌우 화살표 + N/M 표시**부터 검토 — PRD가 "좌우로 추가·이동"이라 명시했고, 20명 동시 환경에서 페이지 썸네일 패널은 과함. 단 교사 뷰에서는 썸네일이 유용할 수 있음(지휘석과 역할 중복 점검 필요)
- 포스트잇 작성자 표시는 FigJam처럼 객체 하단 모서리 small text — OQ-18의 "상시 표시" 옵션의 실존 사례

## 우리 화면에 없는 것 (벤치마크 공백 = 설계 책임)

1. **지휘석(T6)** — 전 캔버스 썸네일 모니터링은 4개 도구 모두 없음. 가장 가까운 유사물은 알로 프로젝트 카드의 진행률 표시 정도. 처음부터 설계해야 하며, 와이어프레임 검토(critique)에서 가장 공들일 화면
2. **QR 전체화면(T5)** — 교실 프로젝터 시나리오는 어느 도구에도 없음 (퀴즈앤 본체의 세션 코드 화면이 오히려 참고 대상)
3. **정원·잠금 안내(S3)** — 20명 정원, 타이머 만료 잠금 상태 화면은 유사물 없음
