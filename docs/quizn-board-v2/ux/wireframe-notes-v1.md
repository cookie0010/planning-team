## 0. 이 문서가 무엇인가

박스 레벨 와이어프레임 v1의 **카피·상태·미정 결정**을 적은 메모다. 화면 그림 자체는 Figma에 있고, 이 문서는 "왜 이렇게 그렸나 / 무엇이 아직 안 정해졌나"를 남긴다.

**박스 레벨이란:** 회색 박스 + 실제 한국어 라벨만 쓰고 색·아이콘·타이포 스타일은 넣지 않은 단계다. 목적은 **구조·위계·문구 검증**이지 미관이 아니다. 색 대비 같은 시각 항목은 하이파이 단계로 미룬다.

## 2. 핵심 카피 (실문구 — 그림에 들어간 그대로)

### 입장(D) — 참여자·뷰어
- 안내: "함께 만들기 (참여자)로 입장해요"
- 필드: "닉네임" (예: 환경지킴이) / "비밀번호 (4글자)" (○ ○ ○ ○)
- 헬퍼: "같은 닉네임+비밀번호로 다시 들어올 수 있어요"
- 버튼: "입장하기"
- 보조: "퀴즈앤 계정으로 로그인 (선택)"

### 입장(D) — 편집자
- 안내: "편집자로 입장하려면 로그인이 필요해요"
- 버튼: "퀴즈앤 계정으로 로그인" · 메모: "로그인하면 바로 편집 화면으로 들어가요."

### 공유 패널(B)
- 제목: "캔버스 공유" · 안내: "초대할 권한을 먼저 고르세요. 권한마다 QR·링크가 따로 발급돼요."
- 탭: 참여자 / 편집자 / 뷰어 (참여자 기본 선택)
- 선택 설명(참여자): "참여자: 함께 만들기. 로그인 없이 닉네임+4글자 비밀번호로 입장."
- 버튼: "링크 복사" · 힌트: "💡 프로젝터로 이 QR을 띄워 학생들이 스캔하게 하세요."

### 설정(C)
- 익명 옵션 설명: "켜면 참여자끼리는 작성자를 못 봐요. 선생님·편집자는 항상 볼 수 있어요."
- 닉네임+비번 설명: "비로그인 입장 시 항상 필요해요 (끌 수 없음)"
- 임명: "+ 편집자 임명 (로그인 사용자) · '편집자' QR로도 초대 가능"

### 안내(F) — 3종
- 정원 초과: "정원이 가득 찼어요" / "이 캔버스는 20명까지 들어올 수 있어요. 자리가 나면 다시 들어와 주세요." / [다시 들어와 보기]
- 잠금: "지금은 잠겨 있어요" / "선생님이 활동을 잠갔어요. 내가 만든 건 그대로 남아 있어요." (버튼 없음 — 읽고 대기)
- 보관: "보관된 캔버스예요" / "이 활동은 끝나서 보관되었어요. 새 활동은 선생님이 알려줄 거예요." (버튼 없음)

### 내 활동(G)
- 안내: "참여했던 캔버스로 다시 들어갈 수 있어요. 그때의 권한 그대로 들어가요."
- 권한 라벨: "✎ 참여자로 참여" (초록) / "👁 뷰어로 봄" (회색) · 버튼: "다시 들어가기"

### 빈 상태(A)
- "아직 만든 캔버스가 없어요" / "첫 캔버스를 만들어 학생들에게 QR로 공유해 보세요." / [+ 캔버스 생성]

### 뷰어 화면(E-뷰어)
- 배지: "👁 보기 전용" · 하단 안내 pill: "👁 보기 전용이에요 — 내용은 실시간으로 바뀌어요"

## 3. 상태 처리

- **빈 상태**: A-빈 상태(첫 진입) 별도 프레임. 중앙 정렬 + 단일 CTA.
- **정원 초과**: F 변형 1. **참여자·뷰어 공통**(PRD v9·IA v5: 뷰어도 정원 20명에 포함). 들어온 권한과 무관하게 "정원이 가득 찼어요"로 통일.
- **잠금(타이머 만료)**: F 변형 2(학생 안내) + B 상단 '🔒 잠금' 버튼(교사 제어) + A 카드 위 '🔒 잠금' 배지(목록에서 상태 확인). 객체 유지 메시지 명시.
- **보관**: F 변형 3.
- **실시간/presence**: B·E·E-뷰어 상단에 아바타 스택(+N), 캔버스 위 원격 커서("▷ 이름"). presence-awareness 스킬의 아바타 스택(3~5개 후 +N) 패턴 적용.
- **공감 투표 상태**: E-참여자 포스트잇에 ♡(미투표)/♥(내가 투표, 강조) 칩으로 표현. E-뷰어에는 투표 칩 없음(기본 보기 전용 — OQ-21).

## 4. 디바이스·모바일 규칙 적용 근거

- **교사 1440 / 학생 태블릿 1024(가로) / 폰 390** 프레임 사용.
- **도구바 = 하단 중앙 가로**(FigJam형) — 벤치마크 §5 + thumb zone(Hoober): 학생 디바이스 한 손 조작 시 하단이 Easy 존. 도구 4종(텍스트·펜·포스트잇·도형)이라 가로 한 줄에 수용.
- **터치 타깃 ≥44px**: 학생 도구바 셀 태블릿 80×56 / 폰 82×52, E-참여자 페이지 화살표 44×44, 입장 버튼 ~52~58px. 모두 충족.
- **폰 입장(D)**: '입장하기' 버튼을 화면 하단(thumb Easy 존)에 배치, 필드는 위에 스택. 폰 도구바는 풀폭 + 하단 safe-area 여백.
- **페이지 내비**: 좌우 화살표 + N/M(PRD "좌우 추가·이동"). 교사 B에는 '+ 페이지 추가'(최대 30) 포함, 참여자 E에는 추가 버튼 없음(navigate만), 뷰어는 'N/M 페이지' 표시만(페이지 변경 불가).

## 5. 첫 사용 2액션 동선 (그림으로 확인)

`퀴즈앤 링크 → A + T(템플릿 모달 자동) → [템플릿 고름] → B → [공유·QR] → 권한 고르고 QR`

- T 모달과 [+캔버스 생성] 후 화면은 **같은 내용**(IA: T 한 화면). 자동/수동은 진입 경로 차이.
- 공유 패널은 **'참여자' 탭을 기본 선택**해 권한 선택을 액션으로 세지 않도록 그렸다(IA §4 미정점에 대한 이번 안). → **사용자 확인 필요** (아래 6절).

## 6. 아직 안 정한 것 / 사용자 확인 필요 (미정 결정)

이번 와이어프레임에서 **한 가지 안으로 그렸지만 확정은 아닌** 것들 — 검증 게이트에서 짚어 주세요.

1. **공유 패널 기본 선택 = 참여자.** 2액션 절약을 위해 참여자 탭을 미리 선택해 그렸다. (IA §4 "QR 공유 패널 권한 선택 UX"의 한 안) — 이대로 좋은지.
2. **공유 패널 = 탭 전환식.** 권한 3종을 탭으로 전환(한 번에 한 QR). "세 QR 동시 표시"안은 안 그렸다. 교실에서 보통 한 권한만 띄우므로 탭이 단순하다고 판단 — 동의 여부.
3. **그룹 = 배경색으로 묶은 카드 영역.** 사용자 본인이 FigJam에 그린 '그룹지정 예시'(그룹명 라벨 + 캔버스 썸네일들)를 따랐다. 그룹 없는 캔버스는 "그룹 없음" 라벨 아래 평평하게. 그룹 만들기/담기 인터랙션(드래그 vs 다중선택→'그룹으로')은 아직 안 그림 — 다음 버전.
4. **템플릿 2종(빈 캔버스 제외).** PRD v9 = 빈 캔버스 + 브레인스토밍·마인드맵 3종. IA 표엔 '예비' 1칸이 더 있었으나 PRD를 따라 빈 캔버스 + 2종으로 그렸다. (OQ-03에서 3종 자체 검증 중이라 개수 유동적.)
5. **E-뷰어 페이지 변경 불가 → 화살표 없이 'N/M 페이지' 표시만.** 뷰어는 페이지를 못 바꾸므로 내비 화살표를 뺐다. 뷰어가 자기 페이지를 따로 넘길 수 있어야 하는지(편집자 따라가기 vs 자유 열람)는 미정 — 확인 필요.
6. **E-뷰어 공감 투표 없음(PRD OQ-21).** 기본 보기 전용으로 투표 칩을 뺐다. 투표 허용으로 확정되면 칩만 추가(화면 수 변화 없음).
7. **타이머 '연장' UI.** B 타이머 칩에 '⏸ ▾'(일시정지·드롭다운)만 박스로 표현. 연장·종료 세부 메뉴는 안 펼침 — 다음 버전.

## 7. 접근성 메모 (박스 레벨 — 색 대비는 하이파이로 이연)

- **터치 타깃**: 위 4절대로 ≥44px 충족(의도적 사이징).
- **입력 라벨**: D 입장 필드 모두 가시 라벨 + 헬퍼 텍스트(WCAG 3.3.2 충족).
- **시간 제한(WCAG 2.2.1)**: 타이머→잠금은 하드 시간 제한. 완화 장치 = ① E 상단 타이머 상시 노출(학생 인지), ② 만료 시 F-잠금이 "내가 만든 건 그대로 남아 있어요"로 데이터 보존 안내, ③ 교사가 타이머 연장 가능(B). 하이파이에서 '곧 종료' 사전 경고(예: 1분 전) 추가 검토 권장.
- **하이파이 이연 항목**: 색 대비 4.5:1(텍스트)·3:1(UI), 포커스 표시, 키보드 동선 상세, 비밀번호 4글자 입력의 스크린리더 처리(현재 placeholder "○ ○ ○ ○"는 하이파이에서 빈 입력 + 힌트 텍스트로 분리 처리할 것).

## 8. 상위 문서로 돌려보낼 갭 (있으면)

이번 작업 중 PRD/IA를 바꿔야 하는 갭은 **발견되지 않음.** IA v5는 PRD v9(뷰어 정원 포함·G 뷰어 재진입)와 정합하며, 그대로 그릴 수 있었다. 단 위 6절 미정점들은 와이어프레임 단계에서 사용자가 정할 사항이다.
