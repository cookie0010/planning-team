## 한 줄 요약
**같은 화이트보드 캔버스 위에서 학생들이 실시간으로 함께(공동 참여) 객체를 배치·조작하며 사고하는** 한국 교실용 협업 보드. 교사가 구조화된 틀을 깔고 실시간 지휘하며, 퀴즈앤 콘텐츠 자동변환으로 준비 마찰을 낮춘다.

> **핵심 정정 (2026-06-09)**: 모트는 템플릿도 콘텐츠 자동변환도 아니다. **"화이트보드 형식의 실시간 공동 참여(동시 공동 편집)" 그 자체**가 차별점이다. 자동변환/템플릿은 보조 무기.
>
> ⚠️ **수요 검증 판정 (2026-06-09, deep-research → `research-notes.md`)**: 이 모트의 핵심 전제 = "교실이 실시간 공동참여 캔버스를 원한다"는 **desk research로 입증도 반증도 안 됨** — revealed-demand 공개 데이터 부재. → **1차 검증(교사 인터뷰·실사용 탐침)만이 유일한 길**, PoC를 '기능 빌드'가 아니라 '수요 탐침'으로 재설계 필요. 단단한 가드레일 2개: ① **동시 자유발산을 핵심에 두지 말 것**(개인<집단 열위가 1958~ robust, "디지털이라 괜찮다"는 EBS 구제론도 검증서 0-3 폐기). ② 능동적 **공동 *구성***(개념도)은 수동 공유보다 우수(통제실험 한정)이나, "수렴에 모트 집중" 재프레이밍은 증거가 깨끗이 지지 안 함 → 확신 하향.

## 1. 합의됨 (단단한 결정)

### 배포 전략 (distribution-first)
- **위치**: 퀴즈앤 연동 별도 제품
- **비대칭 무기**: 퀴즈앤 교사 콘텐츠 자산(문항·보기·이미지)을 보드 카드로 자동변환 → 교사 준비 마찰 ≈ 0
- **시장 공백**: Jamboard 종료(2024.12.31). 구글 안내 대체재(FigJam·Lucidspark·Miro)는 전부 서구권·영어우선·비즈니스 지향. 한국어·교육과정 맞춤·저마찰 대체재 부재

### 제품 코어 primitive
- **모트 = 실시간 공동 참여(동시 공동 편집)**. 같은 캔버스 한 면에서 여러 학생이 *동시에 함께* 객체를 옮기고·묶고·연결하며 만들어감.
  - vs 수집형 담벼락(퀴즈앤 기존 보드·띵커벨·패들렛): 그건 각자 포스트잇을 *제출*해 모으는 것. "만다라트 보드"조차 8칸에 각자 채워넣는 수집형. → v2의 실시간 공동 조작은 이들이 **못 하는 것**.
  - 즉 차별점은 콘텐츠·템플릿이 아니라 **인터랙션 양식(동시성·공동성)**.
- 빈 무한 캔버스(Miro식) **금지** — 자유 캔버스가 아니라 구조화된 틀 위에서의 공동 작업
- **틀 위 배치 = 공간적 사고**: 자유 그리기가 아닌, 항목을 어디 놓을지 결정하는 사고. 교사가 틀을 깔고 실시간 지휘.
- **퀴즈앤 퀴즈와 대비**: 퀴즈는 집계 통계라 개별 산출물 소멸 → 보드는 전원 산출물이 한 공간에 공존

### 참여 구조
- **학생 디바이스**: 태블릿/크롬북
- **참여 단위**: 모둠(최대 10명) 단위 보드 + 교사 지휘석 대시보드(모둠 썸네일 갤러리 → 드릴다운)

### 범위 (scope)
- **1차 목적 A**: 학생 자유 발산/협업 경험
- **1차 목적 B**: 교사 운영 도구 성격도 인정, 시간축에서 둘 다 살림
- **명시적 제외**: 평가/형성평가/기여도 추적
- **교사의 "읽기" 정의**: 채점 아님 → 분포읽기·이상치잡기·막힌모둠돕기·좋은사례 하이라이트(선택→증폭→토론유발)

### 벤치마크 재설정
- **제품 형식 = '화이트보드(canvas)'** (Miro/FigJam 계열). Padlet식 '담벼락(wall)' 아님.
- **Padlet은 벤치마크/경쟁자에서 제외** — 그 영역(담벼락식 게시판)은 **퀴즈앤 기존 '보드'가 이미 Padlet 벤치마킹해 점유**. v2가 Padlet을 다시 베끼면 사내 자기잠식.
- **제품 라인 분리축 = wall vs canvas**
  - 퀴즈앤 기존 보드 = 담벼락(wall): 모아 붙이고 공유 (비정형 게시)
  - quizn-board-v2 = 캔버스(whiteboard): 공간에 배치·구조화 (공간적 사고)
  - → 합의된 primitive("틀 위 배치 = 공간적 사고")가 화이트보드 방향과 정확히 일치
- **레퍼런스**: Miro/FigJam (단 둘 다 비즈니스·영어우선·템플릿 과다 → 우리는 한국 교실·소수 정예 템플릿·구조화된 틀로 차별화. "빈 캔버스 Miro 클론" 함정 경계)
- **Allo**: 제외(AI로 피벗, "화이트보드는 죽었다")
- 한국 교실 사고도구: 비주얼씽킹·마인드맵·만다라트 / 협동학습: 직소·하브루타·피라미드 토론
- **Journey map**: K-12 부적합 판정

## 2. 잠정 (방향은 잡혔으나 확정 전)

### 익명/기명
- **인사이트**: "익명으로 모아 기명으로 푼다" 시퀀스 = 패들렛·멘티미터 빈틈
- **잠정 결정**: 익명/기명은 판매 시 선택 옵션. 우선 (A) 자유발산으로 시작
- (왜 잠정인지: 미해결 #2 카오스/무임승차 문제와 직결 — 풀려야 익명 default 확정)

### PoC 템플릿 3종
1. **브레인스토밍→분류(클러스터링)** — 열린 구조
2. **만다라트(3×3 고정 그리드)** — Padlet 빈틈 + 한국 인기 + 준비마찰 0
3. **마인드맵(방사형 트리)** — 위계 구조
- **선정 논리**: 열린클러스터 / 고정그리드 / 방사형트리로 분산 → 엔진 일반성 검증
- **v2로 미룸**: 2×2 매트릭스, 피라미드 토론

#### 근거 (조사 2026-06-09 / 잠정·확정아님)
> 근거 강도 순위: **마인드맵 > 만다라트 > 브레인스토밍→분류**. 단, 셋 다 "디지털 보드 전환 적합성"에는 공통 위험 있음(아래 ⚠️).

- **마인드맵 (가장 단단)**: 한국 K-12에서 가장 검증됨. 국어·과학·사회 등 전 교과 요점정리·독서 후 줄거리·교육과정 재구성에 광범위 사용. 학술 근거 존재([쓰기부진아 작문능력 효과 논문, KCI](https://www.kci.go.kr/kciportal/ci/sereArticleSearch/ciSereArtiView.kci?sereArticleSearchBean.artiId=ART001428203)), 교사 강사양성과정도 운영([쑥쑥닷컴 마인드맵 강사양성](https://m.suksuk.com/lecture/lecture_note.php?lecture_code=L_101115001)). 디지털화도 가장 흔함(깃마인드·Canva·FigJam).
- **만다라트 (중간~강)**: 교사 블로그·연수·진로교육에서 인기. 사회과 단원정리·모둠협력토의·진로계획에 사용([틀을 깬 수업: 만다라트, 아이스크림](http://sblog.i-scream.co.kr/woorissam/53), [중등 체육 만다라트·마인드맵 수업, 한교닷컴](https://www.hangyo.com/news/article.html?no=87650), [공감대화 모둠협력토의 만다라트, 다음블로그](https://blog.daum.net/orangey/12320936)). 단 학술 근거는 마인드맵 대비 약함(교사 실천 사례 중심).
- **브레인스토밍→분류 (가장 약함, ⚠️ 반대근거 있음)**: 개념 자체는 보편적이나 "초등 모둠 동시 브레인스토밍"의 효과에 의문을 던지는 국내 학술 근거 존재 — [홍기칠(2013), 사고개발 9(2)](https://www.kci.go.kr/kciportal/landing/article.kci?arti_id=ART001795407): 초5 대상에서 **개별 브레인스토밍이 집단 브레인스토밍보다 집단창의성에 더 효과적**, 브레인라이팅은 중간. → 미해결 #2(모둠 동시편집 카오스/무임승차)와 정면 충돌하는 근거. "전원 동시 발산"이 오히려 산출물 질을 떨어뜨릴 수 있음.

##### ⚠️ 디지털 보드 전환 — 단 "수집형"과 "공동 편집형"은 다르다 (정정 2026-06-09)
- 띵커벨·패들렛·퀴즈앤 기존 보드에 만다라트/마인드맵 "보드"가 있는 건 사실이나, **그건 전부 수집형 담벼락**(각자 칸/슬롯에 제출). v2가 노리는 **실시간 공동 편집(같은 면에서 동시에 함께 조작)은 이들이 못 함.**
- → 따라서 "템플릿이 이미 디지털화됨 = 모트 없음"은 **빗나간 분석**. 모트는 템플릿 유무가 아니라 **인터랙션 양식(실시간 공동 참여)**. 템플릿은 그 공동 작업을 담는 그릇일 뿐.
- 아날로그→디지털 전환 시 죽는 것: 만다라트 3×3은 디지털에서 자연스러우나, 브레인스토밍→분류의 "포스트잇 손으로 모아 옮기기"의 촉각적 협응이 약화될 수 있음.

##### 교체 후보 제안 (잠정): **브레인스토밍→분류 ↔ 비주얼씽킹**
- 비주얼씽킹은 근거 강도가 셋보다 강함: 교육부 행복교육 공식 사례([강은이 교사 비주얼씽킹 사회수업, 행복한교육](https://happyedu.moe.go.kr/happy/bbs/selectHappyArticle.do?bbsId=BBSMSTR_000000000212&nttId=4829)), 교육청 직무연수 다수([티처빌](https://www.teacherville.co.kr/trainapply/courseDetail.edu?courseSeq=O1682), 아이스크림 "참쌤의 비주얼씽킹"), 학위논문([조선대 oak](https://oak.chosun.ac.kr/bitstream/2020.oak/16955/2/비주얼%20씽킹을%20활용한%20교육방안%20연구.pdf)).
- 단 비주얼씽킹은 "그림/도식 그리기" 중심 → v2의 합의된 primitive("자유 그리기 금지, 틀 위 배치")와 충돌할 수 있음. **엔진 구조 검증 목적**(열린클러스터/고정그리드/방사형트리 분산)에는 브레인스토밍→분류가 여전히 유효하므로, 교체보다는 **#2 카오스 문제를 푼 뒤 유지 여부 재판단** 권장.

## 3. 미해결 (다음 스파링에서 깰 것 — 우선순위순)

### #1. PoC 성공 기준이 둘로 갈려 있음 (가장 시급 — 나머지 검증의 기준점)
- (a) **기술 검증**: 엔진이 세 구조를 다 그리나?
- (b) **채택 검증**: 교사가 30초 안에 켜서 실제 수업에 쓰나?
- → 안 정해지면 PoC가 무엇을 증명하는지 모호
- **정정 (2026-06-09)**: 모트가 "실시간 공동 참여"로 명확해지면서, "feasibility 낮음" 판단이 흔들림. *세 구조 그리기*는 쉬워도 *10명 실시간 동시 공동 편집의 동기화·충돌·체감 부드러움*은 진짜 미지수이자 **차별점 그 자체**. → (a)의 범위를 "세 구조 그리기"가 아니라 **"1개 틀 위에서 10명 실시간 공동 편집이 매끄럽고 카오스 없이 되나"**로 재정의해야. #1a ≈ #2와 융합됨.

### #2. 모둠 10명 동시편집의 카오스·무임승차
- (A) 자유발산 + 익명 전제에서 어떻게 감당하나?
- → 익명/기명 잠정 결정과 직결

### #3. 퀴즈앤 콘텐츠 → 카드 자동변환이 정말 비대칭 무기인가
- 문항이 분류/만다라트 카드로 의미있게 변환되나?
- → distribution 무기의 핵심 전제. 약하면 "별도 제품" 정당성 흔들림

### #4. Jamboard 난민 유인
- 무료 구글 생태계 사용자가 정말 퀴즈앤으로 올 유인이 있나?
- → distribution 쐐기의 수요측 검증

### #5. 사내 제품 라인 차별화 / 자기잠식 (신규 — 2026-06-09 추가)
- 퀴즈앤 기존 '보드'(Padlet식 담벼락)와 v2(화이트보드 캔버스)를 무엇으로 가르나?
- 분리축 = wall(모아 붙이기) vs canvas(공간 배치). 두 제품이 공존할 명분이 충분한가, 아니면 v2가 기존 보드를 흡수/대체하나?
- → 별도 제품 정당성(#3·#4와 연결)과 직결

## 의존관계 메모
- **#1(성공 기준)이 가장 상위** — 다른 검증들의 측정 기준 제공
- **#2 ↔ 익명/기명** 잠정 결정이 묶여 있음
- **#3(공급측), #4(수요측)** = distribution 가설의 양 날개
